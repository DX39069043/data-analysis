## 第一章 Pands入门

### 1.1 数据结构介绍

Pandas的两个主要数据结构：`Series`,`DataFrame`

#### 1.1.1 Series

Series 是 Pandas 的基础结构，可以看作是一个**定长的有序字典**，也可以看作是**带标签的一维数组**。



##### **1.定义与组成**

- **定义**：一种类似于一维数组的对象 
- **组成**：由一组**数据**（各种 NumPy 数据类型）和一组**数据标签**（即索引）组成 

> **Pandas Series 的“骨架”本质上就是 Numpy 数组，所以它完美继承了 Numpy 数组的各种功能和操作（例如布尔索引等等）**
>
> 我们可以把 Series 理解为：**Numpy 数组穿上了一层“标签”外衣**。

---



##### 2. **Series** 的创建

1. **由列表创建**：
   - 默认索引：`pd.Series([data])`。自动创建 0 到 N-1 的整数索引 
   - 指定索引：`pd.Series([data], index=['a', 'b'...])`。索引长度必须与数据长度一致 
2. **由字典创建**：
   - 直接传入字典：`pd.Series(dict)`。字典的键自动成为索引（**有序**排列） 
   - 传入字典 + 指定索引：`pd.Series(dict, index=[list])`
     - **匹配机制**：会根据 `index` 列表中的标签去字典中找对应的值，这样可以改变顺序
     - **缺失处理**：如果 `index` 中的标签在字典中不存在，结果为 `NaN`；如果字典中有而 `index` 中没有，则该数据被丢弃 



---



##### **3.获取数据、name属性**

- **获取数据**：`obj.values` 返回Numpy数组表示形式 

- **获取索引**：`obj.index` 返回索引对象 

- **Name 属性**：

  - Series 本身有 `name` 属性：`obj.name` 

    > 这是 Series 对象本身的名字。**如果这个 Series 变成 DataFrame 的一列，这个名字就是它的列名（到时候会自动识别）**

  - 索引也有 `name` 属性：`obj.index.name` 

    > **增加可读性：** 打印数据时，能清楚地知道索引代表什么（比如是“日期”还是“用户ID”）。

- **索引的修改**：可以通过赋值的方式**就地修改**索引，例如 `obj.index = ['Bob', ...]` 

| **位置**           | **属性**            | **作用**                         |
| ------------------ | ------------------- | -------------------------------- |
| **表头 (Header)**  | `Series.name`       | 决定了这列数据叫什么（如“价格”） |
| **索引上方的标题** | `Series.index.name` | 决定了行标签叫什么（如“商品ID”） |

```python
s = pd.Series(
    [10, 20, 30],
    index = ['001','002','003'], 
    name='价格') # 设置Series名称为'价格'
s.index.name = '商品ID' # 设置索引名称为 '商品ID'
s

```

```
商品ID
001    10
002    20
003    30
Name: 价格, dtype: int64
```

该表的含义为：ID编号为001的商品价格是10



---



##### **4.Series 的操作特性**

Series 结合了Numpy数组和字典的特性（因为他就是二者的结合）

**4.1类似 NumPy 数组的操作**

- 索引选取**：
  - 选单个值：`obj['a']` 
  - 选一组值：`obj[['c', 'a', 'd']]` (传入索引列表) 
- **数学运算**：支持布尔过滤 (`obj[obj > 0]`)、标量乘法 (`obj * 2`)、应用数学函数 (`np.exp(obj)`)
- **特性**：运算后**索引值的链接会被保留**（即数据和标签依然对应） 

**4.2类似字典的操作**

- **成员检测**：可以使用 `in` 关键字检查索引是否存在，例如 `'b' in obj` 
- **映射关系**：本质是索引到数值的映射 

**4.3自动数据对齐 (重点)**

- **功能**：在进行算术运算（如相加）时，Series 会根据**索引标签**自动对齐数据 
- **类比**：类似于数据库中的 `join` 操作 
- **结果**：索引合并，不重叠的部分会产生 `NaN` 



---



#####  **5.缺失数据处理 **

- **表示**：`NaN` (Not a Number) 表示缺失值或 NA 值 

- **检测函数**：

  - `pd.isnull(obj)` 或 `obj.isnull()`：检测是否缺失，返回布尔值 ，其中`obj`是Series名

  - `pd.notnull(obj)`：检测是否非空 

    > 输出结果是一个和obj形状一样的表格，来表示每个位置是否是NaN



#### 1.1.2 DataFrame

##### **1. DataFrame 的定义与基本创建**

- **定义**：表格型数据结构，含有一组有序的列（每一列都是一个`Series`） 。
- **特性**：
  - 每列可以是不同的值类型（数值、字符串、布尔值等） 
  - 既有行索引也有列索引，可看作是由 **Series 组成的字典**（共用同一个索引） 
  - 数据以一个或多个 **二维块** 存放（而非列表或字典） 



**最常用创建方法**：传入由等长列表或 NumPy 数组组成的字典 

```python
data = {
    'state': ['Ohio', 'Ohio', 'Ohio', 'Nevada', 'Nevada', 'Nevada'],
    'year': [2000, 2001, 2002, 2001, 2002, 2003],
    'pop': [1.5, 1.7, 3.6, 2.4, 2.9, 3.2]}
frame = pd.DataFrame(data)
```

- 字典的 **“键”（Keys）** 会直接变成 DataFrame 的列名，也就是每一列`Series`的`name`属性

- 创建后会自动加上索引，且全部列会被有序排列 

- **显示**：
  - Jupyter Notebook 中会以 HTML 表格形式呈现 
  - 对于大数据集，使用 `head()` 方法选取前五行 

---



##### **2. DataFrame 的列操作 (查改增删)**

**2.1 指定列顺序与缺失处理**

- **指定列序**：创建时通过 `columns` 参数指定列的排列顺序 
  - `pd.DataFrame(data, columns=['year', 'state', 'pop'])`
- **缺失值处理**：如果传入的 `columns` 在数据中找不到，结果中会产生缺失值 (`NaN`) 



**2.2 获取列 (查)**

- **方式一**：字典标记方式 `frame['col_name']` （最常用）
  - 适用于任何列名
- **方式二**：属性方式 `frame.col_name` 
  - 仅当列名是合理的 Python 变量名时适用
- **返回值**：返回一个 Series，拥有与原 DataFrame 相同的索引，且 `name` 属性已设置好 
- **视图 vs 副本**：通过索引返回的列是**视图**。对返回 Series 的就地修改会反映到源 DataFrame 上。如需复制，使用 `.copy()` 方法 



**2.3 修改与赋值 (改)**

- **标量赋值**：`frame['col'] = value`。会将整列设为该值 
- **数组/列表赋值**：`frame['col'] = array/list`。长度必须与 DataFrame 长度匹配 
- **Series 赋值**：`frame['col'] = series`。会精确匹配 DataFrame 的索引，空位填 `NaN` 



**2.4 添加新列 (增)**

- **直接赋值**：为不存在的列赋值会创建新列 
  - 例如：`frame2['eastern'] = frame2.state == 'Ohio'` 
- **注意**：不能用属性方式 （方法二）(`frame2.eastern = ...`) 创建新列 



**2.5 删除列 (删)**

- 使用 `del` 关键字：`del frame['col_name']` 

---



##### **3**. DataFrame 的高级创建形式

**3.1 嵌套字典 (字典的字典)**

- **结构**：`{'Key_Outer': {'Key_Inner': val, ...}, ...}` 
- **解释规则**：
  - **外层键** -> 变成 **列** (Columns) 
  - **内层键** -> 变成 **行索引** (Index) 
  - 内层键会被合并、排序以形成最终索引 
- **转置**：`frame.T` 可以交换行和列 

**3.2 Series 组成的字典**

- 用法与嵌套字典类似 
- 每个 Series 成为一列。如果没有显式指定索引，各 Series 的索引会被合并成结果的行索引 



#### 1.1.3 索引对象

##### **1. 索引对象  详解**

**1.1 定义与构建**

- 它并非只是“行号”或“表头”，实际上它是一个**功能强大的、不可变的数据结构**，是一个**有序的集合**

- **职责**：负责管理轴标签和其他元数据（如轴名称等）
- **自动转换**：构建 Series 或 DataFrame 时，所用的任何数组或序列标签都会被自动转换成一个 Index 对象 

**1.2 核心特性：不可变性 **

- **特性**：Index 对象一旦创建，用户**不能**对其进行修改（例如 `index[1] = 'd'` 会报错 ）

- **意义**：不可变性保证了 Index 对象在多个数据结构之间可以**安全共享** 

  > **Index 对象本身**是**不可变**的：你不能修改它里面的某个特定元素。
  >
  > **Series/DataFrame 的 `.index` 属性**是**可被赋值**的：你可以把旧的 Index 扔掉，换一个新的上去，这个新的index依然无法被修改，只能被替换

**1.3 集合特性 **

- **类似集合**：Index 的功能类似于一个固定大小的集合 
- **成员检测**：可以使用 `in` 关键字检查标签是否存在于索引中（例如 `'Ohio' in frame.columns`）
- **与 Python 集合的区别**：
  - Python 集合（Set）要求元素唯一。
  - Pandas Index **可以包含重复的标签** 
  - **注意**：如果选择重复的标签，会显示所有的结果 

> 比较：`DataFrame`的.index和.column
>
> 它们本质上是同一种东西**（都是 Index 对象）**，只是在 DataFrame 这个二维表格中负责的**方向不同**
>
> 你可以把 DataFrame 想象成一个坐标系：
>
> - **`.index`** 是 **纵轴（Y轴/行索引）**，负责管理**行 (Rows)**。
> - **`.columns`** 是 **横轴（X轴/列索引）**，负责管理**列 (Columns)**。
>
> | **特性**     | **.index**                                        | **.columns**                                      |
> | ------------ | ------------------------------------------------- | ------------------------------------------------- |
> | **方向**     | **垂直方向 (Axis 0)**                             | **水平方向 (Axis 1)**                             |
> | **别名**     | 行标签 (Row Labels)                               | 列名 (Column Names/Headers)                       |
> | **代表意义** | 每一行代表**“是谁”** (如：时间、用户ID、样本编号) | 每一列代表**“有什么特征”** (如：年龄、价格、销量) |
> | **切片方式** | `df.loc['行名']`                                  | `df['列名']` (直接用方括号)                       |
> | **数据类型** | `pandas.Index` 对象                               | **也是 `pandas.Index` 对象！**                    |
>
> **`.columns` 其实就是一个被放到了“表头”位置的 Index 对象。**
>
> “Index 对象不可变”、“支持集合运算”等特性，**对 `.columns` 完全适用**



##### **2. Index 的常用方法与属性**

Index 对象提供了丰富的方法来回答有关数据的常见问题或进行集合运算。

**2.1 集合运算类**

- `append`：连接另一个 Index 对象，产生一个新的 Index 
- `difference`：计算差集，得到一个新的 Index 
- `intersection`：计算交集 
- `union`：计算并集 

**2.2 数据修改/重组类 (返回新 Index)**

- `delete`：删除指定索引位置的元素 
- `drop`：删除传入的具体值 
- `insert`：在指定索引位置插入元素 

**2.3 属性与逻辑判断类**

- `isin`：计算一个布尔型数组，指示各值是否包含在参数集合中 
- `is_monotonic`：判断元素是否单调递增（即各元素均大于等于前一个元素），返回 True 
- `is_unique`：判断 Index 是否没有重复值，返回 True 
- `unique`：计算并返回 Index 中唯一值的数组 



### 1.2 基本功能

#### **1.2.1 重新索引 **

`reindex` 是 Pandas 的重要方法，用于创建一个符合新索引的新对象。

##### **1. 基本用法**

**作用**：按照参数给出的新索引的顺序，重新排序；如果新索引在原数据中不存在，填入 `NaN`



##### **2. DataFrame 的 Reindex**

- **默认**：重新排序**行**。
- **列索引**：使用 `columns` 关键字可以重新排序**列**。
  - 示例：`frame.reindex(columns=['Texas', 'Utah'])`。



##### **3. reindex的参数**

- `index` / `columns`：把行重排序还是把列重排序。
- `method`：插值方式 (如 `ffill`)。
- `fill_value`：引入缺失值时使用的替代值。
- `limit`：填充时的最大填充量。
- `copy`：默认为 True（无论如何都复制）；False 则在索引相等时不复制。



##### **4. 使用场景**

想**引入原本不存在的行（并填充默认值）**，或者**强制让数据符合某个特定的顺序和结构**，`reindex` 是唯一的选择

例：有一份销售记录，**有几天商店没开门，那天的数据在表里是不存在的。**如果你直接画图或者算“近7天平均销量”，结果绝对是错的（因为时间轴是不连续的）。这时必须用 `reindex` 强行把原本不存在的日期补进去，并填为 0

```python
# 1. 原始数据：只有 1号、3号、5号 有销量; 缺了 2号 和 4号
data = {
    '日期': ['2023-01-01', '2023-01-03', '2023-01-05'],
    '销量': [100, 200, 150]
}
df = pd.DataFrame(data).set_index('日期')


# 2. 创建一个完整的“标准日期索引” (1号到5号)
full_idx = ['2023-01-01', '2023-01-02', '2023-01-03', '2023-01-04', '2023-01-05']

# 3. 使用 reindex 重新洗牌
# fill_value=0 用0代替NaN
df_fixed = df.reindex(full_idx, fill_value=0)
```

#### **1.2.2 丢弃指定轴上的项 **

`drop` 方法用于删除指定轴上的数据，**默认返回一个新对象（即可以用另一个变量接收）**，不改变原数据。

#####  **1. Series**

- 删除单个或多个值：`obj.drop('c')` 或 `obj.drop(['d', 'c'])`。

##### **2. DataFrame**

- **删除行**（默认）：`data.drop(['Row1', 'Row2'])`。
- **删除列**：需要指定轴。
  - `axis=1` 或 `axis='columns'`。
  - 示例：`data.drop('col_name', axis=1)`。

##### **3. 就地修改 (In-place)**

- 使用 参数`inplace=True` 可以就地修改对象，**不会返回新对象**。
- **警告**：这会销毁被删除的数据，需谨慎使用。

#### **1.2.3. 索引和选取 ⭐**

##### **1. Series 索引**

- **类似 NumPy**：`obj[...]`。
- **特殊之处**：**标签切片包含末端**。
  - 普通 Python 切片 `[0:2]` 不包含下标 2。
  - Pandas 标签切片 `['a':'c']` **包含** 'c'。

##### **2. DataFrame 索引 (基础方式)**

- **选取列**：
  - 传入单个值或列表：`data['col']` 或 `data[['col1', 'col2']]`。
- **选取行 (特殊情况)**：
  - **切片**：`data[:2]` (选取前两行)。
  - **布尔索引**：`data[data['three'] > 5]` (过滤行)。
  - **布尔 DataFrame**：`data[data < 5] = 0` (用于批量修改数据)。

##### **3. 使用 loc 和 iloc 进行选取 (推荐方式)**

为了解决标签和整数索引的歧义，Pandas 提供了专门的运算符。

- **loc (标签索引)**：
  - 通过**轴标签**选取行和列。
  - 格式：`df.loc[行标签, 列标签]`。
  - 示例：`data.loc['Colorado', ['two', 'three']]`。
- **iloc (整数位置索引)**：
  - 通过**整数下标**选取行和列。
  - 格式：`df.iloc[行位置, 列位置]`。
  - 示例：`data.iloc[2, [3, 0, 1]]`。
- **功能**：两者都支持切片和单个标签选取。

**总结表：DataFrame 选取方法**

| **类型**            | **说明**                            |
| ------------------- | ----------------------------------- |
| `df[val]`           | 选取列；或是特殊的行切片/布尔过滤。 |
| `df.loc[val]`       | **按标签**选取行（单行或多行）。    |
| `df.loc[:, val]`    | **按标签**选取列。                  |
| `df.loc[row, col]`  | **按标签**同时选取行和列。          |
| `df.iloc[where]`    | **按整数位置**选取行。              |
| `df.iloc[:, where]` | **按整数位置**选取列。              |
| `df.iloc[r, c]`     | **按整数位置**同时选取行和列。      |
| `df.at[r, c]`       | 按标签选取单一标量（速度更快）。    |
| `df.iat[i, j]`      | 按位置选取单一标量（速度更快）。    |

#### **1.2.4 算术运算与数据对齐 **

Pandas 的核心特性之一是：在进行算术运算时，会自动根据**索引标签**对齐数据。

##### **1. 自动对齐机制**

- **Series 相加**：根据索引对齐。
  - 结果索引是两个对象索引的**并集**。
  - **不重叠部分**：如果某个索引只在其中一个对象中出现，结果中该位置会引入 `NaN`（缺失值）。
  - 类比：数据库的“自动外连接” (Outer Join)。
- **DataFrame 相加**：
  - 对齐操作会同时发生在**行**和**列**上。
  - 如果没有共用的列或行标签，结果都会是空。

##### **2. 填充缺失值 **

直接使用符号（`+`, `-`）运算时，缺失对应会产生 `NaN`。如果希望填充特定值（如 0），需使用对象的方法。

- **方法**：`add`, `sub`, `div`, `mul` 等。
- **参数**：`fill_value`。
- **逻辑**：如果某个轴标签在另一个对象中找不到，则先用 `fill_value` 填充该位置，再进行运算。
  - 示例：`df1.add(df2, fill_value=0)`

##### **3. 算术方法vs算术符号**

“能否填充缺失值”确实是算术方法相比于算术符号最大的优势之一。

但在实战中，并不是“尽量使用方法”，而是根据场景来定：

- **简单场景（追求代码可读性）**：用符号 `+`, `-`, `*`, `/`。
- **复杂场景（处理缺失值或调整对齐方向）**：必须用方法 `.add()`, `.sub()`等

| **算术符号** | **对应方法 (Method)**      | **反向方法 (Reverse Method)** | **说明**            |
| ------------ | -------------------------- | ----------------------------- | ------------------- |
| `+`          | **`add()`**                | `radd()`                      | 加法                |
| `-`          | **`sub()`**                | `rsub()`                      | 减法                |
| `*`          | **`mul()`**                | `rmul()`                      | 乘法                |
| `/`          | **`div()`** (或 `truediv`) | `rdiv()`                      | 真除法 (结果是小数) |
| `//`         | **`floordiv()`**           | `rfloordiv()`                 | 地板除 (整除)       |
| `%`          | **`mod()`**                | `rmod()`                      | 取模 (求余)         |
| `**`         | **`pow()`**                | `rpow()`                      | 幂运算              |

#### **1.2.5 DataFrame 与 Series 之间的运算**

类似于 NumPy 的广播机制。

##### **1. 默认行为 (匹配列，行广播)**

- Series 的索引匹配 DataFrame 的**列**。

- 运算沿着**行**向下广播（即每一行都减去该 Series）。

  > 即Series是“横”在DataFrame上面，从上往下滑动

- 如果索引找不到，会进行重新索引并形成并集（产生 `NaN`）。

##### **2. 匹配行 (在列上广播)**

- 如果希望 Series 匹配 DataFrame 的**行索引**，必须使用算术方法并指定 `axis`。

  > Series“竖”在DataFrame上，从左往右滑动

- 示例：`frame.sub(series, axis='index')` 或 `axis=0`。

#### **1.2.6 使用函数 **（后续章节详解）

##### 1. **内置函数**

pandas中内置了一些常用功能的函数，并且经过了底层优化，运算速度更快

常用的有：

| **函数名**                | **作用**      | **注意事项**                                 |
| ------------------------- | ------------- | -------------------------------------------- |
| **`sum()`**               | 求和          | 可按行或按列。                               |
| **`mean()`**              | 平均值        | 自动忽略 NaN。                               |
| **`median()`**            | 中位数        | 比平均值更抗干扰（比如工资统计）。           |
| **`max() / min()`**       | 最大 / 最小值 | 谁是第一名？谁是倒数第一？                   |
| **`idxmax() / idxmin()`** | **返回索引**  | 它不返回“最值是多少”，而是返回最值对应的索引 |
| **`count()`**             | 计数          | 只计算非空值（Non-NA）的数量。               |
| **`unique()`**            | 唯一值        | 去重，看看这列有哪些不同的值。               |
| **`nunique()`**           | 唯一值的数量  | 去重后有多少个                               |

语法：`变量名.函数名()`

其中几个最重要的内置函数会在后面详细介绍；在工作中， 90% 的需求都可以用内置函数解决。

##### **2. 自定义函数**

当 Pandas 内置的算术方法（如 `sum`, `mean`）无法满足需求时，使用`apply()` 将**自定义逻辑**（即自定义函数）应用到数据上。

`apply()` 的核心作用是将一个函数“应用”到 DataFrame 的**某一行**或**某一列**（或者 Series 的每一个元素）上。

你可以把它想象成一个**智能的循环器**：

- 它不需要你写 `for` 循环。
- 它会自动把数据拆分成一列一列（或一行一行），喂给你定义的函数。
- 它会自动把函数算出来的结果拼回成一个 Series 或 DataFrame。

| **方法**      | **适用对象**          | **作用范围**   | **典型用途**                          |
| ------------- | --------------------- | -------------- | ------------------------------------- |
| **`map()`**   | **Series/DataFrame**  | 逐元素         | **字典映射**（如把 "男" 换成 1）。    |
| **`apply()`** | **Series /DataFrame** | **整行或整列** | **聚合/复杂逻辑**（如求每行的总分）。 |

例：计算每一科目的“极差”（最大值减最小值）

```python
data = {
    '语文': [80, 90, 85],
    '数学': [70, 95, 60],
    '英语': [60, 80, 90]
}
df = pd.DataFrame(data, index=['张三', '李四', '王五'])
#自定义一个函数
def calculate_range(x):
    return x.max - x.min  #x是Series对象，可以使用Series的方法
df.apply(calculate_range,axis = 0)  #逐列应用函数
```

结果：

```
语文    10
数学    35
英语    30
dtype: int64
```

#### **1.2.7 排序和排名 **

##### **1. 按索引排序 **

- 作用**：根据行或列的索引标签（字典顺序）进行排序。
- **参数**：
  - `axis`：指定排序的轴（0 为行索引，1 为列索引）。
  - `ascending`：`False` 为降序，默认为 `True`。

##### **2. 按值排序 **

- **Series**：`obj.sort_values()`。
  - **注意**：缺失值 `NaN` 默认会被放到末尾。
- **DataFrame**：
  - **单列排序**：`frame.sort_values(by='col_name')`。
  - **多列排序**：`frame.sort_values(by=['a', 'b'])`。

##### **3. 排名 (`rank`)**

- **作用**：计算数据在数组中的排名（从 1 开始）。
- **平级关系处理 (`method` 选项)**：
  - `'average'` (默认)：相等的值分配平均排名（如两个第 6 名，排名均为 6.5）。
  - `'first'`：按原始数据出现的顺序排名（先出现的排名靠前）。
  - `'min'`：使用分组的最小排名。
  - `'max'`：使用分组的最大排名。
  - `'dense'`：类似 min，但这组排名之间没有间隔（+1 递增）。

#### **1.2.8 带有重复标签的轴索引**

Pandas 的索引标签**不必是唯一**的。

- **检查唯一性**：使用 `obj.index.is_unique` 属性。
- **数据选取行为**：
  - 如果标签对应**单个值** -> 返回标量。
  - 如果标签对应**多个值** -> 返回 Series（如果是 DataFrame 则返回 DataFrame）。
- **影响**：这会使代码变复杂，因为输出类型取决于数据内容。

## 第二章 数据加载与存储

### 前言：CSV文件介绍

csv文件全名：“逗号分隔值”文件，是只有字和逗号构成的文本文件

```
姓名,年龄,城市
张三,25,北京
李四,30,上海
王五,28,广州
```

csv文件也可以用Microsoft Excel打开，他和excel文件的区别如下：

| **特性**     | **CSV 文件 (.csv)**                                          | **Excel 文件 (.xlsx / .xls)**                                |
| ------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **存储内容** | **只有纯数据**（文本和数字）。                               | 数据 + **格式**（加粗、颜色）+ **公式** + **图表** + **图片**。 |
| **多工作表** | **不支持**。一个 CSV 只能存一张表。                          | **支持**。一个 Excel 文件可以包含多个 Sheet（工作表）。      |
| **文件大小** | **非常小**，轻量级。                                         | 相对较大，包含很多额外信息。                                 |
| **通用性**   | **全宇宙通用**。任何程序（Python, SQL, Java, 记事本）都能读。 | **专用的**。主要给微软 Office 和 WPS 用，程序读取较慢。      |
| **公式**     | 保存时公式会丢失，**只保留计算结果**。                       | 可以保存 `=SUM(A1:B1)` 这种动态公式。                        |

csv文件的优点：

1. **速度快：** Pandas 读取一个 100MB 的 CSV 可能只需要几秒钟，但读取同等大小的 Excel 文件可能需要几分钟
2. **跨平台：** 你的数据可能来自一台服务器（Server）、一个数据库或者一个网页后台，这些系统可能没有安装微软 Office，但它们都能轻松生成和读取简单的文本（CSV）。
3. **版本控制：** CSV 是纯文本，可以很方便地对比“今天改了哪一行”，而 Excel 是二进制文件，很难对比差异

#### **2.1.2`read_csv`  参数详解**

##### 第一组：基础指令

1. **`path` (文件路径)**
   - **含义：** 告诉 Pandas 读取哪个文件。
2. **`sep` (分隔符)**
   - **默认值：** 逗号 `,` （因为 CSV 全称就是逗号分隔值）。
   - **特殊情况：** 有些数据是用“制表符（Tab）”或者“分号”隔开的，这时候就要改这个参数。
3. **`encoding` (编码方式)**
   - **痛点：** 英文系统通常用 `utf-8`，中文 Windows 里的 Excel 经常搞出 `gbk` 编码。如果你打开文件发现全是**“烫烫烫”**或者乱码，就把这个改成 `'gbk'`。



##### 第二组：处理表头和索引

1. **`header` (表头/列名行)**

   - **含义：** **“第几行是表头/列名行”**
   - **默认值：** `0`（表示第一行是表头，数据从第二行开始）。
   - **特殊情况：** 如果文件里全是数据，没有表头，你要设为 `None`。

2. **`names` (自定义列名)**

   - **含义：** **“放弃csv原本自带的的表头，指定新的列表作为表头”**。
   - **用法：**传一个列表 `['名字', '年龄', '班级']`

3. **`index_col` (索引列)**

   - **含义：** **放弃默认生成的0,1,2…行索引，指定某一列来作为行索引**
   - **例：** 比如你的数据第一列是“学号”，你想让“学号”变成每一行的唯一标识（变粗的那一列），就设 `index_col=0`（第0列）。
   - `names`和`index_col`的比较：

   | **特性**               | **names**                                          | **index_col**                                                |
   | ---------------------- | -------------------------------------------------- | ------------------------------------------------------------ |
   | **管哪里？**           | **管头顶（列名）**                                 | **管左边（行索引）**                                         |
   | **是否改变数据结构？** | **否**。只是换了个标签名。                         | **是**。把某一列从数据里“提”出来变成了索引。                 |
   | **什么时候用？**       | 当原始文件的列名是乱码、空的，或者你想换英文名时。 | 当你有一列是**ID、时间、学号**，你想用它来作为查询的依据时。 |
   | **如果不设置会怎样？** | Pandas 会默认用文件的第一行作为列名。              | Pandas 会默认生成 `0, 1, 2...` 作为行索引。                  |



##### 第三组：数据清洗

1. **`skiprows` (跳过行)**
   - **含义：** **忽略前n行，从n+1行开始读取**。
   - **场景：** 有些报表前三行写着“XX公司机密文件、生成日期：xxx”，第四行才是真正的表格。这时你就设 `skiprows=3`。
2. **`na_values` (替换空值)**
   - **通俗解释：** **“什么字算是‘空’？”**
   - **场景：** 正常的空值是空的，但有的表格里会写“未知”、“Unknown”或者“?”。Pandas 不认识这些，会把它们当成普通的字。你要告诉它：“只要看到‘?’，就把它当成 NaN（空值）处理”。
   - **用法：** 添加参数`na_values=['NULL','未知','']`，用用一个列表或集合的字符串表示缺失值
3. **`parse_dates` (解析日期)**
   - **通俗解释：** **“把字符串变成真正的日期”**。
   - **场景：** 文件里的 `2023-01-01` 读进来通常只是个字符串（文本）。如果你开启这个参数，Pandas 会把它变成“时间对象”，这样你才能做“下个月”、“星期几”这种计算。



##### 第四组：处理超大文件

1. **`nrows` (读取行数)**
   - **含义：** **只读取一部分数据**。
   - **场景：** 文件有 1 亿行，无法全部读进来（内存会爆），只想读前 5 行看看长什么样。
2. **`chunksize` (块大小)**
   - **含义：** **分批次处理**。
   - **场景：** 文件太大，无法一次性读入。设置这个参数后，Pandas 会把数据切成一块一块地读取（比如一次 1000 行），适合用于循环处理。

#### 2.1.3 将数据保存到csv文件

相关函数：`to_csv`

相关参数

- `path_or_buf`：保存文件的路径或文件对象。如果不指定该参数，则返回一个表示 CSV 格式的字符串。
- `sep`：字段之间的分隔符，默认为逗号。
- `na_rep`：缺失值的表示，默认为空字符串。
- `columns`：要写入文件的列，默认为所有列。
- `header`：是否将列名写入文件，默认为 True。
- `index`：是否将索引写入文件，默认为 True。
- `mode`：打开文件的模式，{‘w’, ‘x’, ‘a’}，默认为 `'w'`（覆盖写入）。

### 2.2 处理Excel文件

#### 2.2.1 读取指定工作表

一个 Excel 文件里可能有几十个 Sheet（工作表），你必须告诉 Pandas 读哪个。

- **`sheet_name` (指定工作表)**
  - **含义**：**读取指定工作表**
  - **默认值**：`0` （读取第一个 Sheet）。
  - **常用写法**：
    - `sheet_name='Sheet1'`：按名字读（推荐，最不容易错）。
    - `sheet_name=1`：按顺序读（读取第2个 Sheet，因为索引从0开始）。
    - `sheet_name=None`：**一次性读取所有 Sheet**（返回一个字典，Key是表名，Value是 DataFrame）。
  - **场景**：财务报表通常一个月一个 Sheet，你需要循环读取。

------

#### 2.2.2 定位数据范围

Excel 表格通常很乱，不像 CSV 那么规整。有时候数据是从第 5 行开始的，有时候只想要 A 到 C 列。

- **`header` (表头行)**
  - **含义**：**“第几行是标题？”**
  - **用法**：和 `read_csv` 一样。
  - **注意**：如果你设了 `header=1`，Pandas 就会自动忽略第 0 行的数据。
- **`usecols` (筛选列)**
  - **含义**：**“我只要这几列，其他的杂乱数据别读进来。”**
  - **痛点**：Excel 右边经常有人写备注、草稿数据，如果不限制，Pandas 会读出一堆 `Unnamed: 99` 这种空列。
  - **常用写法**：
    - `usecols="A:C"`：读取 A 列到 C 列。
    - `usecols="A,C,E"`：读取 A、C、E 列。
    - `usecols=[0, 2]`：读取第 0 列和第 2 列。
- **`skiprows` (跳过行)**
  - **含义**：**“跳过上面几行废话。”**
  - **场景**：很多 Excel 报表最上面几行是“xxx公司2023年度报表”、“单位：万元”这种合并单元格的大标题。如果不跳过，Pandas 会报错或读乱。

------

#### 2.2.3 数据类型参数

- **`index_col` (索引列)**
  - **含义**：**选取一列当行索引**
  - **用法**：和 `read_csv` 完全一样。比如 `index_col='ID'`。
- **`dtype` (强制类型)**
  - **含义**：**“这一列必须是字符串，别给我转成数字。”**
  - **痛点**：**身份证号**、**股票代码（00700）**。
  - Excel 会自动把 `00700` 变成 `700`。
  - **解决**：`dtype={'股票代码': str}`，这样 `00700` 就会被保留为文本。



#### 2.2.4 `read_excel` 和 `read_csv`参数对比

| ------------ | ------------ | ---------------- | --------------------------------------------- |
| **文件来源** | `path`       | `io` (path)      | 一样。                                        |
| **选表**     | **无**       | **`sheet_name`** | **核心区别！Excel 必须选表。**                |

### 2.3 读取MySQL数据库

#### 2.3.1 数据库驱动与连接引擎

1. 数据库驱动：安装`pymysql`软件包

2. 连接引擎`SQLAlchemy`

`SQLAlchemy`的核心就是一个函数：**`create_engine`**。 用它来生成一个“连接对象”，然后把这个对象传给 Pandas：

```python
from sqlalchemy import create_engine

# 1. 准备连接信息 (这是最关键的一步)
# 格式：数据库类型+驱动://用户名:密码@主机IP:端口/数据库名
db_url = "mysql+pymysql://root:123456@localhost:3306/test_db"

# 2. 创建引擎 (这一步通常不会报错，除非 URL 格式写错)
engine = create_engine(db_url)

# 3. 传给 Pandas 使用
import pandas as pd
df = pd.read_sql("SELECT * FROM users", engine)
```

特殊情况：密码里有特殊字符，例如 `@`、`:` 或者 `/` 

解决办法：先使用 `urllib.parse.quote_plus` 对密码进行编码

```python
import urllib.parse  # 引入 URL 处理库

# 假设你的密码很复杂
password = "My@Password:123"

# 先编码
encoded_password = urllib.parse.quote_plus(password)

# 然后再拼接到 URL 里
db_url = f"mysql+pymysql://root:{encoded_password}@localhost:3306/test_db"

engine = create_engine(db_url)
```



#### 2.3.2 核心函数与参数

核心函数：`pd.read_sql()`

```python
pd.read_sql(
    sql, 
    con, 
    index_col=None, 
    coerce_float=True, 
    params=None, 
    parse_dates=None, 
    columns=None, 
    chunksize=None)
```

常用参数详解

| **参数**          | **含义**            | **说明**                                                     |
| ----------------- | ------------------- | ------------------------------------------------------------ |
| **`sql`**         | **SQL语句 或 表名** | 这是最核心的参数。 1. **写查询语句**：`"SELECT * FROM users WHERE age > 18"` 2. **写表名**：`"users"` (它会自动读取整张表) |
| **`con`**         | **连接对象**        | 通常是 SQLAlchemy 的 `engine` 对象。它告诉 Pandas 连哪个库、用户名密码是多少。 |
| **`index_col`**   | **索引列**          | **非常常用**。指定某一列（比如 `id` 或 `date`）直接变成 DataFrame 的索引 (Index)。如果不指定，Pandas 会自动生成 0, 1, 2... |
| **`parse_dates`** | **解析日期**        | 告诉 Pandas 哪几列是时间，读取时直接转为 `datetime` 类型，省得读完后再用 `pd.to_datetime` 转。 |
| **`params`**      | **查询参数**        | 用于安全地传递参数（防止 SQL 注入）。比如 `sql="SELECT * FROM table WHERE id = %s"`, `params=[10]`。 |
| **`chunksize`**   | **分块读取**        | 如果表里有 1000 万行数据，一次读会撑爆内存。设置 `chunksize=10000`，它会返回一个迭代器，让你一批一批地处理。 |



## 第三章 数据清洗和准备

### **3.1 处理缺失数据 **

**缺失值的表示**

- **数值数据**：使用浮点值 `NaN` (Not a Number) 作为“哨兵值”。
- **对象/字符串**：使用 `NaN` 或 Python 内置的 `None`。
- **统计特性**：Pandas 的描述性统计方法（如 sum, mean）默认都会**排除**缺失数据。

**检测缺失值**

| **方法**        | **说明**                           |
| --------------- | ---------------------------------- |
| **`isnull()`**  | 返回布尔对象（True 表示缺失）。    |
| **`notnull()`** | isnull 的否定式（True 表示非空）。 |

------

#### **3.1.1 过滤缺失数据**：dropna

`dropna` 用于丢弃含有缺失值的行或列。

##### **1. Series**

- `data.dropna()`：返回一个仅含非空数据和索引值的 Series。
- 等价于 `data[data.notnull()]`。

##### **2. DataFrame**

- **丢弃行 (默认)**：`data.dropna()`。
  - 只要某行包含**任何** NaN，该行就会被丢弃。
- **只丢弃全为 NA 的行**：`data.dropna(how='all')`。
- **丢弃列**：传入 `axis=1` 或 `axis='columns'`。
  - 示例：`data.dropna(axis=1, how='all')`。
- **阈值控制 (`thresh`)**：
  - `thresh=n`：保留至少有 `n` 个非 NA 值的行（即丢弃非 NA 值少于 n 的行）。
  - 示例：`df.dropna(thresh=2)`。

------

#### **3.1.2 填充缺失数据**：fillna

`fillna` 用于将缺失值替换为指定值或使用插值方法填充。



##### **1. 基础用法**

- **填充常数**：`df.fillna(0)`（将所有 NaN 替换为 0）。
- **字典填充**：为不同的列指定不同的填充值。
  - 示例：`df.fillna({1: 0.5, 2: 0})`（第 1 列填 0.5，第 2 列填 0）。
- **统计值填充**：使用平均值或中位数填充。
  - 示例：`data.fillna(data.mean())`。



##### **2. 其他常用参数**

1. `method`：

**含义：如果没有材料，如何填充？**

- **两个核心选项：**
  - **`ffill` (Forward Fill)**：**“抄上面的”**。比如如果不记得今天股价多少，就默认等于昨天的。这是处理时间序列最常用的。
  - **`bfill` (Backward Fill)**：**“抄下面的”**。（比较少用，因为通常无法预知未来）。

2. `axis`

- **含义：** **填充方向，横着填还是竖着填**。
- **`axis=0` (默认)**：**竖着填**（按行操作）。如果你用 `ffill`（抄上面的），就是竖着看，把上一行的值拿下来填。
- **`axis=1`**：**横着填**（按列操作）。拿左边列的值填右边列的坑（很少用，除非你在做特殊的表格整理）。

3. `inplace`

- **含义：** **是否就地修改**
- **`inplace=False` (默认)**：Pandas 会给你看一张“修补好后的**新表格**”，但**原来的表格**（原变量）还是破破烂烂的。如果你不把结果赋值给新变量，操作就白做了。
- **`inplace=True`**：Pandas 直接在**原表格**上修改，原来的空值彻底被覆盖了。

4. `limit`

- **含义：** **连续填充的上限，最多允许抄几次？**。
- **场景：** 假设你用 `ffill`（抄上面的）。如果只有 1 个空值，抄昨天的没问题。但如果中间**连续空了 100 行**（比如传感器坏了 3 天），你还全部填成 3 天前的数据，那数据就太假了。
- **用法：** 设置 `limit=2`，意思就是：遇到连续的空值，我只填补前 2 个，后面的还是让它空着吧，因为隔太久了，参考价值不大。

------



### 3.2 数据预处理

#### **3.2.1 删除重复数据**

- **`duplicated()`**：
  - 返回一个布尔型 Series，表示各行是否是**重复行**（即前面出现过的行）。
- **`drop_duplicates()`**：
  - 返回一个移除了重复行的 DataFrame。
- **参数**：
  - **默认**：判断**全部列**。
  - **指定列**：传入列名列表，如 `['k1']`，只根据这些列判断重复。
  - **`keep`**：
    - 默认保留**第一个**出现的值。
    - `keep='last'`：保留**最后一个**出现的值。

------

#### **3.2.2 利用函数或映射进行数据处理**

- **场景**：根据 Series 或 DataFrame 列中的值来实现转换（如将“肉类名”映射为“动物名”）。
- **方法**：`series.map(arg)`。
- **参数 `arg`**：
  - **字典**：定义映射关系（如 `{'bacon': 'pig'}`）。
  - **函数**：对每个元素应用函数。
- **注意**：在使用字典映射前，通常需要先统一数据格式（如 `str.lower()` 转小写），以确保匹配成功。

------

#### **3.3.3 替换值 **

- **作用**：`fillna` 是值替换的特殊情况，`replace` 提供更通用的替换功能。
- **用法**：
  - **单值替换**：`data.replace(-999, np.nan)`。
  - **多值替换**：
    - 列表对单值：`data.replace([-999, 1000], np.nan)`。
    - 列表对列表：`data.replace([-999, 1000], [np.nan, 0])`（一一对应）。

#### **3.3.4 重命名轴索引 **

- **`rename` 方法**：
  - 返回新对象（默认不修改原数据）。
  - **结合字典**：可只更新部分标签。
  - 例：`data.rename(index={'OHIO': 'INDIANA'}, columns={'three': 'peekaboo'})`。
  - 参数 `inplace=True` 可实现就地修改。

#### **3.3.5 数据分类 **

把**连续的数值**按照大小，映射成有限的**类别标签**。

例如：现在有 1 万个用户的年龄（1岁到 100岁都有），想做分析。 如果你直接用原始年龄，会有 100 个柱子，图表太乱了。 你需要把它变成：

- 0-18岁 (未成年)
- 18-35岁 (青年)
- 35-60岁 (中年)
- 60+ (老年)

这样更方便进行分析。下面介绍两个常用的分类函数，二者的返回结果都是`Categorical` 对象



##### **1. 按照数值距离划分：`pd.cut` **

**核心逻辑：** 关注的是 **“数值的大小”**。如果让它自动分 4 组，它会算出 `(最大值 - 最小值) / 4`，然后按这个间距切分。

常用参数含义：

- **`x`**：你要切的那一列数据。
- **`bins`**（最重要）：
  - **传整数（比如 4）**：自动分成 4 等份（每组的数值跨度一样）。
  - **传列表（比如 [0, 60, 100]）**：**手动指定在哪划分**（最常用）。
- **`right`**：`True`（默认）还是 `False`。
  - 默认是 `(a, b]`（**左开右闭**）。意思是不包含左边，包含右边。
  - 比如 `(0, 60]`：0 不算，60 算。
- **`labels`**：设置标签名称（比如 `['不及格', '及格']`）。



##### **2. 按照数量比例划分：`pd.qcut` **

**核心逻辑：** 它关注的是 “排名的比例”。`q` 代表 Quantile（分位数）。如果让它分 4 组，它会保证 **每一组里的人数是一样多的**。

常用参数含义：

- **`x`**：数据。
- **`q`**（最重要）：
  - **传整数（比如 4）**：按照 25%, 50%, 75% 切分（四分位数）。
  - **传列表（比如 [0, 0.8, 1]）**：我要前 80% 的人一组，后 20% 的人一组。
- **`labels`**：设置标签名称。





------

#### **3.3.6 检测和处理异常值 **

**核心思想**：运用数组运算。

**第一步：检测异常值**：例如找出绝对值大于 3 的值。

- `np.abs(data) > 3`。



**第二步：找出包含异常值的行**：使用 `any(1)` 函数

- `data[(np.abs(data) > 3).any(1)]`

原理解释：假设有一个 3行 x 3列 的表格：

|         | **A** | **B**   | **C**    |
| ------- | ----- | ------- | -------- |
| **行0** | 0.1   | 0.5     | -0.2     |
| **行1** | 0.2   | **5.5** | 0.3      |
| **行2** | -0.1  | 0.1     | **-4.0** |

1. **`np.abs(data) > 3`** 会变成一个全是 True/False 的表格：

- 行0：[False, False, False]
- 行1：[False, **True**, False]
- 行2：[False, False, **True**]

2. **`.any(1)`**：这是关键。

- `1` 代表 **axis=1（横向/按行扫描）**。
- `any` 代表 **“只要有一个是 True，我就算 True”**。
- **结果**：
  - 行0：没有 True -> **False**
  - 行1：有一个 True -> **True**
  - 行2：有一个 True -> **True**

3. 最终筛选：Pandas 会根据上面的结果，把 **行1** 和 **行2** 拿出来给你看。



- **第三步：处理异常值** ：将异常值修正到合理的值
  - 例如上面的例子中，可以这样处理`data[np.abs(data) > 3] = np.sign(data) * 3`。

------

#### **3.3.8 排列和随机采样 **

- **随机重排序**：
  1. `np.random.permutation(length)`：生成一个随机整数序列。
  2. `df.take(sampler)` 或 `df.iloc[sampler]`：按序列重排数据。
- **随机采样 (`sample`)**：
  - `df.sample(n=3)`：随机抽取 3 行。
  - `replace=True`：允许重复选择（有放回采样）。

------

#### **3.3.8 计算指标/哑变量 **

常用于统计建模或机器学习（One-Hot Encoding）。

- **`pd.get_dummies(Series)`**：
  - 将分类变量转换为 0/1 指标矩阵。
  - 如果一列有 k 个不同值，则派生出 k 列。
- **前缀**：`prefix='key'`（给列名加前缀，便于合并）。
- **多重归属问题**（如一部电影属于 "Action|Comedy"）：
  - 需要结合字符串分割与迭代。
  - 步骤：
    1. 构建全零矩阵。
    2. 使用 `dummies.columns.get_indexer` 获取对应的列索引。
    3. 将对应位置设为 1。
- **结合离散化**：`pd.get_dummies(pd.cut(values, bins))`。



### 3.3 处理字符串

#### **3.3.1 Python的字符串方法 **

Python 内置的字符串方法简单易用，适合处理大部分基础文本操作。

##### **1. 拆分与连接**

- **`split`**：按指定分隔符拆分字符串。
  - 示例：`'a,b,c'.split(',')` -> `['a', 'b', 'c']`。
  - 方法可以实现**分列**的操作
- **`join`**：将序列中的元素连接成一个字符串（推荐方式，比 `+` 更快）。
  - 示例：`'::'.join(['a', 'b', 'c'])` -> `'a::b::c'`。

##### **2. 去除空白**

- **`strip`**：去除两端空白符（包括换行符）。
- **`lstrip`** / **`rstrip`**：仅去除左边或右边的空白符。
- 常与 `split` 结合使用：`[x.strip() for x in val.split(',')]`。

##### **3. 子串定位与检测**

- **`in` 关键字**：检测子串是否存在（最佳方式）。
  - 示例：`'guido' in val` -> `True`。
- **`index`**：返回子串第一个字符的位置。
  - **注意**：如果找不到，会抛出 `ValueError` 异常。
- **`find`**：返回子串第一个字符的位置。
  - **注意**：如果找不到，返回 `-1`。
- **`count`**：返回指定子串出现的次数。

##### **4. 替换**

- **`replace`**：将指定模式替换为另一个模式。
  - 删除模式：传入空字符串 `''` 作为替换目标。

##### **5. 大小写转换**

- **`lower`** / **`upper`**：转小写 / 转大写。
- **`casefold`**：转为通用的可比较形式（比 lower 更激进，适合多语言）。

#### **3.3.2 正则表达式 **

用于复杂的模式匹配和文本操作。Python 使用 `re` 模块。

##### **1. 常用函数 (re 模块)**

- **`re.split`**：根据正则模式拆分字符串。
  - 示例：`re.split('\s+', text)`（按任意数量空白符拆分）。
- **`re.compile`**：编译正则表达式对象。

- **`sub` 的引用**：可以使用 `\1`, `\2` 符号引用第一个、第二个分组的内容。

## 第四章 数据表的合并与调整

在许多应用中，数据可能分散在许多文件或数据库中，存储的形式也不利于分析。本章关注可以聚

### 4.1 多级索引

含义：**在一个轴（行或列）上拥有两个或两个以上的索引层级**

> 像一本教科书，有**章**，章下面有**节**，节下面才是**内容**。



#### 4.1.1 创建多级索引

1. 将已有数据转换为多级索引`set_index([索引名列表])`
   ```python
   data = {
       '班级': ['一班', '一班', '二班', '二班'],
       '姓名': ['张三', '李四', '王五', '赵六'],
       '分数': [80, 90, 85, 95]
   }
   df = pd.DataFrame(data)
   
   # 把“班级”设为第一层，“姓名”设为第二层
   df_multi = df.set_index(['班级', '姓名'])
   # 注意要用另一个变量去接收，原DataFrame不会改变
   df
   ```

   逆运算：`rest_index`，把索引重新变成列

2. 索引数据分散在两个独立的列表里，把它们“拉链”式地拼起来`pd.MultiIndex.from_arrays`

   ```python
   # 两个等长的列表
   arrays = [
       ['一班', '一班', '二班', '二班'],  # 第一层
       ['男', '女', '男', '女']         # 第二层
   ]
   
   # 拼装
   index = pd.MultiIndex.from_arrays(arrays, names=['班级', '性别'])
   
   df = pd.DataFrame({'人数': [10, 20, 15, 25]}, index=index)
   print(df)
   ```

多级索引生成的索引对象是 `MultiIndex`，包含 `levels`（唯一值）和 `labels`（整数标签），他继承了`Index`，是他的**子类**



#### 4.1.2 多级索引的选取和切片

| **你的需求**                          | **推荐方法**     | **代码示例**               |
| ------------------------------------- | ---------------- | -------------------------- |
| **只选最外层** (比如所有 2023)        | **直接 `loc`**   | `df.loc['2023']`           |
| **精准选某一行** (比如 2023 年的上海) | **元组 `loc`**   | `df.loc[('2023', '上海')]` |
| **只选内层** (跨层选，比如所有上海)   | **利用冒号 `:`** | `df.loc[:,'上海']`         |



#### 4.1.3 调整层级顺序以及重排序

在处理多级索引时，经常需要调整索引的层级顺序（比如把“内层”换到“外层”），这就是 `swaplevel` 的作用。

但调整完顺序后，数据的物理排列往往是乱序的，这会导致切片报错或性能下降，所以紧接着必须用 `sort_index` 来重新整理数据。

1. `swaplevel()`：交换层级（换位置）

作用：**调整索引层级的顺序**。

- **形象比喻：**

  你有一个衣柜，原来是按 **“年份-> 城市”** 分类的

  现在你想改成按 **“城市 -> 年份”** 分类。这就是 `swaplevel`。

- **语法：**

  ```python
  # i, j 是你要交换的两个层级的名称
  df.swaplevel(i, j, axis=0)
  ```

- **代码演示：**

  ```python
  # 1. 原始数据：第一层是年份，第二层是城市
  index = pd.MultiIndex.from_product(
      [
          ['2023', '2024'], 
          ['北京', '上海']], 
      names=['年份', '城市'])
  df = pd.DataFrame({'销量': [100, 200, 150, 300]}, index=index)
  
  # 2. 交换层级：把“城市”换到外层，“年份”换到内层
  df_swapped = df.swaplevel('年份', '城市')
  df_swapped
  ```

  **输出结果：**

  ```
             销量
  城市  年份      
  北京  2023   100  <-- 北京
  上海  2023   200  <-- 上海 (乱入)
  北京  2024   150  <-- 北京 (又回来了)
  上海  2024   300
  ```

  **问题来了：** 虽然层级换了（城市跑到左边了），但数据并没有重新排序。北京和上海是**穿插**在一起的。这时候如果你想进行范围切片（比如取“北京”到“上海”），Pandas 会直接报错。这是就需要重新进行排序

------

2. `sort_index()`：重新排序（理顺）

它的作用是：**把索引按顺序排列整齐**（通常是字典序，A-Z 或 0-9）。

- **为什么必须做这一步？**

  Pandas 的切片操作（Slicing）强烈依赖于索引的有序性。如果索引是乱的，`loc` 切片会失效，性能也会极差。

- **语法：**

  ```python
  # level: 指定按哪一层排（通常不写，默认全排）
  # ascending: True(升序) / False(降序)
  df.sort_index(level=0, inplace=True)
  ```

- **接上面的代码演示：**

  ```python
  # 3. 对交换后的数据进行排序
  df_sorted = df_swapped.sort_index()
  print(df_sorted)
  ```

  **输出结果：**

  ```
             销量
  城市  年份      
  上海  2023   200  <-- 上海的数据聚在一起了
      2024   300
  北京  2023   100  <-- 北京的数据聚在一起了
      2024   150
  ```

  现在，你可以放心地使用 `df_sorted.loc['上海']` 了。



在实际工作中，这两个函数通常是连在一起写的：

```python
df_final = df.swaplevel('年份','城市').sort_index()
```



### 4.2 合并数据集

#### 4.2.1 SQL风格的合并

函数`pd.merge` 可根据一个或多个键将不同的 DataFrame 连接起来，类似于 SQL 中的 JOIN 操作

##### **1. 基本用法**

- **默认行为**：

  - 如果没有指定键，merge 会将**重叠列的列名**当作键。
  - 默认执行 **"inner"** 连接（取交集）。

- **指定连接键**：

  - `on='key'`：指定列名。

  - `left_on='lkey', right_on='rkey'`：当左右列名不同时分别指定

    > 类似SQL中指定外键和主键

##### **2. 连接方式 (`how` 参数)**

| **选项**    | **说明**                            |
| ----------- | ----------------------------------- |
| **`inner`** | (默认) 使用两个表都有的键（交集）。 |
| **`left`**  | 使用左表中所有的键。                |
| **`right`** | 使用右表中所有的键。                |
| **`outer`** | 使用两个表中所有的键（并集）。      |
| `cross`     | 产生笛卡尔积                        |

> 类似SQL中的内连接、外连接、左/右连接

##### **3. 多对多合并 **

- 如果两个表是多对多的关系，则产生的是行的**笛卡尔积**。
- 例如：左表有 3 个 "b"，右表有 2 个 "b"，结果中会有 6 个 "b" 行。

##### **4. 处理重复列名 **

- 参数：`suffixes`

- 当左右两个 DataFrame 有重叠的列名（非连接键）时，merge 会自动添加后缀 `_x`, `_y`。
- 可通过 `suffixes=('_left', '_right')` 自定义后缀。

##### **5. 使用索引作为连接键**

- **`left_index=True`** / **`right_index=True`**：使用左表/右表的索引作为连接键。

------

#### **4.2.1 轴向连接 **

`pd.concat` 用于沿着一条轴将多个对象堆叠到一起。他的核心思想是沿着某条轴（x轴或y轴），把两个表硬生生地“粘”在一起

> **场景：** 1月份的数据表 + 2月份的数据表 +...+ 12月份的数据表= 总数据表。

##### **1. 基本用法**

- **`axis=0` (默认)**：沿着行堆叠（纵向拼接）
- **`axis=1`**：沿着列堆叠（横向拼接）

##### **2. 连接逻辑**

- 参数：**`join` **
  - `outer` (默认)：取索引的并集。
  - `inner`：取索引的交集。

##### **3. 设置多级索引**

- 参数：`key`

- **作用**：在连接轴上创建一个**多级索引**，用于区分数据来源。

- **Series**：`concat([s1, s2], keys=['one', 'two'])` -> 在最外层创建名为`one`、`two`的索引

  > 场景：设置为 1月份的数据表 、 2月份的数据表，用以区分数据属于哪个月

- **DataFrame**：

  - `axis=1` 时，`keys` 会成为表头（Column Headers）。
  - `axis=0` 时，`keys` 会成为行索引的外层。

##### **4. 忽略索引 **

- 参数：`ignore_index`

- 如果行索引没有实际意义，可设置 `ignore_index=True`。
- 结果会产生一组新的整数索引 `0, 1, ..., n-1`。

**concat 常用参数表**

| **参数**           | **说明**                                   |
| ------------------ | ------------------------------------------ |
| `objs`             | 参与连接的 Pandas 对象列表或字典（必需）。 |
| `axis`             | 指明连接的轴向（默认 0）。                 |
| `join`             | 'inner' 或 'outer'（默认）。               |
| `keys`             | 用于形成多级索引，标识数据来源。           |
| `names`            | 用于创建分层级别的名称。                   |
| `ignore_index`     | 不保留连接轴上的索引，产生新索引。         |
| `verify_integrity` | 检查结果新轴上的重复情况（默认 False）。   |

------

#### **4.3 合并重叠数据**

用于处理两个数据集索引部分重叠，且需要“打补丁”的情况。

- **`combine_first`**：
  - 用传递对象（参数）中的值，填充调用对象（调用者）中的缺失值。
  - **即：优先用自己的数据，如果自己是 NaN，就用别人的数据补上。**
- **示例**：`df1.combine_first(df2)` -> 用 `df2` 修补 `df1` 的空缺。
- 同时也带有 Pandas 的**数据对齐**功能（自动对齐索引）。



### 4.3 重塑和轴向旋转

#### 4.3.1 重塑多级索引

多级索引为 DataFrame 数据的重排任务提供了一种一致性的方式。

- **`stack()` (堆叠)**：
  - **作用**：把最里面的**列索引 (Columns)** 旋转 90 度，压到**行索引 (Index)** 的最内层。
  - **结果**：表格变长、变瘦了。行索引层级变多了。
  - **缺失值处理**：默认会**滤除**缺失数据（NaN），使运算可逆。传入 `dropna=False` 可保留 NaN。
  - **指定级别**：可以传入分层级别的编号（如 `0`）或名称（如 `'state'`）来指定旋转哪一层。
- **`unstack()` (拆堆)**：
  - **作用**：把最里面的**行索引 (Index)** 旋转 90 度，推到**列索引 (Columns)** 上去。
  - **结果**：表格变宽、变扁了。列索引层级变多了。
  - **默认行为**：操作的是**最内层**索引。
  - **缺失数据引入**：如果不是所有的级别值都能在各分组中找到，unstack 操作可能会引入缺失数据（NaN）。

------

#### **4.3.2 将“长表”旋转为“宽表”**

时间序列或数据库数据常以“长表”或“堆叠格式”存储（每行代表一次观察，列包含主键和值）。分析时常需转换为“宽表”。

- **`pivot()` 方法**：
  - **作用**：将“长表”数据转换为“宽表” 。
  - **参数**：
    
    1. `index`： 指定某一列作为新表的 **行索引**（左边的行号） 。
    2. `columns`（核心）：指定某一列作为新表的 **列名**（上面的表头），把原来的“值”变成了“列名”。
    
       > 即把原来**竖着排一列的数据横过来，作为表头**
    3. `values`（可选）： 指定哪一列的数据填入 **单元格** 中。
       - **多级索引**：如果不填这个参数，Pandas 会默认把**剩下所有列**都作为数据填进去（形成多级**列**索引，其中`columns`的列是最内层）。
  - **本质**：`pivot` 其实就是 `set_index` 创建多级索引，再用 `unstack` 重塑的快捷方式。
    - 即：`df.pivot('date', 'item', 'value')` 等价于 `df.set_index(['date', 'item'])['value'].unstack('item')`。

场景：有一份简单的销售数据（长表）

```python
data2 = {
    '日期': ['1月1日', '1月1日', '1月2日', '1月2日'],
    '商品': ['苹果', '香蕉', '苹果', '香蕉'],
    '销量': [100, 200, 150, 300],
    '价格': [5, 3, 5, 3]
}
df2 = pd.DataFrame(data2)

print("--- 原始长表 ---")
print(df2)
```

原始数据：

```
日期  商品   销量  价格
0  1月1日  苹果  100   5
1  1月1日  香蕉  200   3
2  1月2日  苹果  150   5
3  1月2日  香蕉  300   3
```

要求：我想看“不同日期的商品销量”

```python
df2_pivoted = df2.pivot(index='日期', columns='商品', values='销量')
print(df2_pivoted)
```

结果：（宽表）

```
商品    苹果   香蕉
日期           
1月1日  100  200
1月2日  150  300
```

如果不填`values`参数：形成多级索引

```
		 销量       	 价格     
商品   苹果   香蕉   苹果  香蕉
日期                     
1月1日  100  200    5   3
1月2日  150  300    5   3
```



------

#### **4.3.3 将“宽表”旋转为“长表”**

旋转 DataFrame 的逆运算是 `melt`。它将多个列合并成为一个，产生一个比输入长的 DataFrame。

- **`pandas.melt()` 函数语法**：
  
  ```python
  pd.melt(frame, 
          id_vars=None, 
          value_vars=None, 
          var_name=None, 
          value_name='value')
  ```
  
  **作用**：将“宽表”数据转换为“长表”。
  
  
  
  参数含义：
  
  1. **`id_vars` (身份列/锚点)**：
     - **含义：** 你不想融化的列。
     - **作用：** 这些列会作为**标识符**保留下来，每一行数据都会重复携带这个标识。
     - **例子：** “姓名”、“学号”。
  2. **`value_vars` (要融化的列)**：
     - **含义：** 你要把哪些列的**表头**变成数据？
     - **作用：** 这些列会被“折叠”进一列里。如果不填，默认融化除了 `id_vars` 以外的所有列。
     - **例子：** “语文”、“数学”、“英语”。
  3. **`var_name` (新变量名)**：
     - **含义：** 原来的**表头**（语文、数学）变成了新的一列，给这新的一列起个名字。
     - **例子：** 叫它 “科目”。
  4. **`value_name` (新数值名)**：
     - **含义：** 原来的**数值**（90, 80）变成了新的一列，给这新的一列起个名字。
     - **例子：** 叫它 “分数”。

例：有一张典型的“宽表”成绩单

```python
df = pd.DataFrame({
    '姓名': ['张三', '李四'],
    '语文': [90, 80],
    '数学': [85, 95]
})

print("--- 原始宽表 (Wide) ---")
print(df)
```

结果

```python
   姓名  语文  数学
0  张三  90  85
1  李四  80  95
```

把“语文”和“数学”这两列融化掉，变成“科目”，剩下的数据形成新的一列，取名“分数”

```python
df_melted = pd.melt(df, 
                    id_vars=['姓名'], 
                    value_vars=['语文', '数学'],
                    var_name='科目', 
                    value_name='分数')

print("\n--- 融化后的长表 (Long) ---")
print(df_melted)
```

```
   姓名  科目  分数
0  张三  语文  90  <-- 张三的第一条数据
1  李四  语文  80
2  张三  数学  85  <-- 张三的第二条数据 (姓名重复了)
3  李四  数学  95
```

四大变形函数的对比

| **特性**     | **pivot**             | **melt**                  | **stack**              | **unstack**                 |
| ------------ | --------------------- | ------------------------- | ---------------------- | --------------------------- |
| **变换方向** | **长 $\to$ 宽**       | **宽 $\to$ 长**           | **宽 $\to$ 长**        | **长 $\to$ 宽**             |
| **操作对象** | **列里的值** (Values) | **列的名字** (Headers)    | **列索引**(列索引)     | **行索引** (Row Index)      |
| **主要用途** | 制作透视表、报表展示  | 数据清洗、为绘图/ML做准备 | 多级索引处理、矩阵压缩 | `groupby`结果展开，矩阵展开 |
| **索引变化** | 指定列变为行/列索引   | 不涉及复杂索引变化        | **列索引变行索引**     | **行索引变列索引**          |
| **逆运算**   | 对应 `melt`           | 对应 `pivot`              | 对应 `unstack`         | 对应 `stack`                |



## 第五章 聚合与分组运算

**数据聚合与分组运算**是数据分析中极其重要的一环，Pandas 提供了比 SQL 更强大、更灵活的分组运算能力。

### 5.1 groupby：分组

Hadley Wickham（R 语言大神）提出了 Split-Apply-Combine（拆分-应用-合并），这句话是整个现代数据分析的**“灵魂”**。



#### 5.1.0 groupby函数的用法

##### 一、`GroupBy` **底层逻辑**

- **第一步：拆分 (Split)**  将 Pandas 对象中的数据，根据提供的一个或多个**键 (Key)** 拆分为多组 。
  - 拆分是在特定轴上执行的（例如：行 `axis=0` 或 列 `axis=1`） 。

- **第二步：应用 (Apply)** 将一个函数应用到每一个独立的分组上，从而产生一个新值 

- **第三步：合并 (Combine)** 将所有函数的执行结果合并到最终的结果对象中 。最终返回的形式取决于执行的操作类型 。



##### 二、`groupby`基础语法

```python
df.groupby('按谁分组')['对谁计算'].怎么算()
```

实例：假设有一张学生成绩表，需求是计算每个班级的平均分

```python
# 准备一份简单的测试数据
data = {
    '班级': ['一班', '一班', '二班', '二班', '一班'],
    '性别': ['男', '女', '男', '女', '男'],
    '姓名': ['张三', '李四', '王五', '赵六', '孙七'],
    '分数': [80, 90, 85, 95, 88]
}
df = pd.DataFrame(data)
```

按照**拆分-应用-合并**的来使用`groupby`函数

**第一步：拆分（根据“班级”这一列分组）**

```python
# 动作 1：按“班级”分组
grouped = df.groupby('班级')
```

注意！变量`grouped`是一个`GroupBy`对象。它实际上还没有进行任何计算，只是含有一些有关分组键`班级`的中间数据而已。它不知道你要对这两堆数据做什么，所以它就停在那里，返回一个“对象”给你。

> 如果想根据多个列进行分组，**只需要在 `groupby()` 中传入一个包含列名的列表即可**
>
> 按两个维度分组，它就会生成一个两层的多级索引来展示结果。
>
> 但在绝大多数数据分析和导出的场景下，多级索引处理起来稍微有点麻烦，所以遇到多列分组时，**强烈建议在最后加上 `.reset_index()`**，把它变回清爽的普通二维表格（DataFrame）：

**第二步：应用（设置计算方式或者函数）**

对`groupby`对象使用`mean()`函数，并且只对`分数`这一列使用`mean()`函数

```python
grouped['分数'].mean() 
```

> 在作为分组依据的键中，任何缺失值（NA）都会被自动从最终的分组结果中剔除，不会作为一个单独的 "NaN" 组出现

得到了一个`DataFrame`变量

**第三步：合并（这一步`pandas`会自动进行）**



#### 5.1.1 ⭐如何分组

##### 一、按列名分组

最常用的方式。直接传入列名，如 `df.groupby('key1')`，数据就会按照 `key1` 这一列里面不同的值被切分开 

##### 二、按多个列名分组

- 传入一个列表，如 `df.groupby(['key1', 'key2'])` 
- **结果**：产生的结果会带有一个**多级索引**，即用两个键的组合来唯一标识一行 

##### 三、用外部数组进行分组

分组的依据甚至不需要在你的表格里。只要你手头有一个和表格行数一样长的数组（比如 NumPy 数组），你就可以直接把它当成分组依据传进去 。

> 只要数量对得上，Pandas 就会按顺序把外部数组里的标签，跟表格里的每一行“贴”在一起进行分组。**相当于把数组粘贴上了，构成了临时一列**

##### 四、通过字典进行映射分组

假设你有一张表，以及一个字典，记录了每个员工所属的部门

```python
df = pd.DataFrame(
    {'销量': [100, 200, 150, 300, 250],
     '退货量': [5, 10, 0, 15, 8]},
    index=['张三', '李四', '王五', '赵六', '孙七']
)

mapping_dict = {
    '张三': '一部',
    '李四': '一部',
    '孙七': '一部',
    '王五': '二部',
    '赵六': '二部'
}
```

原始数据：

```
     销量  退货量
张三  100    5
李四  200   10
王五  150    0
赵六  300   15
孙七  250    8
```

需求：根据部门名单，算出每个部门的总销量

```python
grouped3 = df.groupby(mapping_dict)['销量'].sum()
```

结果：

```
一部    550
二部    450
Name: 销量, dtype: int64
```

**前提条件**：字典的“键”必须与表格的 **行索引 (Index)** 对应。如果名字在普通的列里（比如叫 `姓名` 列），这种写法会找不到人。

**字典没写全怎么办？**：如果在字典里找不到某个行索引（比如突然来了个“周八”），Pandas 会在分组时**自动忽略**这一行，不会报错。

**为什么不加进原表？**：这种写法不需要向原表插入一个真实的“部门”列，非常适合做**临时性**的快速统计，能保持原表数据的干净。



##### 五、通过函数进行分组

**Pandas 会把你表格的**行索引（Index）一个一个地喂给这个函数。函数吐出什么结果，这行数据就被分到哪个组。

现有一张员工业绩表

````python
df = pd.DataFrame(
    {'业绩': [10, 20, 30, 40, 50]},
    index=['Joe', 'Steve', 'Wes', 'Jim', 'Travis']
)
````

```
        业绩
Joe     10
Steve   20
Wes     30
Jim     40
Travis  50
```

需求：把名字长度一样的人，分到同一个组，然后计算每组的总业绩。

```python
result_len = df.groupby(len).sum()
```

```
   业绩
3  80
5  20
6  50
```



##### 六、混合分组

`groupby`可以把函数、字典、列名列表这些分组方式混合起来使用，Pandas 会把它们分别计算出一套分组标签，然后把这些标签拼成一个**多级索引 (MultiIndex)**。

##### 七、根据多级索引分组

如果数据自带多级索引，可以直接使用 `level` 关键字（如 `level='cty'`）告诉 Pandas 你想按哪一级的索引进行分组



---



#### 5.1.2 GroupBy对象相关操作

##### 一、查看每组大小 

- 不管你按什么分组，加上 `.size()` 方法就能返回一个 Series，清晰地告诉你每个分组里包含多少行数据 。



##### 二、遍历迭代分组 (for 循环)

- `GroupBy` 对象是可以通过 for 循环遍历的 。
- 每次循环，它会吐出一个**二元元组**：`(分组的名字, 这个分组对应的数据块)` 。

语法：

```python
for name, group in groupby对象名
# 注意，for后面跟的必须是 name, group,不能是其他名字
```

每一次循环，它都会“吐”出一个包含两个元素的**元组 (Tuple)**：

- **`name` (组名)**：也就是你用来分组的那个标签（比如“一班”）。
- **`group` (数据块)**：这是一个**真实的 DataFrame**，里面只包含属于当前这个组的所有原始数据。

例子

```python
grouped2 = df.groupby('班级')
for name,group in grouped2:
    print(f"当前组名：{name}")
    print(group)
    print("="*20)
```

结果：

```
当前组名：一班
   班级 性别  姓名  分数
0  一班  男  张三  80
1  一班  女  李四  90
4  一班  男  孙七  88
====================
当前组名：二班
   班级 性别  姓名  分数
2  二班  男  王五  85
3  二班  女  赵六  95
====================
```

如果我们是按两个列（比如按“班级”和“性别”）分组， 逻辑完全一样。唯一的区别是：**此时的 `name`（组名）会变成一个元组**，比如 `('一班', '男')`。



##### 三、把分组变成字典

对于初学者来说，最直观的提取特定组数据的方法是：把迭代出来的片段做成一个字典 `dict(list(df.groupby('key1')))` 。这样你就可以通过字典的键，直接提取那一组的数据表。



### 5.2 agg：数据聚合

所谓数据聚合，指的就是把一个数组（多行数据），经过某种数学运算后，**浓缩/合并成一个单一的值**的过程（即**使用聚合函数的过程**）。之前的 `.mean()`、`.sum()`、`.max()`、`.count()`，它们全都是标准的“聚合函数”。

**不是所有的计算都叫聚合**

为了让你更清晰地理解，我们对比一下**“非聚合”**的计算：

- **聚合计算（多变一）**： 一班有 50 个人，计算这 50 个人的**平均分**。结果只有 1 个数字。
- **非聚合计算（一对一，比如映射或转换）**： 一班有 50 个人，给这 50 个人的分数**每人加 10 分**。算完之后，结果还是 50 个数字。这就不能叫聚合。



#### 5.2.1 如何进行数据聚合

##### 一、官方聚合函数

Pandas 底层对一些高频函数进行了速度优化，主要包括：

- `count`（非NA值数量）
- `sum`
- `mean`（平均值）
- `median`（中位数）
- `std`/`var`（标准差/方差）
- `min`/`max`、`prod`（乘积）
- `first`/`last`（首尾非NA值） 



##### 二、自定义聚合函数 

如果你想用自己发明的算法，只需将自定义的函数传入 `agg` 方法即可：

```python
# 自定义函数：计算最大值与最小值的差
def get_diff(x):
    return x.max() - x.min()
grouped = df.groupby('班级')

result = grouped.agg(get_diff)

```



#### 5.2.2 同时应用多种函数

在实际分析中，我们往往希望**对同一列应用多个函数**，或者**对不同的列使用不同的函数** 。 `agg` 方法对此提供了极其强大的支持。

以一张店铺销售记录表为例

```python
df = pd.DataFrame({
    '店铺': ['A店', 'A店', 'A店', 'B店', 'B店'],
    '商品': ['苹果', '香蕉', '苹果', '香蕉', '西瓜'],
    '销量': [100, 200, 150, 300, 50],
    '利润': [20, 10, 30, 15, 40]
})
# 自定义函数
def get_diff(x):
    return x.max() - x.min()
```

```
店铺  商品   销量  利润
0  A店  苹果  100  20
1  A店  香蕉  200  10
2  A店  苹果  150  30
3  B店  香蕉  300  15
4  B店  西瓜   50  40
```



##### 一、对同一列应用多个函数

**操作**：传入一个包含多个函数名（字符串）或实际函数的列表。例如：

**需求：**查看每个店铺的**总销量、平均销量，还有最高销量与最低销量之差**

```python
grouped3 = df.groupby('店铺')
result = grouped3['销量'].agg(['sum','mean',get_diff])
print(result)
```

其中`'mean', 'sum'`是官方聚合函数，需要用引号`''`，而`get_diff`是用户自定义函数，不需要引号

**结果**：得到的 DataFrame 的列会以相应的函数名来自动命名 

```python
    sum   mean  get_diff
店铺                      
A店  450  150.0       100
B店  350  175.0       250
```



##### 二、不同的列应用不同的函数

**操作**：向 `agg` 传入一个“从列名映射到函数”的**字典**(键是列名，值是计算方法) ，指定哪一列用哪个函数

**需求：**查看每个店铺的**总销量**、**平均利润**，还要知道他们卖了**几种不同的商品**

```python
result2 = grouped3.agg(
    {
        '销量':'sum',
        '利润':'mean',
        '商品':'count'
    })
```

输出结果：

```
    销量  利润  商品
店铺               
A店  450  20.0   3
B店  350  27.5   2
```



##### 三、自定义结果的列名

前两节的输出结果中，聚合列名要么是函数名，要么是原来的列名，可读性的很差。看表时可能会产生疑问：“这个 450 是总销量还是平均销量？”

**解决方法：**向`agg`函数中传入一个由 `(自定义列名, 函数)` 元组组成的列表 。各元组的第一个元素就会被用作最终 DataFrame 的列名 。

**语法：**`新列名 = ('旧列名', '计算方法')`

需求：查看每个店铺的**总销量**、**平均利润**，还要知道他们卖了**几种不同的商品**

```python
opt_result = grouped3.agg(
    总销量=('销量','sum'),
    平均利润 = ('利润','mean'),
    商品种类数 = ('商品','count')
)
```

```
    总销量  平均利润  商品种类数
店铺                  
A店  450  	20.0      3
B店  350  	27.5      2
```



#### 5.2.3 消除多级索引

在前面的学习中提到过一个痛点：`groupby` 算完之后，**用来分组的列（比如“班级”或“店铺”）默认会变成结果的“行索引 (Index)”**。

为了把它变回普通的列，之前教的方法是：在最后面加上一根小尾巴 `.reset_index()`。

而 **`as_index=False`** 的意思就是：**“从一开始就别把分组列当成索引，直接给我一个清清爽爽的普通表格”**



### 5.3 apply：拆分-应用-合并

此处的`apply`函数就是第一章中介绍的那个函数，可以对`Series`和`DataFrame`使用自定义函数。



`apply` 完美执行了前面讲过的底层逻辑：它会将待处理的数据**拆分**成多个片段，然后对每个片段**调用**你传入的函数，最后再将各个片段的结果**组合**到一起 。

`apply()` 不需要绑定 `groupby` 一起使用（正如第一章演示的那样），`apply` 本质上就是一个披着 Pandas 外衣的 **for 循环**。只要你想对数据做某种定制化的“遍历加工”，不管分没分组，都可以使用这个函数



#### 5.3.1 apply和agg的区别

`apply`和`agg`都可以使用自定义的函数，这两个方法有什么区别？

**`agg()` 是“收敛的计算器”（必须多变一），而 `apply()` 是“无所不能的加工厂”（随便你怎么变）。**



##### 一、 函数“看到”的输入不同（核心区别）

这是它俩在底层机制上最大的不同。

- **`agg()`：按“列”切片送进去**

  当你在 `groupby` 后使用 `agg(自定义函数)` 时，Pandas 会把当前分组的**每一列**（作为一个一维的 Series）分别、单独地送进你的函数里。你的函数在同一时间，只能看到“某一列”的数据，看不到其他列。

- **`apply()`：把“整个组”打包送进去**

  当使用 `apply(自定义函数)` 时，Pandas 会把当前分组的**整个子表格**（作为一个二维的 DataFrame）一次性端给你的函数。你的函数可以同时调用这个组里的好几列数据来做复杂运算。

**场景对比：**

假设我们要计算“利润率（利润/销售额）”。

- `agg()` **做不到**。因为它每次只能拿到“利润”这一列或者“销售额”这一列，它没法同时拿着两列相除。
- `apply()` **可以做到**。因为它拿到的是包含了这两列的整个表格。



##### 二、 函数的输出不同

- **`agg()` 的死规矩：必须是“单值 (Scalar)”**

  `agg` 的全称是聚合，它带着“把多行浓缩成一行”的死命令。因此，你的自定义函数最后 `return` 的结果，**必须是一个单一的数字或字符串**。如果你用 `agg` 吐出了一个列表或一列数据，通常会报错。

- **`apply()` 的极度自由：啥都行**

  `apply` 完全不限制你。你的函数可以返回一个单一的值（这时候它表现得就像 `agg`），也可以返回一列数据（Series），甚至可以返回一个全新表格（DataFrame）。

##### 三、多任务并行能力

- **`agg()` 是“多面手”**

  正如之前的字典法 `agg({'销量': 'sum', '利润': 'mean'})`，`agg` 最强大的地方在于它可以**一次性对不同的列下达不同的指令**，甚至对同一列下达多个指令（比如同时求和与求最大值）。

- **`apply()` 是“专一的流水线”**

  `apply` 通常只能接受**一个**函数，把这套逻辑死死地套在整个分组上。你很难像 `agg` 那样用优雅的字典去告诉它“A列这么算，B列那么算”。

| **对比维度**       | **agg() (聚合)**                       | **apply() (应用)**                     |
| ------------------ | -------------------------------------- | -------------------------------------- |
| **底层逻辑**       | **收敛汇总**（多变一）                 | **灵活加工**（极其自由）               |
| **函数接收的输入** | 当前分组的**每一列** (Series)          | 当前分组的**整个子表** (DataFrame)     |
| **函数必须的输出** | **单一值** (Scalar，如总和、均值)      | 单一值、Series 甚至是 DataFrame 都行   |
| **跨列计算能力**   | ❌ 极弱（只能看一列算一列）             | ✅ 极强（可以随意调用多列互相加减乘除） |
| **多任务能力**     | ✅ 极强（支持字典法、列表法同时算多个） | ❌ 弱（通常一次只执行一个自定义逻辑）   |
| **运行速度**       | 较快（有底层优化）                     | 极慢（本质是 Python for 循环）         |

**总结**

如果想**“算总账”**（求和、极差、平均数），并且是各列算各列的，用 `agg()`。 只要计算**必须跨列**（比如 A列除以B列），或者你想返回的是**一个表而不是一个数**（比如筛选前两名），用 `apply()` 。



#### 5.3.2 apply的使用方法

##### 一、对单列(Series)使用

这是最简单、也最常用的场景。它就像一条流水线，把这一列的数据**挨个**送进你写好的函数里加工，然后吐出新数据。

**场景**：有一列“成绩”，你想把大于等于 60 分的变成“及格”，小于 60 分的变成“不及格”。

```python
df = pd.DataFrame({'姓名': ['张三', '李四', '王五'], '分数': [85, 50, 90]})

# 1. 自己写个加工函数
def check_score(score):
    if score >= 60:
        return '及格'
    else:
        return '不及格'

# 2. 用 apply 把函数“应用”到这列的每一个数据上
df['状态'] = df['分数'].apply(check_score)

print(df)
```

```
姓名  分数   状态
0  张三  85   及格
1  李四  50  不及格
2  王五  90   及格
```

##### 二、对整表 (DataFrame) 使用

当你需要同时用到**两列或多列**的数据来算出一个新结果时，就需要对整个表使用 `apply`，并加上参数 **`axis=1`**。

> 为什么加**`axis=1`**？
>
> 因为跨列运算是横着算（从上往下遍历），所以要用**`axis=1`**
>
> 如果默认**`axis=0`**,就会变成竖着算（从左到右遍历）

**场景**：表里有“单价”和“数量”，新增一列“总价”（单价 $\times$ 数量）。

```python
df2 = pd.DataFrame({
    '商品': ['苹果', '香蕉'],
    '单价': [5, 3],
    '数量': [10, 20]
})

def sum_money(x):
    return x['单价']*x['数量']
df2['总价'] = df2.apply(sum_money, axis=1)

print(df2)
```

```
商品  单价  数量  总价
0  苹果   5  10  50
1  香蕉   3  20  60
```



##### 三、在 GroupBy 之后使用

`agg()` 只能返回一个浓缩的值（比如平均值），但 `apply()` 可以返回一个值、一个列表、甚至一个表格

**场景（经典面试题）**：找出每个部门业绩排名前 2 的员工。

**痛点**：`agg()` 算不出排名（因为他是聚合函数）。必须用 `apply()`

```python
df3 = pd.DataFrame({
    '部门': ['销售', '销售', '销售', '技术', '技术', '技术'],
    '姓名': ['A', 'B', 'C', 'D', 'E', 'F'],
    '业绩': [100, 300, 200, 500, 400, 600]
})
```

```python
# 1. 写一个处理“小表格”的函数
def top_2(group_df):
    # 对进来的小表格按业绩降序排，然后取前两行
    return group_df.sort_values(by='业绩', ascending=False).head(2)

# 2. 分组并 apply
# 这里的 apply 会把“销售部的小表”和“技术部的小表”分别送进 top_2 函数里
result = df3.groupby('部门').apply(top_2)

print(result)
```

##### 四、消除多级索引

默认情况下，使用 `apply` 后，Pandas 会和`agg`把“分组键”变成索引，和原始数据的索引拼在一起，形成多级索引。这有时候会让表格看起来很复杂

**解决办法**：如果你不需要这个外层的分组键索引，可以在调用 `groupby` 时传入参数 `group_keys=False` 



### 5.4 transform：窗口函数

Pandas 的 `groupby(...).transform(...)`，就是 SQL 里的 `OVER (PARTITION BY ...)`

**场景：**假设有一张 1000 行的员工表，我想算出每个部门的平均工资，并且**不把表格浓缩成几行，而是直接把这个平均工资作为新的一列，原封不动地贴回这 1000 行的明细表里**，用来计算每个人的工资差额。

也就是说，`transform`等价于：使用apply应用函数后，再手动把结果列逐个加回原表

例：

```python
df = pd.DataFrame({
    '姓名': ['张三', '李四', '王五', '赵六'],
    '部门': ['销售', '销售', '技术', '技术'],
    '工资': [5000, 7000,  8000, 12000]
})
```

使用`transform`:

```python
df['部门平均工资'] = df.groupby('部门')['工资'].transform('mean')

# 算一下每个人比部门平均工资高多少
df['工资差额'] = df['工资'] - df['部门平均工资']

print(df)
```



### 5.5 数据透视表

相关函数：`pivot_table`

#### 5.5.1 pivot_table 基础用法

功能和Excel中的数据透视表一样。在用 Excel 做透视表时，我们会把字段拖拽到四个框框里：行、列、值、计算方式。Pandas 的 `pivot_table` 完全对应这四个动作：

```python
pd.pivot_table(df,
               index='行',
               columns='列',
               values='值',
               aggfunc='怎么算')
```

| **参数**         | **Excel对应操作** | **含义与作用**                                               |
| ---------------- | ----------------- | ------------------------------------------------------------ |
| **`index`**      | 拖到“行”区域      | 报表左侧的行表头（分组依据 1）。                             |
| **`columns`**    | 拖到“列”区域      | 报表上方的列表头（分组依据 2）。                             |
| **`values`**     | 拖到“值”区域      | 你要用来计算的数字列（比如销量、利润）。                     |
| **`aggfunc`**    | 值的汇总方式      | 怎么算？默认是 `'mean'`（求平均）。可以改成 `'sum'`, `'count'`, 甚至传一个字典。 |
| **`margins`**    | 勾选“显示总计”    | 设为 `True` 时，会在表格的最右边和最下边自动加上“总计(All)”行/列。 |
| **`fill_value`** | 空白单元格显示    | 如果某个交叉点没有数据（比如北方没卖过香蕉），用什么数字填补 NaN（通常填 `0`）。 |

案例：假设有一份全国连锁店的销售明细表

```python
df = pd.DataFrame({
    '区域': ['华北', '华北', '华东', '华东', '华北', '华东'],
    '城市': ['北京', '天津', '上海', '杭州', '北京', '上海'],
    '商品': ['手机', '手机', '电脑', '手机', '电脑', '电脑'],
    '销售额': [1000, 800, 2000, 1500, 1200, 2500],
    '利润': [100, 80, 300, 200, 150, 400]
})
```

需求：查看每个城市的每个商品的销售总额

```python
df.pivot_table(index='城市',
               columns='商品',
               values='销售额',
               aggfunc='sum',
               fill_value=0,
               margins=True
               )
```

结果：

```
商品     手机    电脑   All
城市                   
上海      0  4500  4500
北京   1000  1200  2200
天津    800     0   800
杭州   1500     0  1500
All   3300  5700  9000
```



#### 5.5.2 `pivot_table`和`groupby`

##### 一、两个函数本质相同

`pivot_table` 是专门为了“极简地创建二维数据表”而发明的快捷工具，但 `groupby` 配合上 `unstack()`，也完全可以做出和 `pivot_table` 一模一样的二维表。

在 Pandas 的底层源码里，`pivot_table` 其实就是调用了 `groupby`，然后帮你做了一些额外的“排版”工作。

例如前面的场景也可以用`groupby`来实现

```python
result2 = df.groupby(['城市','商品'])['销售额'].sum().unstack()
```

因为`groupby`可以创建多层行索引，再通过`unstack`把内层行索引变成列，从而变成了二维表



##### 二、优势对比

`pivot_table`的优势

- **`margins=True`（一键生成总计行/列）**： 在报表边缘自动算出一行“All”和一列“All”。如果你用 `groupby` 去做，你需要单独算一个总和，再用 `concat` 拼接到原表上。
- **`fill_value=0`（一键填补空缺值）**： 如果上海没卖过苹果，交叉出来的格子里会是 `NaN`。`pivot_table` 可以直接填 0，而 `groupby` 算完之后还得再追加一步 `.fillna(0)`。



`groupby`的优势

- **`pivot_table` 只能做“聚合 (`agg`)”**：它的全称叫数据透视表，它的使命就是把数据汇总、变少。
- **`groupby` 是“全能神”**：`groupby` 不仅能做 `agg`（聚合），还能做 `transform`（贴回原表，行数不变），还能做 `apply`（各种花式自定义操作），甚至能做 `filter`（按组过滤数据）。在这些场景下，`pivot_table` 毫无用武之地。

| **特性**       | **pivot_table**                        | **groupby**                                  |
| -------------- | -------------------------------------- | -------------------------------------------- |
| **设计初衷**   | 模仿 Excel，制作给人类看的二维报表     | 模仿 SQL，执行底层的“拆分-应用-合并”         |
| **二维表创建** | 天生支持（通过 `columns` 参数）        | 需要配合 `.unstack()`                        |
| **总计行/列**  | 极简（`margins=True`）                 | 极其麻烦（需要手动拼接）                     |
| **功能范围**   | **仅限聚合** (Aggregation)             | 聚合、转换(`transform`)、应用(`apply`)、过滤 |
| **工作角色**   | 处于数据分析流程的**最末端**（出报表） | 处于数据清洗/分析的**全过程**                |



#### 5.5.3 数据交叉表

相关函数：`pd.crosstab`，是数据分析中专门用来**“数人头”（统计频次）**的函数。

##### 一、基础用法

**基础语法：**

```python
result = pd.crosstab(index=df['列名1'], columns=df['列名2'])
```

- 基础交叉表：只传两个参数，谁在行(index)，谁在列(columns)

- 传的是 df['列名']，而不是 '列名'

  > `crosstab` 是 Pandas 的顶层函数，需要直接把具体的列（Series）对象传给它，而不是传列名。



**场景：** 统计不同部门里，男生和女生分别有多少人。

```python
df = pd.DataFrame({
    '部门': ['销售', '技术', '销售', '技术', '技术', '销售', '技术'],
    '性别': ['男',   '男',   '女',   '女',   '男',   '女',   '男']
})

result = pd.crosstab(index=df['部门'], columns=df['性别'])
```

结果

```
性别  女  男
部门      
技术  1  3
销售  2  1
```



##### 二、进阶用法

如果仅仅是数人头，`pivot_table(aggfunc='count')` 也能做到。但 `crosstab` 之所以不可替代，是因为它有参数：**`normalize`（标准化/归一化）**

参数`normalize`的取值：

- `normalize='all'`：看全局每个格子占总人数的百分比。
- **`normalize='index'`**：看**行**的百分比（最常用！比如看“各个部门内”的男女比例）。
- `normalize='columns'`：看**列**的百分比（比如看“所有男生中”，销售和技术各占多少）。
