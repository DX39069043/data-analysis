# 第一章 数据结构与基础操作 

## Series

Series 是一种类似于一维数组的对象，可以看作是**定长的有序字典**，或**带标签的一维数组**。

**组成**：一组数据 + 一组数据标签（索引）。

> **本质**：Numpy 数组穿上了一层“标签”外衣，完美继承了 Numpy 的各种功能（例如布尔索引）。



### 创建方式

1.**由列表创建**：

**语法：**`s = pd.Series([data])`

- 默认情况下，索引是从 0 到 N-1 的整数

- 添加参数：`index=['a', 'b'...]`，指定索引为`['a', 'b'...]`，索引列表的长度须与`Series`长度保持一致）。



2.**由字典创建**：`pd.Series(dict)`（键自动成为有序索引）



### **核心属性与操作**

- `obj.values` (返回 Numpy 数组)；`obj.index` (返回索引对象)。

- **获取与修改索引**：可通过 `obj.index = ['Bob', ...]` 就地修改。

- **Name 属性**：

  - `Series.name`：对象本身的名字（变 DataFrame 一列时，这就是列名）。

  - `Series.index.name`：索引的名字（增加可读性，如“商品ID”）。

    ```python
    s = pd.Series(
        [10, 20, 30],
        index = ['001','002','003'], 
        name='价格')        # 设置Series名称为'价格'
    s.index.name = '商品ID' # 设置索引名称为 '商品ID'
    ```

    ```
    商品ID
    001    10
    002    20
    003    30
    Name: 价格, dtype: int64
    
    # 该表的含义为：ID编号为001的商品价格是10
    ```

- **操作特性**：Series 支持 Numpy 数学运算（索引链接保留）；支持类似字典的 `in` 成员检测。

- **自动数据对齐 (重点)**：算术运算时根据**标签**自动对齐（类似 SQL 里的 `JOIN` 操作），不重叠部分产生 `NaN`。

- **缺失数据处理**：`NaN` 表示缺失。用 `pd.isnull(obj)` 或 `pd.notnull(obj)` 检测，返回布尔值,其中`obj`是Series名

  > 输出结果是一个和obj形状一样的表格，来表示每个位置是否是NaN



## DataFrame

由一组有序的列（Series）组成的数据结构。数据以二维块存放。

**创建方式**：

- **基础字典**：传入等长列表/Numpy数组的字典。键变列名，自动加索引。

  ```python
  data = {
      'state': ['Ohio', 'Ohio', 'Ohio', 'Nevada', 'Nevada', 'Nevada'],
      'year': [2000, 2001, 2002, 2001, 2002, 2003],
      'pop': [1.5, 1.7, 3.6, 2.4, 2.9, 3.2]}
  frame = pd.DataFrame(data)
  ```

- **嵌套字典**：外层键变列（Columns），内层键变行索引（Index），内层键会自动合并排序。支持 `frame.T` 转置。

- **Series 字典**：每个 Series 成为一列，索引自动合并。



## Index

管理轴标签和元数据的强大、**不可变**的数据结构（有序集合）。

- **不可变性**：一旦创建不能修改内部元素（如 `index[1] = 'd'` 报错），保证多结构间安全共享。

  > **Index 对象本身**是**不可变**的：你不能修改它里面的某个特定元素。
  >
  > **Series/DataFrame 的 `.index` 属性**是**可被赋值**的：你可以把旧的 Index 扔掉，换一个新的上去，这个新的index依然无法被修改，只能被替换

- **集合特性**：类似固定集合，支持 `in`。但与 Python 原生 Set 不同，Pandas Index **可以包含重复标签**（选重复标签会展示所有结果）。

- **对比：`.index` 和 `.columns`**本质都是 Index 对象

## 处理文本格式数据

### 2.1 常用读取函数全景图

Pandas 提供了一组函数用于将表格型数据读取为 DataFrame。其中最常用的是 `read_csv`。

| **函数**         | **说明**                                            |
| ---------------- | --------------------------------------------------- |
| **`read_csv`**   | 加载带分隔符的数据。默认分隔符为**逗号**。          |
| **`read_table`** | 加载带分隔符的数据。默认分隔符为**制表符 (`\t`)**。 |
| `read_fwf`       | 读取定宽列格式数据（没有分隔符）。                  |
| `read_clipboard` | 读取**剪贴板**数据（网页转表格神器）。              |
| `read_excel`     | 读取 Excel (XLS/XLSX) 文件。                        |
| `read_html`      | 读取 HTML 文档中的所有表格。                        |
| `read_json`      | 读取 JSON 字符串中的数据。                          |
| `read_pickle`    | 读取 Python pickle 格式的对象。                     |
| `read_sql`       | 读取 SQL 查询结果。                                 |



### 2.2 `read_csv` 核心参数解析

 `read_csv` 的参数分为四组：

#### 第一组：基础寻址与解码

- **`path` (文件路径)**：告诉 Pandas 读取哪个文件

  > `path`可以省略不写： df = pd.read_csv("数据.csv")
  >
  > 当文件不在工作目录中：使用`../`来表示父目录

- **`sep` (分隔符)**：默认值是逗号 `,`。有些数据是用“制表符（Tab）”或者“分号”隔开的，这时候就要改这个参数。

- **`encoding` (编码方式)**：英文系统通常用 `utf-8`，中文 Windows 里的 Excel 经常搞出 `gbk` 编码。如果你打开文件发现全是**“烫烫烫”**或者乱码，就把这个改成 `'gbk'`。



#### 第二组：表头与索引

- **`header` (表头/列名行)**：“第几行是表头/列名行”。默认值 `0`（第一行是表头）。如果文件里全是数据，没有表头，设为 `None`。
- **`names` (自定义列名)**：“放弃 csv 原本自带的的表头，指定新的列表作为表头”。用法：传一个列表 `['名字', '年龄', '班级']`。

  > 在实际工作中，可能会遇到**”读取文件的某几列，并把这几列的列名改掉“**
  >
  > 但是pandas 的处理顺序是：
  >
  > 1. 先用 `names` 替换表头 → 列名变成 `['ID', '岗位名称', '薪资']`
  > 2. 再用 `usecols` 按原列名筛选 → 找不到 `positionId` 等列了
  >
  > 所以报错：`columns expected but not found: ['positionName', 'positionId', 'salary']`
  >
  > 解决方法：
  >
  > - 使用`usecols=[0,1,17]`，用位置标号代替名字，避免冲突
  > - 先`usecols=["positionId","positionName","salary"]`,之后再用`rename`函数改名
- **`index_col` (索引列)**：放弃默认生成的 0,1,2… 行索引，指定某一列来作为行索引。比如让第一列“学号”变成唯一标识，设 `index_col=0`

  > `index_col`作为一个参数，是在读取数据时指定索引列；如果在后续处理数据时想要再修改索引列，需要使用`set_index()`函数



#### 第三组：初步清洗

- **`nrows`(只读前n行)**
- **`skiprows` (跳过行)**：忽略前 n 行，从 n+1 行开始读取。

  > `skiprows`后面除了跟一个数字，还可以跟整数列表、布尔值列表、函数
  >
  > | 传法           | 说明                 | 示例                         |
  > | -------------- | -------------------- | ---------------------------- |
  > | **整数列表**   | 指定要跳过的行索引   | `[0, 2, 4]` 跳过第1,3,5行    |
  > | **布尔值列表** | `True` 表示跳过该行  | `[False, True, False, True]` |
  > | **函数**       | 返回 `True` 表示跳过 | `lambda x: x % 2 == 1`       |
- **`usecols`(读取指定列):**  "A:C"`、`[0, 2, 3]`、`["姓名", "年龄"]

  > 下标是从0开始的
- **`na_values` (替换空值)**：什么字算是`空`？。正常的空值是空的，但有时会写`未知`或`?`。Pandas 不认识这些，你要告诉它：只要看到`?`，就把它当成`NaN`处理。用法:`na_values=['NULL','未知','?']`。
- **`keep_default_na=False`(禁止自动转化NA)：**读取 CSV 文件时，不要把空值自动转换成 `NaN`，而是保留原始的空字符串或特定标记。

  > 如果不使用 `keep_default_na=False`，pandas 默认会将以下值自动识别为 `NaN`：
  >
  > | 值           | 说明             |
  > | ------------ | ---------------- |
  > | `''`         | 空字符串         |
  > | `'#N/A'`     | Excel 错误值     |
  > | `'#N/A N/A'` | Excel 错误值变体 |
  > | `'#NA'`      | Excel 错误值     |
  > | `'-1.#IND'`  | 浮点错误值       |
  > | `'-1.#QNAN'` | 浮点错误值       |
  > | `'-NaN'`     | 负 NaN           |
  > | `'-nan'`     | 负 nan           |
  > | `'1.#IND'`   | 浮点错误值       |
  > | `'1.#QNAN'`  | 浮点错误值       |
  > | `'N/A'`      | 常见缺失标记     |
  > | `'NA'`       | 常见缺失标记     |
  > | `'NULL'`     | 数据库空值       |
  > | `'NaN'`      | Not a Number     |
  > | `'n/a'`      | 小写变体         |
  > | `'nan'`      | 小写变体         |
  > | `'null'`     | 小写变体         |
  >
  > **什么时候会用到这个参数？**
  >
  > | 原始数据         | 自动转 NaN 后 | 问题                           |
  > | ---------------- | ------------- | ------------------------------ |
  > | `""`（空字符串） | `NaN`         | 分不清是"没填"还是"真为空"     |
  > | `"NULL"`         | `NaN`         | 丢失了原始标记                 |
  > | `"NA"`           | `NaN`         | 可能"NA"是正常值（如国家代码） |
  > | `-999`           | `NaN`         | 可能`-999`是业务上的特殊标记   |
- **`na_filter = false`(关闭缺失值检测机制)**

  > `keep_fault_na`和`na_filter`有什么区别？
  >
  > | 参数              | 作用                       | 效果                                          |
  > | ----------------- | -------------------------- | --------------------------------------------- |
  > | `keep_default_na` | 控制**默认缺失值列表**     | `False` 时不用默认列表，但 `na_values` 仍有效 |
  > | `na_filter`       | 控制**整个缺失值检测机制** | `False` 时完全关闭检测，包括 `na_values`      |
- **`parse_dates` (解析日期)**：“把字符串变成真正的日期”。开启此参数，Pandas 会把 `2023-01-01` 变成“时间对象”，方便做“下个月”等时间计算。
- **`dtype`(强制数据类型)：**设置 `dtype={'股票代码': str}` 保留文本格式。



#### 第四组：大文件处理

- **`nrows` (读取行数)**：只读取一部分数据。场景：文件有 1 亿行，只想读前 5 行看看长什么样，防止内存爆掉。
- **`chunksize` (块大小)**：分批次处理。设置后 Pandas 会把数据切块读取（如一次 1000 行），适合用于循环处理。
  例如：`data = pd.read_csv("某招聘网站数据.csv", chunksize= 1000)`



### 2.3 将数据保存为`csv`文件

处理完数据后，我们通常需要保存结果。使用 `to_csv` 函数。

相关参数：

- `path_or_buf`：保存文件的路径或文件对象。如果不指定，则返回 CSV 格式的字符串。
- `sep`：字段分隔符，默认为逗号。
- `na_rep`：缺失值的表示，默认为空字符串。
- `columns`：要写入文件的列，默认为所有列。
- `header` / `index`：是否将列名/索引写入文件，默认为 True。
- `mode`：打开文件的模式，如 `'w'`（覆盖写入）。



##  处理Excel文件数据

> 处理Excel文件需要用到`read_excel`函数，下面介绍他的参数含义

### 3.1 读取指定数据范围

- **`sheet_name` (指定工作表)**：必须告诉 Pandas 读哪个 Sheet。
  - 默认 `0`（第一个 Sheet）。
  - `sheet_name='Sheet1'`：按名字读（推荐）。
  - `sheet_name=1`：按顺序读（第2个 Sheet）。
  - `sheet_name=None`：**一次性读取所有 Sheet**（返回字典）。
- **`usecols` (筛选列)**：“我只要这几列，其他的杂乱数据别读进来”。应对右侧的备注和草稿，防止读出一堆 `Unnamed` 列。支持字母写法如 `"A:C"`, `"A,C,E"` 或数字 `[0, 2]`。
- **`skiprows` (跳过行)** / **`header` (表头行)**：跳过上面的“年度报表”等合并单元格标题。如果设了 `header=1`，Pandas 自动忽略第 0 行的数据。

### 3.2 数据类型控制

- **`dtype` (强制类型)**：强制指定某一列的数据类型
- **痛点**：Excel 会自动把股票代码 `00700` 变成 `700`。
- **解法**：设置 `dtype={'股票代码': str}` 保留文本格式。



保存结果使用 `to_excel` 函数。

## 处理读取MySQL数据

### 4.1 建立数据库连接

连接数据库需要两个工具：数据库驱动 `pymysql` 和连接引擎 `SQLAlchemy`。核心是使用 `create_engine` 函数生成连接对象。

建立连接的代码模板如下：

```python
from sqlalchemy import create_engine
import pandas as pd

# 格式：数据库类型+驱动://用户名:密码@主机IP:端口/数据库名
db_url = "mysql+pymysql://root:123456@localhost:3306/test_db"
engine = create_engine(db_url)
df = pd.read_sql("SELECT * FROM users", engine)
```

**⚠️ 避坑指南：密码包含特殊字符**

如果密码里有 `@`、`:` 等特殊字符，会破坏 URL 结构。必须先用 `urllib.parse.quote_plus` 对密码进行编码：

```python
import urllib.parse  # 引入 URL 处理库

# 假设你的密码很复杂
password = "My@Password:123"

# 先编码
encoded_password = urllib.parse.quote_plus(password)

# 然后再拼接到 URL 里
db_url = f"mysql+pymysql://root:{encoded_password}@localhost:3306/test_db"

engine = create_engine(db_url)
```

### 4.2 `read_sql`函数详解

| **参数**          | **含义与实战解析**                                           |
| ----------------- | ------------------------------------------------------------ |
| **`sql`**         | **SQL语句 或 表名**。最核心参数。可写查询语句 `"SELECT * FROM users WHERE age > 18"`，也可直接写表名 `"users"`。 |
| **`con`**         | **连接对象**。通常是 `engine` 对象，告诉 Pandas 连哪个库。   |
| **`index_col`**   | **索引列**。指定某一列（如 `id` 或 `date`）直接变成 DataFrame 的索引，非常常用。 |
| **`parse_dates`** | **解析日期**。告诉 Pandas 哪几列是时间，直接转为 `datetime` 类型。 |
| **`params`**      | **查询参数**。用于安全传递参数防 SQL 注入。如 `sql="SELECT * FROM table WHERE id = %s"`, `params=[10]`。 |
| **`chunksize`**   | **分块读取**。表太大数据撑爆内存时，设置 `chunksize=10000` 返回迭代器，分批处理。 |

# 第三章：数据预览和预处理

## 数据预览

### 1. 宏观浏览

**`shape`**：查看dataframe的大小

**用法：**`df.shape`

**返回值：** 返回一个包含两个数字的元组 `(行数, 列数)`

**注意：**`shape` 是 DataFrame 的一个**属性（Attribute）**，而不是动作/方法，所以它**不能加括号 `()`**

> **实战场景：**每次做完 `merge`（表连接）之后，打印一下 `df.shape`，看看行数有没有异常膨胀（用来排查我们之前聊过的隐式 JOIN 导致的数据笛卡尔积膨胀）



**`size`**:查看总数据量（行数*列数）

**用法：**`df.size`



### 2. 局部预览

> 当表太大时（比如一百万行），直接打印 `df` 会把电脑卡死。这三个函数就是帮你“管中窥豹”的。

**`head()`：**查看前几行

- **用法：** `df.head()` 默认看前 5 行；你可以传数字进去，比如 `df.head(10)` 看前 10 行。
- **实战场景：** 刚把 Excel/CSV 文件读进内存时，看一眼**列名**有没有读错、数据有没有乱码。

**`tail()`** ：查看最后几行

- **用法：** `df.tail()` 默认看最后 5 行。
- **实战场景：** 比如数据是按日期排序的流水账，用 `tail()` 可以快速看一眼最新一天的记录长什么样。

**`sample()`** ：随机抽查

- **用法：**
  - `df.sample(5)`：随机抽 5 行。
  - `df.sample(frac=0.1)`：按比例抽，随机抽出 10% 的数据。
- **实战场景：** 强烈推荐用 `sample` 替代 `head` 来做日常的数据长相预览，它能帮你扫除因为数据排序规律导致的“幸存者偏差”。



### 3. 查看整体信息

 **`info()`** ：查看**缺失值**和数据类型的相关情况

- **用法：** `df.info()`
- **输出结果：**
  1. 总行数和总列数。
  2. 每一列的名字。
  3. **Non-Null Count（非空值数量）：**若某一列显示 `80 non-null`，说明这列有 20 个缺失值（NULL）。
  4. **Dtype（数据类型）：** 告诉你这列是整数(int64)、小数(float64) 还是字符串(object)。
  5. 这张表占用了你电脑多少内存。

`describe()` ：能**数字列**进行全面统计

- **用法：** `df.describe()`

- **输出每一列的：**

  - `count`：有几个数字
  - `mean`：平均值
  - `std`：标准差（波动大小）
  - `min` / `max`：最小值、最大值
  - `25%`, `50%`, `75%`：分位数（50% 就是中位数）。

- **重要参数：** 

  - 默认情况下，它只统计数字类型的列

  - `df.describe(include='O')`：统计文本列的相关信息(**`O` 代表的是 `Object`（对象）**)

  - | **指标名**   | **业务含义**                                         |
    | ------------ | ---------------------------------------------------- |
    | **`count`**  | **非空值数量**。这一列一共有多少个没填空的文字记录。 |
    | **`unique`** | **唯一值数量**。这列一共有多少种**不同**的词。       |
    | **`top`**    | **众数**。出现频次**最高**的那个词是什么。           |
    | **`freq`**   | **频次**。出现频次最高的那个词，到底出现了多少次。   |

  - `df.describe(include='all')`统计所有列的相关信息





## 缺失值的处理

> 缺失值可以用`np.nan`或`pd.NA`来表示

### 1. 检测缺失值

- **检测方法**：

  - `isnull()`：返回布尔对象（True 表示缺失）。

  - `notnull()`：isnull 的否定式（True 表示非空）。

    >  输出结果是一个形状一样的表格，来表示每个位置是否是`NaN`
    >
    >  注：`isnull()` 和 `isna()` **没有任何区别，它们是完全等价的**

- **统计特性**：Pandas 的描述性统计方法（如 `sum`, `mean`）默认都会**排除**缺失数据。



**进阶用法：查看某列的缺失值个数**

因为`isnull()`返回的是一个矩阵，所以我们可以对这个矩阵**应用聚合函数**：`isnull().sum()`

**返回值：**Pandas 默认会**沿着垂直方向**把上面的 0 和 1 加起来。二维的矩阵被降维成了一个一维的 Series（序列），它极其直观地告诉你**每一列**有多少个缺失值：

```
姓名    0
年龄    1
薪水    1
```

**进阶用法：查看表中空值的总数**

上一步已经得到了每一列的缺失数量序列 `[0, 1, 1]`，你再对这个序列追加一个 `.sum()`，即`isnull().sum().sum()`

就是简单粗暴地把这些数字再加一遍： `0 + 1 + 1 = 2`

最终输出一个干干净净的数字：**`2`**，这就是这个表中所有空值的个数



### 2. 定位缺失值

找到含有缺失值的行：`df[df.isnull().any(axis=1)]`  或者  `df.loc[df.isnull().any(axis=1)]`

找到含有缺失值的列：`df.loc[:, df.isna().any(axis=0)]`

> 为什么列的时候要用`.loc`？因为`df[...]` 如果接收的是一组 `True/False` 的布尔值，它**默认这是用来过滤“行”的**。它期待这个序列的长度必须和**行数**一模一样。



### 3. 删除缺失值

`dropna`：丢弃含有缺失值的行或列

**参数含义：**

- **无参数**：`data.dropna()`。只要某行包含**任何** NaN，该行就会被丢弃。
- **只丢弃全空行**：`data.dropna(how='all')`。
- **指定生效范围：**`df.dropna(subset=['语言']) `只有当“语言”这一列是空的时候才删，其他列的情况不关心
- **丢弃列**：传入 `axis=1` 
- **阈值控制 (`thresh`)**：`thresh=n` 表示保留至少有 `n` 个非 NA 值的行（即丢弃非 NA 值少于 n 的行）。例如 `df.dropna(thresh=2)`。

### 4. 填充缺失值

**`fillna`：用于将缺失值替换为指定值**

**填充方式：**

- **常数填充**：`df.fillna(0)`（将所有 NaN 替换为 0）。
- **对某一列进行填充：**`df["评分"].fillna("0")`
- **统计值填充**：使用平均值或中位数填充，如 `data.fillna(data.mean())`
- **传入字典**：传入一个字典，为不同的列指定不同的填充值，如 `df.fillna({1: 0.5, 2: 0})`（第 1 列填 0.5，第 2 列填 0）。
- **传入`Series`对象**：`df['A'].fillna(Series_B)`, pandas会根据**索引对齐**机制，往”A“列填入与其索引相同的Series_B中的数据
- **传入`DataFrame`对象**：同理，根据索引对其进行填充



**`ffill/bfill`：连续填充**

| 功能               | 旧写法（弃用）           | 新写法（推荐） |
| ------------------ | ------------------------ | -------------- |
| **用上一个值填充** | `fillna(method="ffill")` | `ffill()`      |
| **用下一个值填充** | `fillna(method="bfill")` | `bfill()`      |

使用`axis`参数可以改变应用的方向



**`interpolate()`：等差填充**

**插值方式：** **用相邻已知值构造等差数列**来填充缺失值，而不是简单复制上一个或下一个值，让数据变化更平滑自然

- 适合：时间序列、有序数据、数值连续变化的场景

- 不适合：分类数据、文本数据、无规律的离散数据



### 5. 动态填充缺失值

前文介绍的都是填入固定的数值，有时更好的方式是根据已有数据，动态填充缺失值

**例：**在豆瓣评分TOP250的数据表中，部分电影的”语言“这一列出现缺失值。现在填充 “语言” 列的缺失值，要求根据 “国家/地区” 列的值进行填充

> 例如 《海上钢琴师》国家/地区为 意大利，根据其他意大利国家对应的语言来看，应填充为 意大利语

**答案：**

```python
# 第一步：提取出没有缺失值的数据，去重后做成一个字典 (小抄)
# mapping 的结果会是：{'意大利': '意大利语', '美国': '英语'}
mapping = (
    df.dropna(subset=['语言'])              # 踢掉语言为空的行
    .drop_duplicates(subset=['国家/地区'])  # 每个国家只保留一条正确答案
    .set_index('国家/地区')['语言']         # 把国家设为索引，提取语言列
    .to_dict()                            # 终极变身：转为 Python 字典
)

# 第二步：使用 map 函数去字典里查答案，然后用 fillna 填进去
# df['国家/地区'].map(mapping) 会生成一列满满当当的正确语言
df['语言'] = df['语言'].fillna( df['国家/地区'].map(mapping) )
```

> `df['国家/地区'].map(mapping)`会**生成一个和原表行数相同的Series**
>
> 根据python语法，map应该传入一个自定义函数作为参数，这里为什么直接传入了一个字典？
>
> 答：
>
> Pandas 里的 `Series.map()` 是被其作者强化的超级版本。为了极其贴合数据分析师的日常需求，Pandas 底层赋予了 `map()` **“多态（Polymorphism）”** 的能力。
>
> 当 Pandas 发现你喂给 `map()` 的是一个字典时，它的底层逻辑会瞬间切换为**“查字典模式”**。
>
> - **执行逻辑：** 它会把你这一列的每一个元素拿出来，当作字典的 `Key`（键）去查。查到了，就用对应的 `Value`（值）替换；查不到，就默认替换成缺失值 `NaN`。
> - **为什么这么设计？** 因为在底层，字典哈希查找（Hash Table Lookup）的速度是极其恐怖的。如果你有 100 万行数据，用字典映射几乎是瞬间完成的，这比一行行去执行你自定义的 Python 函数要快上几十倍甚至上百倍。
>
> 如果严格按照python语法，应该写成这样：
>
> ```python
> # 传入一个 lambda 函数，利用字典的 get 方法去查
> df['国家/地区'].map(lambda x: mapping_dict.get(x, pd.NA))
> ```
>
> 





## 重复值的处理

### 1. 检测重复值

**检测方法：**

- `duplicated()`：返回布尔对象（True 表示与重复列）

  > 地位和用法与`isna()`类似

**参数含义：**

- `subset`：定义生效范围
- `keep`：决定哪个是本尊
  - `keep='first'`**（默认项）**: 第 1 个是 False，后面的都是 True
  - `keep='last'`: 最后一个是 False，前面的都是 True。
  - `keep=False`：全都是True

### 2. 定位重复值

找出重复的行：`df[df.duplicated()]`

> 和前文的定位缺失值比较，为什么这里不需要`.any()`？
>
> - `df.isnull()` 返回的是二维矩阵，它会把表里的每一个格子都变成 True 或 False。因为是二维的，所以必须用 `.any(axis=1)` ：“横向扫描，把这一行的结果压缩成一个 True/False”。
> - `df.duplicated()` 返回的直接是一维序列，不需要使用`any()`



### 3. 删除重复值

- **删除**：`drop_duplicates()` 返回移除了重复行的 DataFrame。
- **参数**：
  - `keep`：
    - `keep = first`**(默认)**：保留**第一个**出现的值
    - `keep='last'` 保留**最后一个**出现的值
  - `subset`：设置生效范围



## 异常值的替换

> 除了缺失值和重复值，还有其他的异常值（异常数据），比如年龄为-1，999

**`replace`**：`fillna` 是替换空值的特例，而 `replace` 是更通用的替换功能。

- **单值替换**：`data.replace(-999, np.nan)`。

- **多值替换**：
  - 列表对单值：`data.replace([-999, 1000], np.nan)`。
  - 列表对列表（一一对应）：`data.replace([-999, 1000], [np.nan, 0])`。
  - 字典：`data.replace({-999: np.nan, 1000: 0})`。
  
- **定向替换：**

  > 假设表里有一列叫“年龄”，一列叫“商品库存”。它们里面可能都有 `999` 这个数字。 如果用`df.replace({999: np.nan})`，库存里的正常数字 `999` 也会被替换

  ```python
  # 注意：这里是一个字典里嵌套了字典
  df.replace({
      '年龄': {999: np.nan}, 
      '库存': {0: '缺货'}
  })
  ```

### pandas矢量化处理

**用法：**在列名后面加上 `.str`，就可以把**一整列数据**当成一个巨大的字符串，对该列所有数据同时使用python的内置函数

- **`.str.contains()`**: 模糊查询。判断这列的每个格子里是否包含某个关键词，返回 True/False。适合搭配布尔索引使用

  ```python
  # 结合布尔索引，找出商品名里包含“苹果”的数据

  ```

  # 把“地址”一列，拆成三列：'省', '市', '区'

  ```

- `.str.extract()`：正则表达式提取

- `.str.zfill()` ：补齐位数

  # 工号规定必须是 5 位数，但表里存的是int类型： 1, 23, 456
  df['标准工号'] = df['原始工号'].astype(str).str.zfill(5)
  # 先转化数据类型，再自动在左边补0，变成 00001, 00023, 00456
  ```

## 数据排序

### 根据数据内容排序

**相关函数：**`sort_values()`

**参数含义：**

- **`by`**：按哪列排；（可以传一个列名，也可以传一个包含多个列名的列表）。
- **`ascending`**：升序还是降序；默认为 `True`（从小到大）。如果传 `False` 就是从大到小。
- **`na_position`**：如何处理空值 `NaN`； 默认是 `'last'`（统统赶到队伍最后面），也可以设为 `'first'`。

> 实战中经常有“按某一列进行排序，并查看前10名”的需求
>
> ```python
> df.sort_values(by="总分",ascending=False).head(10)
> ```



### 根据索引进行排序

**相关函数：**`sort_index()`

**参数含义：**

- **`ascending`**：控制正序还是倒序。
- **`axis`**：
  - 默认是 `0`（对**行索引**排序）。
  - 如果设为 `1`，就会**对“列名”按字母顺序重新排列**



### TOP K 截断排序

很多时候，我们其实并不需要把 100 万行数据全部排得明明白白。例如只想找出**“薪水最高的前 5 个人”**

如果用 `sort_values().head(5)`，底层会把 100 万行全部排完，再截取前 5 个，极其浪费 CPU 算力。

这时候，用 `nlargest()/ nsmallest()`，Pandas 会在底层使用极其优化的堆排序算法（Heap Queue），只挑出前 5 个，速度非常快

```python
# 找出“薪水”最高的前 5 行数据（等价于降序排序后取前5）
df.nlargest(5, '薪水')

# 找出“年龄”最小的前 3 行数据（等价于升序排序后取前3）
df.nsmallest(3, '年龄')
```



### 排序并返回索引

例：查看各项得分最高的学校名称

相关函数：`idxmax() / idxmin()`

函数功能：`max()`返回的是某一列的最高值，`idxmax()`返回的是这个最高值的`index`

> 默认情况下index是从0开始的整数，不好分辨；所以一般要先用`set_index()`把index设置成方便辨认的列，例如“学校名称”

```python
# 查看各项得分最高的学校名称
df_temp = df.set_index("学校名称")
df_temp.iloc[:,4:].idxmax()
# 中间.iloc[:,4:]的作用：前3列是学校的详细信息，不参与排名
```

# 第五章 数据的增删改与筛选

## 筛选数据

### 1.切片筛选

**选取列：**

- 根据列名：指定某一列`frame['列1']`
  - 一次性选取多个列：`frame[['列1', '列2']]`  传入一个列名的列表

- 




**选取行：**主要函数：`loc[]` 和 `iloc[]`

| 特性         | `loc`                  | `iloc`                  |
| ------------ | ---------------------- | ----------------------- |
| **索引依据** | **标签**（列名和行名） | **位置**（从0开始计数） |
| **切片规则** | **包含**结束标签       | **不包含**结束位置      |
| **适用场景** | 已知行列名称           | 已知行列位置            |
| **灵活性**   | 支持布尔条件           | 仅支持整数索引          |

> 两个函数的参数位置都是：`[行,列]`（先行后列），并且都可以引入第二个引号`:`来设置步长`[:, start:stop:step]`

**行选取对比：**

```python
# loc 用标签
df.loc['a']          # 返回第1行（标签为'a'）
df.loc['a':'c']      # 返回 a,b,c 三行（包含'c'）

# iloc 用位置（从0开始）
df.iloc[0]           # 返回第1行（位置0）
df.iloc[0:2]         # 返回位置0,1两行（不包含位置2）
```

**这两个函数配合`,`和`:`,也能实现选取列的功能**

```python
# loc 用列名
df.loc[:, '姓名']           # 姓名列
df.loc[:, '姓名':'年龄']    # 姓名到年龄列（包含年龄）

# iloc 用列位置
df.iloc[:, 0]              # 第1列
df.iloc[:, 0:2]            # 第1-2列（不包含第3列）
```





### 2.布尔索引筛选

**功能：**通过**布尔条件表达式**来筛选满足条件的行或列

**原理：**用一个布尔数组（True/False）作为"面具"（mask），只保留 True 的位置

**基础用法：**

```python
df = pd.DataFrame({
    'name': ['Alice', 'Bob', 'Charlie', 'David', 'Eve'],
    'age': [25, 30, 35, 28, 22],
    'salary': [5000, 8000, 12000, 6000, 4500],
    'dept': ['IT', 'HR', 'IT', 'Finance', 'HR']
})

mask = df['age'] > 25
df[mask]  
# 或直接用 
df[df['age'] > 25]
```

> - 多条件组合：使用逻辑运算符 `&`（与）、`|`（或）、`~`（非），**必须用括号包裹每个条件**
>   ```python
>   df[(df['dept'] == 'IT') & (df['salary'] > 6000)]
>   ```
>
> - 常常和切片筛选一起组合使用：`df.loc[df>10,:]`进行局部布尔筛选



### 3.SQL筛选

**`query()`**函数可以通过字符串来直接表达筛选条件，写法与`mysql`中的`where`非常相似

例：假设有一份员工表，老板想找出**“年龄大于 30 岁，且在销售部工作”**的人：

```python
import pandas as pd

df = pd.DataFrame({
    '姓名': ['张三', '李四', '王五', '赵六'],
    '年龄': [25, 35, 45, 28],
    '部门': ['销售', '销售', '技术', '技术'],
    '底薪': [5000, 8000, 12000, 6000]
})

# 传统写法（布尔索引）：极其繁琐
result_old = df[(df['年龄'] > 30) & (df['部门'] == '销售')]

# ✅ query 写法：
result_new = df.query("年龄 > 30 and 部门 == '销售'")

```

> 在 `query` 的字符串里，可以直接写列名，甚至可以直接使用 `and`、`or`、`not` 这样的英文单词
> 注意：这里面的相等条件要用`python`中的`= =`（两个等号）

**将外部`python`变量引入条件语句**：**在变量名前面加一个 `@` 符号**

```python
min_age_limit = 30
target_dept = '技术'

# 用 @ 符号表示变量名
result = df.query("年龄 > @min_age_limit and 部门 == @target_dept")

```

**处理带空格的列名**：用反引号``把列名框起来(和MySQL中处理方式相同)

```python
df_messy = pd.DataFrame({'员工 薪资': [5000, 10000]})

# 使用反引号把带有空格（或特殊字符）的列名包起来
result = df_messy.query("`员工 薪资` > 6000")
```



**使用列表判断`in`**

```python
# 找销售部和技术部的人
target_list = ['销售', '技术']

# 写法 A：直接写在字符串里
df.query("部门 in ['销售', '技术']")

# 写法 B：配合 @ 符号引用外部列表（推荐）
df.query("部门 in @target_list")
```

> 类似MySQL中的`WHERE name IN ('A', 'B')`



**SQL筛选与布尔筛选的对比**

| 场景               | 推荐方式                            |
| ------------------ | ----------------------------------- |
| 简单的单条件       | 布尔筛选                            |
| 多条件复杂筛选     | `query()`（可读性更好）             |
| 性能敏感（大数据） | 布尔筛选                            |
| 需要动态构建条件   | 布尔筛选（Python 表达式更灵活）     |
| 列名很长/重复多    | `query()`（避免重复写 `df['xxx']`） |

## 数据计算

> Pandas 的核心特性之一是：在进行算术运算时，会自动根据**索引标签**对齐数据。

- **机制**：根据**索引标签**自动对齐。结果索引是两者的**并集**，找不到的位置产生 `NaN`。
- **算术方法 vs 算术符号**：

| **算术符号** | **对应方法 (Method)**      | **反向方法 (Reverse Method)** | **说明**            |
| ------------ | -------------------------- | ----------------------------- | ------------------- |
| `+`          | **`add()`**                | `radd()`                      | 加法                |
| `-`          | **`sub()`**                | `rsub()`                      | 减法                |
| `*`          | **`mul()`**                | `rmul()`                      | 乘法                |
| `/`          | **`div()`** (或 `truediv`) | `rdiv()`                      | 真除法 (结果是小数) |
| `//`         | **`floordiv()`**           | `rfloordiv()`                 | 地板除 (整除)       |
| `%`          | **`mod()`**                | `rmod()`                      | 取模 (求余)         |
| `**`         | **`pow()`**                | `rpow()`                      | 幂运算              |

算数方法示例：

```python
df1.add(df2)
```

- **核心优势**：符号适合简单场景；而方法（如 `add`）可以通过传入 `fill_value=0` 参数，先填充缺失值再计算，避免结果产生`NaN`。

## 新增行

### 新增少量数据

方法：使用 `.loc[]` 强行塞入

**语法：**传入一个原本不存在的行索引（Index），它就会自动在表格最底下开辟出一行新空间，并把数据填进去。

```python
df = pd.DataFrame({
    '姓名': ['张三', '李四'],
    '年龄': [25, 30]
})

df.loc[2] = ['王五', 28]
```

**优点：** 极其直观，代码短。

**缺点：** 它是在原表上原地动刀子。如果操作频繁，性能会变差。



### 新增大量数据

> 绝对不要在 `for` 循环里，一行一行地向 `DataFrame` 里加数据！
>
> 因为Pandas 的底层逻辑是**“列优化”**的（按列存储和计算极快，按行改动非常慢）。

**方法：**使用`concat()`方法拼接

语法：传入一个包含多个`dataframe`变量名的列表

```python
df = pd.concat([df, new_row], ignore_index=True)
# ignore_index=True 的意思是：拼完之后，把索引重新按 0, 1, 2... 洗牌排好
```

**优点：** 极其正规！而且如果你要一次性新增 5 行，把它放进列表里一起 concat，速度极快。

**实战场景：**先用 Python 原生列表攒起来，最后一次性打包

```python
data_list = [] # 建立一个空列表
for i in range(100000):
    data_list.append({'A': i, 'B': i*2}) 

# 循环结束后，一次性把攒好的列表转化为 DataFrame
df = pd.DataFrame(data_list)
```

> 把for循环的操作用在python的原生列表中（**因为python原生列表的append速度非常快**），最后一次性合并；而不是在dataframe上循环



### 新增到指定位置

> Pandas 底层是极度排斥“在中间插队”这个动作的,因为 Pandas 的底层是基于 NumPy 的 C 语言连续内存数组设计的，所以Pandas 官方**压根没有提供**专属函数

**思路：**把原表在指定位置分为上半表和下半表，把新数据作为一个新的小表，放在中间，最后把三个表按顺序重新拼起来

```python
# 目标：插在物理位置的第 2 行后面
split_idx = 2

# 1. 切出上半截：前两行（张三、李四）
top_half = df.iloc[:split_idx]

# 2. 准备中间的新数据
new_row = pd.DataFrame({'姓名': ['王五'], '年龄': [28]})

# 3. 切出下半截：剩余的所有行（赵六）
bottom_half = df.iloc[split_idx:]

# 4. 把 按照顺序 concat 拼起来，并洗牌索引
df = pd.concat([top_half, new_row, bottom_half], ignore_index=True)
```





## 删除数据

### 删数据不如留数据

在真实的业务中，我们很少会用 `drop()` 去按具体的数字索引删行（因为数据一多，你根本不知道你要删的人在第几行）。 比起“把符合条件的人删掉”，大厂更喜欢反向操作：**“把符合条件的人保留下来（过滤）”**。

**并且，查询数据的速度要比删除数据的速度快得多**



### 删除行\列

函数：`drop()`

用法：直接向`columns index`这两个参数传入一个标签名的列表

```python
df_new = df.drop(columns=['身份证号', '手机号', '家庭住址'])
df_new = df.drop(index=['EMP-001', 'EMP-005'])
```



### 条件删除

**推荐写法：保留不符合条件的，而不是删除符合条件的**

如果必须删除：先利用布尔索引找出符合条件的标签，再用`drop`删除

```python
# 1. 找出所有总分小于60的行的 index（行号）
bad_indexes = df[df['总分'] < 60].index

# 2. 把这些行号喂给 drop 进行删除
df = df.drop(index=bad_indexes)
```

# 第六章：数据聚合与分组运算 

## 数据分组

Hadley Wickham（R 语言大神）提出的 **Split-Apply-Combine（拆分-应用-合并）** 是整个现代数据分析的灵魂所在。

### 1. `groupby` 基础用法

**`groupby()`的作用与MySQL中的`GROUP BY`相同：分组**

**基础语法格式**：

```python
df.groupby('按谁分组')['对谁计算'].怎么算()
```

**参数含义：**

- `by`:核心参数，根据谁进行分组

- **`as_index`:**

  - **`as_index=True`（默认值）：** Pandas 默认会把你分组的依据（比如“部门”）**提拔成新表的 Index（行索引）**。它会变成加粗的表头，你无法像普通列一样用 `df['部门']` 直接提取它。

  - **`as_index=False`**：让分组的依据依然保持为**普通的列**，新表会生成默认的 `0, 1, 2...` 数字索引。

    > 如无特殊情况，可以无脑令**`as_index=False`**

- **`dropna`:**

  - **`dropna=True`（默认值）：** 在分组前，把所有分组依据为空值`NaN`的数据**直接丢弃**。
  - **`dropna=False`（更安全的做法）：** Pandas 会专门为你建一个叫 `NaN` 的组，把那些没填部门的人全都聚在一起进行统计，**防止数据流失。**

- **`sort`:**

  - **`sort=True`（默认值）：** 分完组之后，Pandas 默认会把分组列按字母表或拼音顺序（A到Z）重新排列一次再展示
  - **`sort=False`：** 如果处理的是千万级的数据，且不在乎最后展示的顺序，加上 `sort=False` 可以**极大提升代码的运行速度**，因为它省去了底层排序的算力开销。



> **⚠️ 核心特性注意：**
>
> - 调用 `df.groupby('班级')` 返回的 `GroupBy` 对象**没有进行任何计算**，只含有分组键的中间数据。
>
> - 如果不写 `['对谁计算']`，默认：**把你表格里除了“分组列”之外的所有剩余列，全都一股脑地拉过来，强行执行那个“怎么算”的操作。**
>
>   





### 2. 分组方式

Pandas 极其灵活，不仅能按列名分组，还能用外部数据分组：

| **分组方式**       | **语法示例**                                        | **实战应用场景解析**                                         |
| ------------------ | --------------------------------------------------- | ------------------------------------------------------------ |
| **按单列/多列**    | `df.groupby('key1')`/`df.groupby(['key1', 'key2'])` | **最常用**。多列分组会产生多级索引，即用两个键的组合来唯一标识一行 |
| **按外部数组**     | 传入与行数等长的 NumPy 数组                         | 只要数量对得上，Pandas 就会按顺序把外部数组里的标签，跟表格里的每一行“贴”在一起进行分组。**相当于把数组粘贴上了，构成了临时一列** |
| **按字典映射**     | `df.groupby(mapping_dict)`                          | **临时统计神器**。字典键需对应**行索引 (Index)**。字典没写全的行会被自动忽略，不报错。 |
| **按自定义函数**   | `df.groupby(len)`                                   | 将**行索引**喂给函数（如按名字长度分组）。函数吐出什么结果就分到哪组。 |
| **混合分组**       | 混合函数、字典、列名列表                            | Pandas 会分别计算标签，拼成多级索引 (MultiIndex)。           |
| **按多级索引层级** | `df.groupby(level='cty')`                           | 数据自带多级索引时，直接告诉 Pandas 按哪一级聚合。           |

> **例：按字典映射分组**
>
> 假设你有一张表，以及一个字典，记录了每个员工所属的部门
>
> ```python
> df = pd.DataFrame(
>     {'销量': [100, 200, 150, 300, 250],
>      '退货量': [5, 10, 0, 15, 8]},
>     index=['张三', '李四', '王五', '赵六', '孙七']
> )
> 
> mapping_dict = {
>     '张三': '一部',
>     '李四': '一部',
>     '孙七': '一部',
>     '王五': '二部',
>     '赵六': '二部'
> }
> ```
>
> 原始数据：
>
> ```
>      销量  退货量
> 张三  100    5
> 李四  200   10
> 王五  150    0
> 赵六  300   15
> 孙七  250    8
> ```
>
> 需求：根据部门名单，算出每个部门的总销量
>
> ```python
> grouped3 = df.groupby(mapping_dict)['销量'].sum()
> ```
>
> 结果：
>
> ```
> 一部    550
> 二部    450
> Name: 销量, dtype: int64
> ```
>
> **前提条件**：字典的“键”必须与表格的 **行索引 (Index)** 对应。如果名字在普通的列里（比如叫 `姓名` 列），这种写法会找不到人。
>
> **字典没写全怎么办？**：如果在字典里找不到某个行索引（比如突然来了个“周八”），Pandas 会在分组时**自动忽略**这一行，不会报错。
>
> **为什么不加进原表？**：这种写法不需要向原表插入一个真实的“部门”列，非常适合做**临时性**的快速统计，能保持原表数据的干净。

> **例：按自定义函数分组**
>
> **Pandas 会把你表格的**行索引（Index）一个一个地喂给这个函数。函数吐出什么结果，这行数据就被分到哪个组。
>
> 现有一张员工业绩表
>
> ````python
> df = pd.DataFrame(
>     {'业绩': [10, 20, 30, 40, 50]},
>     index=['Joe', 'Steve', 'Wes', 'Jim', 'Travis']
> )
> ````
>
> ```
>         业绩
> Joe     10
> Steve   20
> Wes     30
> Jim     40
> Travis  50
> ```
>
> 需求：把名字长度一样的人，分到同一个组，然后计算每组的总业绩。
>
> ```python
> result_len = df.groupby(len).sum()
> ```
>
> ```
>    业绩
> 3  80
> 5  20
> 6  50
> ```
>



### 3. `GroupBy` 对象相关操作

- **查看组大小**：加上 `.size()` 方法返回 Series，显示每个分组的数据行数。

- **遍历迭代**：`GroupBy` 对象本质上是一个可迭代的集合，每一次循环都会吐出两样东西：**组名（name）** 和 **这组对应的数据表（group_df）**。
  
  ```python
  grouped = df.groupby('district')
  # 遍历查看内容
  for name, group_df in grouped:
      print(f"========== 【{name}】的分组数据 ==========")
      print(group_df)
  ```
  
- **查看某一组的内容：**`get_group(['组名', ...])`

  ```python
  # 将数据按照 `地址`、`薪资` 进行分组，并查看西湖区薪资为 30000 的工作
  df.groupby(['地址','薪资']).get_group(("西湖区",30000))
  ```

  

- **转化为字典**：`dict(list(df.groupby('key1')))`，可通过字典键直接提取那一组的数据表。

## 数据聚合:`agg`

Pandas聚合函数：**`agg()`**, `agg()`主要有以下四大功能

### 1.使用自定义函数

官方内置的聚合函数有时无法解决复杂的业务需求，就需要使用`agg()`将自定义函数应用到`Dataframe`中去

**语法：**直接将自定义函数传入 `agg`。例如 `grouped.agg(get_diff)`  其中`get_diff`是自定义函数的名称

```python
# 自定义函数：计算最大值与最小值的差
def get_diff(x):
    return x.max() - x.min()
grouped = df.groupby('班级')

result = grouped.agg(get_diff)

```

也可以直接将**匿名函数**传入`agg`：

```python
df.groupby('district')['salary'].agg(lambda x: x.max() - x.min())
```



### 2.对同一列使用多种聚合函数

**业务场景：** 老板想看各部门的薪资情况，不仅要平均数，还要最高薪资、最低薪资和最高工资与最低工资之差。

**语法：**传入一个由聚合函数名构成的列表

```PYTHON
# 对同一列（salary），同时应用 4 种不同的聚合计算
df.groupby('district')['salary'].agg(['mean', 'max', 'min', get_diff])
```

> - **官方函数需加引号，自定义函数不加**
> - 生成的新列，列名会自动变为函数名



### 3.多列数据定向聚合

**业务场景：** 查看薪资的平均值，年龄的最大值

语法：向`agg()`传入一个字典，其中**键是列名，值是你对这列使用的聚合函数**

```python
df.groupby('district').agg({
    'salary': 'mean',  # 薪资求均值
    'age': 'max'       # 年龄求最大值
})
```



### 4.命名聚合(重要)

将前面三个功能结合起来，并且直接命名新生成的列

**语法：**传入一系列元组，格式为：`新列名称 = (数据列名，聚合函数名)`，每个元组用逗号隔开

```python
df.groupby('district', as_index=False).agg(
    平均薪资 = ('salary', 'mean'),
    最老员工年龄 = ('age', 'max'),
    部门总人数 = ('salary', 'count')
)
```

## 数据聚合:`apply`



`apply()`的基础功能与`agg()`类似：对`dataframe`使用自定义函数，主要有以下三种应用场景



### 1.对单列使用自定义函数

**基础语法：**`df.apply(自定义函数)`

```python
# 场景：把所有人的薪资扣除 1000 元税后，再打 8 折（这种奇葩逻辑没有内置函数）
df['实发薪资'] = df['薪资'].apply(lambda x: (x - 1000) * 0.8)
```

> 基础语法也与`agg()`类似



### 2.跨列计算

**语法：**必须加上参数 **`axis=1`**，实现**跨列横向运算**

**场景**：表里有“单价”和“数量”，新增一列“总价”（单价 $\times$ 数量）

```python
df2 = pd.DataFrame({
    '商品': ['苹果', '香蕉'],
    '单价': [5, 3],
    '数量': [10, 20]
})

def sum_money(x):
    return x['单价'] * x['数量']
df2['总价'] = df2.apply(sum_money, axis=1)

print(df2)
```



### 3.在`groupby`后面使用

即便是在`groupby`后面，有的功能`agg()`是无法实现的，例如：排名（因为排名不是聚合函数）

**场景（经典面试题）**：找出每个部门业绩排名前 2 的员工。

```python
df3 = pd.DataFrame({
    '部门': ['销售', '销售', '销售', '技术', '技术', '技术'],
    '姓名': ['A', 'B', 'C', 'D', 'E', 'F'],
    '业绩': [100, 300, 200, 500, 400, 600]
})
```

```python
# 1. 写一个处理“小表格”的函数
def top_2(group_df):
    # 对进来的小表格按业绩降序排，然后取前两行
    return group_df.sort_values(by='业绩', ascending=False).head(2)

# 2. 分组并 apply
# 这里的 apply 会把“销售部的小表”和“技术部的小表”分别送进 top_2 函数里
result = df3.groupby('部门').apply(top_2)

print(result)
```





### 4.`apply`和`agg`的区别



`apply()`和`agg()`有着本质区别：

- `agg()`:只能处理一列，对某一列进行聚合处理；且必须配合`groupby()`一起使用

- `apply()`:本质上是Python 写了巨大的 `for` 循环，用法非常自由，不一定要和`groupby()`一起用

  > 但当数据量大时，性能开销非常大

| **对比维度**              | **agg() **                   | **apply() **                      |
| ------------------------- | ---------------------------------- | --------------------------------------- |
| **底层逻辑**              |只能传入聚合函数|任何函数都可以|
| **函数接收的输入(核心!)** | 当前分组的**每一列** | 当前分组的**整个子表** |
| **函数输出结果**          | **单一值**                         | 单一值、Series 甚至 DataFrame      |
| **跨列计算能力**          | 极弱（只能看一列算一列）           | **极强**（可调用多列互相加减乘除）      |
| **多任务能力**            | 极强（支持字典法同时算多个）       | 弱（通常一次执行一个自定义逻辑）        |

> **函数“看到”的输入不同(跨列计算能力)**
>
> - **`agg()`：按“列”切片送进去**
>
>   当你在 `groupby` 后使用 `agg(自定义函数)` 时，Pandas 会把当前分组的**每一列**（作为一个一维的 Series）分别、单独地送进你的函数里。你的函数在同一时间，只能看到“某一列”的数据，看不到其他列。
>
> - **`apply()`：把“整个组”打包送进去**
>
>   当使用 `apply(自定义函数)` 时，Pandas 会把当前分组的**整个子表格**（作为一个二维的 DataFrame）一次性端给你的函数。你的函数可以同时调用这个组里的好几列数据来做复杂运算。
>
> **场景对比：**
>
> 假设我们要计算“利润率（利润/销售额）”。
>
> - `agg()` **做不到**。因为它每次只能拿到“利润”这一列或者“销售额”这一列，它没法同时拿着两列相除。
> - `apply()` **可以做到**。因为它拿到的是包含了这两列的整个表格。





## 窗口函数:`transform`

- **等价于**：SQL 里的 `OVER (PARTITION BY ...)`。也等同于使用 apply 后，再手动把结果列逐个加回原表。

- **实战场景**：算出每个部门的平均工资，**不浓缩表格**，直接作为新的一列原封不动贴回 1000 行的明细表中，用于计算每个人的工资差额。

  ```python
  df = pd.DataFrame({
      '姓名': ['张三', '李四', '王五', '赵六'],
      '部门': ['销售', '销售', '技术', '技术'],
      '工资': [5000, 7000,  8000, 12000]
  })
  ```

  使用`transform`:

  ```python
  df['部门平均工资'] = df.groupby('部门')['工资'].transform('mean')
  
  # 算一下每个人比部门平均工资高多少
  df['工资差额'] = df['工资'] - df['部门平均工资']
  
  print(df)
  ```

## 透视表与交叉表

### 1. `pivot_table` (数据透视表)

完全对应 Excel 数据透视表的拖拽动作：

- **`index`**：拖到“行”区域（左侧表头）。
- **`columns`**：拖到“列”区域（上方表头）。
- **`values`**：拖到“值”区域（用来计算的数字列）。
- **`aggfunc`**：值的汇总方式，默认 `'mean'`，可改 `'sum'`, `'count'` 或传字典。
- **`margins=True`**：自动生成总计(All)行列。
- **`fill_value`**：填补空白单元格（如填 `0`）。

**对决 `groupby`**：

在底层源码里，`pivot_table` 其实就是调用了 `groupby`。 `groupby` 配合 `unstack()` 也能做二维表，但 `pivot_table` 的优势在于**极简生成总计行列**和**一键填补空缺值**。不过，`pivot_table` 仅限于聚合，而 `groupby` 还能进行转换和过滤。

### 2. `crosstab` (数据交叉表)

专门用来**“数人头”（统计频次）**的顶层函数。

- **基础陷阱**：必须传入具体的列对象（如 `df['列名']`），而不是列名字串。
- **杀手锏参数 (`normalize`)**：用于计算百分比。
  - `'all'`：全局占比。
  - **`'index'` (最常用)**：行的百分比（如各个部门内的男女比例）。
  - `'columns'`：列的百分比。

# 第七章：数据塑形 

## 为什么要数据塑形？

在数据建模的规范里，**事实表必须是“长表”（一维表）**。所以无论是用 Python 写脚本，还是用 Power Query 点按钮，我们最常做的动作其实都是 **逆透视 (Unpivot / melt)**，把人类爱看的二维横向报表，洗成数据库喜欢的一维流水线。

### 1. 宽表VS长表

**宽表（给人类看的报告）**：列数很多，常见于 Excel 表格。

| **分店** | **1月销量** | **2月销量** | **3月销量** |
| -------- | ----------- | ----------- | ----------- |
| 北京店   | 100         | 120         | 150         |
| 上海店   | 80          | 90          | 110         |

**长表（给机器和数据库看的表）**：核心原则是**“每一行只记录一个不可分割的最小业务动作或观察结果”**。列数极少，只有清晰的“属性（维度）”和“数值（度量）”。行数变多了（原来的 2 行变成了 6 行）。

| **分店 (维度)** | **月份 (维度)** | **销量 (度量)** |
| --------------- | --------------- | --------------- |
| 北京店          | 1月             | 100             |
| 北京店          | 2月             | 120             |
| 北京店          | 3月             | 150             |
| 上海店          | 1月             | 80              |
| 上海店          | 2月             | 90              |
| 上海店          | 3月             | 110             |

### 2. 长表的优势分析

- **极度灵活**：如果想算总销量，直接对“销量”这一列做 `SUM()` 即可，不管你有几个月。
- **方便切片**：可以直接拿“月份”这一列作为筛选器（切片器），选“1月”，数据就自动过滤了。
- **扩展性强**：如果 4 月份的数据来了，宽表需要“增加一列”，这会破坏表结构；而长表只需要在最下面“追加几行”即可，表结构坚如磐石。

## 多级索引

> 多级索引（MultiIndex）是处理高维数据（例如“省份-城市-门店”层级结构）的利器。把它想象成文件管理系统中的文件夹路径。

### 1. 多级索引的概念与创建

**含义**：在一个轴（行或列）上拥有两个或两个以上的索引层级。就像一本教科书，有**章**，章下面有**节**，节下面才是**内容**。

- **对象本质**：生成的索引对象是 `MultiIndex`，包含 `levels`（唯一值）和 `labels`（整数标签），它是 `Index` 的**子类**。

**创建方式：**

1. **基于已有列转换**：将已有数据转换为多级索引`set_index([索引名列表])`

   ```python
   data = {
       '班级': ['一班', '一班', '二班', '二班'],
       '姓名': ['张三', '李四', '王五', '赵六'],
       '分数': [80, 90, 85, 95]
   }
   df = pd.DataFrame(data)
   
   # 把“班级”设为第一层，“姓名”设为第二层
   df_multi = df.set_index(['班级', '姓名'])
   # 注意要用另一个变量去接收，原DataFrame不会改变
   ```

2. **数组拉链式拼装**：索引数据分散在两个独立的列表里，把它们“拉链”式地拼起来`pd.MultiIndex.from_arrays`

   `index = pd.MultiIndex.from_arrays(arrays, names=['班级', '性别'])`

   ```python
   # 两个等长的列表
   arrays = [
       ['一班', '一班', '二班', '二班'],  # 第一层
       ['男', '女', '男', '女']         # 第二层
   ]
   
   # 拼装
   index = pd.MultiIndex.from_arrays(arrays, names=['班级', '性别'])
   
   df = pd.DataFrame({'人数': [10, 20, 15, 25]}, index=index)
   print(df)
   ```

   

### 2. 选取与切片速查

| **需求**                            | **推荐方法**     | **代码示例**               |
| ----------------------------------- | ---------------- | -------------------------- |
| **只选最外层** (比如所有 2023)      | **直接 `loc`**   | `df.loc['2023']`           |
| **精准选某一行** (比如 2023 年上海) | **元组 `loc`**   | `df.loc[('2023', '上海')]` |
| **只选内层** (跨层选，比如所有上海) | **利用冒号 `:`** | `df.loc[:,'上海']`         |

### 3. 层级重组

在处理多级索引时，经常需要调整索引的层级顺序（比如把“内层”换到“外层”），这就是 `swaplevel` 的作用。

但调整完顺序后，数据的物理排列往往是乱序的，这会导致切片报错或性能下降，所以紧接着必须用 `sort_index` 来重新整理数据。

- **`swaplevel(i, j, axis=0)`**：**交换层级（换位置）**。

  - **含义**：i, j 是你要交换的两个层级的名称

  - **比喻**：原来按“年份 -> 城市”分类，改成按“城市 -> 年份”分类。

    ```python
    # 1. 原始数据：第一层是年份，第二层是城市
    index = pd.MultiIndex.from_product(
        [
            ['2023', '2024'], 
            ['北京', '上海']], 
        names=['年份', '城市'])
    df = pd.DataFrame({'销量': [100, 200, 150, 300]}, index=index)
    
    # 2. 交换层级：把“城市”换到外层，“年份”换到内层
    df_swapped = df.swaplevel('年份', '城市')
    df_swapped
    ```

    ```
                销量
    城市  年份      
    北京  2023   100  <-- 北京
    上海  2023   200  <-- 上海 (乱入)
    北京  2024   150  <-- 北京 (又回来了)
    上海  2024   300
    ```

  - **隐患**：交换后，数据的物理排列往往是乱序的（北京和上海穿插在一起），这会导致切片报错或性能下降。

- **`sort_index()`**：**重新排序（理顺）**。

  - **作用**：把索引按字典序（A-Z 或 0-9）排列整齐。

  - **语法**：

    ```python
    # level: 指定按哪一层排（通常不写，默认全排）
    # ascending: True(升序) / False(降序)
    df.sort_index(level=0, inplace=True)
    ```

    接上面的代码演示:

    ```python
    df_sorted = df_swapped.sort_index()
    print(df_sorted)
    ```

    ```
               销量
    城市  年份      
    上海  2023   200  <-- 上海的数据聚在一起了
        2024   300
    北京  2023   100  <-- 北京的数据聚在一起了
        2024   150
    ```

  - **必要性**：Pandas 的切片操作强烈依赖索引有序性。如果索引是乱的，`loc` 切片会失效

在实际工作中，这两个函数通常是连在一起写的：

```python
df_final = df.swaplevel('年份','城市').sort_index()
```

## 多表查询

> **分析师视角**：这是日常工作的高频场景。`merge` 用于横向扩展字段（类似 Excel 的 VLOOKUP 或 SQL 的 JOIN）；`concat` 用于纵向追加数据（比如把 1-12 月的 CSV 报表叠在一起）。

### 1. 横向合并

函数：`pd.merge`

可根据一个或多个键将不同的 DataFrame 连接起来，类似于 SQL 中的 JOIN 操作。

- **连接键指定**：
  - 默认将**重叠列的列名**当作键。
  - `on='key'`：指定相同列名。
  - `left_on='lkey', right_on='rkey'`：左右列名不同时分别指定。
  - `left_index=True` / `right_index=True`：使用索引作为连接键。
- **连接方式 (`how` 参数)**：
  - `inner` (默认)：使用两个表都有的键（交集）。
  - `left` / `right`：使用左表 / 右表中所有的键。
  - `outer`：使用两个表中所有的键（并集）。
  - `cross`：产生笛卡尔积。
- **多对多与重复列名**：多对多产生行的**笛卡尔积**。重叠的非连接键会通过 `suffixes=('_x', '_y')` 自动添加后缀。

### 2. 竖向堆叠 

函数：`pd.concat`

核心思想是沿着某条轴，把两个表硬生生地“粘”在一起（如：1月份表 + 2月份表 = 总表）。

- **方向**：`axis=0` (默认，纵向堆叠)；`axis=1` (横向堆叠)。
- **连接逻辑**：`join='outer'` (默认并集)；`join='inner'` (交集)。
- **实用参数进阶**：
  - **`keys`**：在连接轴上创建**多级索引**，用于区分数据来源（如区分数据属于哪个月）。
  - **`ignore_index`**：如果不保留连接轴上的索引，产生新整数索引 `0, 1, ..., n-1`。
  - **`verify_integrity`**：检查新轴上的重复情况（默认 False）。

### 3. 合并重叠数据

函数：`combine_first`

用于处理两个数据集索引部分重叠，需要修补的情况。

- **逻辑**：优先用自己的数据，如果自己是 NaN，就用别人的数据补上（同时带有数据自动对齐功能）。
- **示例**：`df1.combine_first(df2)` 用 df2 修补 df1 的空缺。

## 重塑与轴向旋转

> **分析师视角**：当你能熟练区分并在脑海中可视化 `stack/unstack` 和 `pivot/melt` 的变形过程时，你的 Pandas 功底就已经超越了 80% 的初学者。

### 1. 依托于多级索引的重排

多级索引为数据重排提供了一致性方式。

- **`stack()` (堆叠)**：把最里面的**列索引 (Columns)** 旋转 90 度，压到**行索引 (Index)** 的最内层。
  - 结果：表格变长、变瘦。默认滤除缺失数据（NaN）使运算可逆（可传 `dropna=False` 保留）。
- **`unstack()` (拆堆)**：把最里面的**行索引 (Index)** 旋转 90 度，推到**列索引 (Columns)** 上去。
  - 结果：表格变宽、变扁。如果级别值未在各分组找全，会引入缺失数据 (NaN)。

### 2. 透视与逆透视

时间序列或数据库多为长表，分析制图多需宽表。

**将“长表”旋转为“宽表”/透视 (`pivot`)**

- `index`：指定作为新表**行索引**的列（类似 SQL 的 `GROUP BY`）

  > 相当于 SQL 里 `GROUP BY` 后面的那个字段。

- `columns`：指定作为新表**列名**的列

  > 即把原来**竖着排一列的数据横过来，作为表头**

- `values`：填入**单元格**的数据列

  > 如果不填这个参数，Pandas 会默认把**剩下所有列**都作为数据填进去（形成多级**列**索引，其中`columns`的列是最内层）。

- *底层本质*：`pivot` 等价于 `set_index` 再 `unstack`。

**场景：有一份简单的销售数据（长表）**

```python
data2 = {
    '日期': ['1月1日', '1月1日', '1月2日', '1月2日'],
    '商品': ['苹果', '香蕉', '苹果', '香蕉'],
    '销量': [100, 200, 150, 300],
    '价格': [5, 3, 5, 3]
}
df2 = pd.DataFrame(data2)

print("--- 原始长表 ---")
print(df2)
```

原始数据：

```
    日期  商品   销量  价格
0  1月1日  苹果  100   5
1  1月1日  香蕉  200   3
2  1月2日  苹果  150   5
3  1月2日  香蕉  300   3
```

要求：我想看“不同日期中，各个商品的销量”

```python
df2_pivoted = df2.pivot(index='日期', columns='商品', values='销量')
print(df2_pivoted)
```

结果：（宽表）

```
商品    苹果   香蕉
日期           
1月1日  100  200
1月2日  150  300
```

如果不填`values`参数：形成多级索引

```
		 销量       	 价格     
商品   苹果   香蕉   苹果  香蕉
日期                     
1月1日  100  200    5   3
1月2日  150  300    5   3
```



**将“宽表”旋转为“长表”/逆透视 (`melt`)**

`pivot` 的逆运算，将多个列合并为一个，产生比输入更长的 DataFrame。

- `id_vars` (身份列/锚点)：不想融化的列，作为**标识符**保留（如姓名）

  > `id_vars`参数的那一行就是在变成长表时，自动向下复制填充（拉伸）的那一行，即`pivot()`中的`index`参数的反面：
  > 假如我对一张表进行了pivot，如果我想使用melt函数再把它变回原样，我pivot的index参数和melt的id_vars参数应该是同一个列名

- `value_vars` (要融化的列)：要把哪些列表头折叠进一列（默认融化除 `id_vars` 外所有列）。

- `var_name`：给融化后的表头（新产生的一列）起的新变量名（如“科目”）。

- `value_name`：给融化后的数值产生的新一列起的名字（如“分数”）

**场景：有一张成绩单(宽表)**

```python
df = pd.DataFrame({
    '姓名': ['张三', '李四'],
    '语文': [90, 80],
    '数学': [85, 95]
})

print("--- 原始宽表 (Wide) ---")
print(df)
```

结果

```python
   姓名  语文  数学
0  张三  90  85
1  李四  80  95
```

把“语文”和“数学”这两列融化掉，变成“科目”，剩下的数据形成新的一列，取名“分数”

```python
df_melted = pd.melt(df, 
                    id_vars=['姓名'], 
                    value_vars=['语文', '数学'],
                    var_name='科目', 
                    value_name='分数')

print("\n--- 融化后的长表 (Long) ---")
print(df_melted)
```

```
   姓名  科目  分数
0  张三  语文  90  <-- 张三的第一条数据
1  李四  语文  80
2  张三  数学  85  <-- 张三的第二条数据 (姓名重复了)
3  李四  数学  95
```



### 3. 四大变形函数对比

| **特性**     | **pivot**             | **melt**                  | **stack**                 | **unstack**                 |
| ------------ | --------------------- | ------------------------- | ------------------------- | --------------------------- |
| **变换方向** | **长 -> 宽**          | **宽 -> 长**              | **宽 -> 长**              | **长 -> 宽**                |
| **操作对象** | **列里的值** (Values) | **列的名字** (Headers)    | **列索引** (Column Index) | **行索引** (Row Index)      |
| **主要用途** | 制作透视表、报表展示  | 数据清洗、为绘图/ML做准备 | 多级索引处理、矩阵压缩    | `groupby`结果展开，矩阵展开 |
| **索引变化** | 指定列变为行/列索引   | 不涉及复杂索引变化        | **列索引变行索引**        | **行索引变列索引**          |
| **逆运算**   | 对应 `melt`           | 对应 `pivot`              | 对应 `unstack`            | 对应 `stack`                |

## Pandas 独有功能

Pandas 是用 Python 写的，这意味着它不仅能做关系型代数，还能做大量复杂的数据结构转换和科学计算。以下这些函数，如果在 MySQL 里写，往往需要几百行复杂的 SQL 语句，而在 Pandas 里只需一行

### 1. 结构重塑

这部分是数据清洗师最爱的功能，MySQL 里需要用无数个 `SUM(CASE WHEN...)` 才能勉强实现。

- **`pd.pivot_table()`**：数据透视表（长表转宽表，带行列总计）。
- **`pd.melt()`**：逆透视（宽表变长表），这是机器学习特征工程前必做的动作。
- **`pd.crosstab()`**：交叉频次表，秒算各类别的占比 (`normalize='index'`)。

### 2. 高阶缺失值处理

MySQL 遇到 NULL 通常只能用 `COALESCE` 给个默认值，但 Pandas 提供了插值等高级玩法：

- **`df.fillna(method='ffill')`**：向下填充（比如把上面的“北京”自动填充给下面的空单元格）。
- **`df.interpolate()`**：线性插值。如果第一天卖了 10，第三天卖了 30，第二天是空值，它会自动帮你算出 20 填进去。

### 3. 文本与时间操作

Pandas 有两个超级命名空间：`.str` 和 `.dt`。

- **`df['时间'].dt.day_name()`**：直接告诉你这一天是星期几。
- **`df['文本'].str.extract(r'([A-Z]\d{3})')`**：完美融合正则表达式，瞬间从杂乱的一大段文本中把包含“一个大写字母加3个数字”的特定编号提取成新的一列。

### 4. 自定义函数

- **`df.apply(自定义函数)`**：这就是我们在上一个回合讲的，如果所有的内置规则都搞不定，直接扔一个 Python 函数进去暴力遍历，MySQL 虽然有自定义函数，但在易用性和生态上完全无法与 Python 匹敌。





# Pandas 链式编程

和 SQL 里的视图或者很多原地生效的编程命令不同，Pandas 非常“谨慎”。默认情况下，Pandas 里的绝大多数函数（比如 `rename`, `drop`, `fillna`）都**不会**修改原本的表格。想要对函数生效后的表格进行下一步操作，就必须用一个新的变量去接收它。

> 在实际工作中，经常需要对同一个表进行多个操作，如果中间每个操作都要一个新的变量来接收的话，就会产生大量的临时变量，这样难写也难读

**解决方法：链式编程**

Pandas支持链式编程，多个操作之间可以用`.`来连接，避免产生过多的中间临时变量

## 处理换行

Python 原生是不允许代码随便换行的。如果用 `.` 一直连写下去，代码会变成长长的一条线，极其反人类。

**解决方法：** 在整段代码的最外层套上一个大括号 `()`。之后就可以在每个 `.` 前面自由换行，写出像 SQL 一样层次分明的代码

```python
# 优雅的链式排版，结构就像 SQL 一样清晰
result = (
    df
    .query("age > 18")           # 相当于 WHERE
    .groupby('department')       # 相当于 GROUP BY
    .agg({'salary': 'mean'})     # 相当于 SELECT AVG(salary)
    .reset_index()               # 把索引还原成普通列
    .sort_values('salary')       # 相当于 ORDER BY
)
```

> 虽然排版看起来像`mysql`，但是这些命令的执行顺序是按照你编写的顺序从上到下执行，而不是像mysql那样每个命令之间有先后关系
