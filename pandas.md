# 第一章 数据结构与基础操作 

## 三大数据结构

### 1. Series

Series 是一种类似于一维数组的对象，可以看作是**定长的有序字典**，或**带标签的一维数组**。

- **本质**：Numpy 数组穿上了一层“标签”外衣，完美继承了 Numpy 的各种功能（如布尔索引）。

- **组成**：一组数据（各种 Numpy 数据类型） + 一组数据标签（索引）。

- **创建方式**：

  - **由列表创建**：`pd.Series([data])`（默认 0 到 N-1 整数索引）；`pd.Series([data], index=['a', 'b'...])`（指定索引，长度须一致）。
  - **由字典创建**：`pd.Series(dict)`（键自动成为有序索引）；`pd.Series(dict, index=[list])`（按 index 列表找值，字典没找的填 `NaN`，多余的丢弃）。

- **核心属性与操作**：

  - `obj.values` (返回 Numpy 数组)；`obj.index` (返回索引对象)。

  - **获取与修改索引**：可通过 `obj.index = ['Bob', ...]` 就地修改。

  - **Name 属性**（极其重要）：

    - `Series.name`：对象本身的名字（变 DataFrame 一列时，这就是列名）。

    - `Series.index.name`：索引的名字（增加可读性，如“商品ID”）。

      ```python
      s = pd.Series(
          [10, 20, 30],
          index = ['001','002','003'], 
          name='价格')        # 设置Series名称为'价格'
      s.index.name = '商品ID' # 设置索引名称为 '商品ID'
      ```

      ```
      商品ID
      001    10
      002    20
      003    30
      Name: 价格, dtype: int64
      
      # 该表的含义为：ID编号为001的商品价格是10
      ```

  - **操作特性**：Series 支持 Numpy 数学运算（索引链接保留）；支持类似字典的 `in` 成员检测。

  - **自动数据对齐 (重点)**：算术运算时根据**标签**自动对齐（类似 SQL 里的 `JOIN` 操作），不重叠部分产生 `NaN`。

- **缺失数据处理**：`NaN` 表示缺失。用 `pd.isnull(obj)` 或 `pd.notnull(obj)` 检测，返回布尔值,其中`obj`是Series名

  > 输出结果是一个和obj形状一样的表格，来表示每个位置是否是NaN



### 2. DataFrame

由一组有序的列（Series）组成的数据结构。数据以二维块存放。

- **创建方式**：

  - **基础字典**：传入等长列表/Numpy数组的字典。键变列名，自动加索引。

    ```python
    data = {
        'state': ['Ohio', 'Ohio', 'Ohio', 'Nevada', 'Nevada', 'Nevada'],
        'year': [2000, 2001, 2002, 2001, 2002, 2003],
        'pop': [1.5, 1.7, 3.6, 2.4, 2.9, 3.2]}
    frame = pd.DataFrame(data)
    ```

  - **嵌套字典**：外层键变列（Columns），内层键变行索引（Index），内层键会自动合并排序。支持 `frame.T` 转置。

  - **Series 字典**：每个 Series 成为一列，索引自动合并。

- **大小查看**：`DataFrame.shape`（返回 `[行数, 列数]` 的列表，是属性无括号）。

- **基础增删改查**：

  - **指定列序**：创建时用 `columns=['A', 'B']` 指定，找不到的填 `NaN`。
  - **查**：`frame['列名']` 或 `frame[['列1', '列2']]`。返回的是**视图**，修改会影响原数据（需复制用 `.copy()`）
  - **改**：
    - **标量赋值**：`frame['col'] = value`。会将整列设为该值 
    - **数组/列表赋值**：`frame['col'] = array/list`。长度必须与 DataFrame 长度匹配 
    - **Series 赋值**：`frame['col'] = series`。会精确匹配 DataFrame 的索引，空位填 `NaN` 
  - **增**：直接给不存在的列赋值 `frame['新列'] = ...`
  - **删**：
    - 删除指定数据：`drop`
      - 删除行（默认）：`data.drop(['Row1', 'Row2'])`。
      - 删除列：需要指定轴:`axis=1` 或 `axis='columns'`。
        - 示例：`data.drop('col_name', axis=1)`。
      - **就地修改**：加参数 `inplace=True` 会直接销毁原数据修改对象，不返回新对象（需谨慎）。
    - 删除重复值：`drop_duplicates(subset=['user_id'])`（不加 参数`subset` 时，要求整行全同才删）
    - 删除缺失值：`dropna(subset=['手机号'])`（不加参数`subset`时，一行有一个空值就整行删掉）
  - **修改某列数据类型**：`df['列名'] = df['列名'].astype(目标类型)`



### 3. Index

管理轴标签和元数据的强大、**不可变**的数据结构（有序集合）。

- **不可变性**：一旦创建不能修改内部元素（如 `index[1] = 'd'` 报错），保证多结构间安全共享。

  > **Index 对象本身**是**不可变**的：你不能修改它里面的某个特定元素。
  >
  > **Series/DataFrame 的 `.index` 属性**是**可被赋值**的：你可以把旧的 Index 扔掉，换一个新的上去，这个新的index依然无法被修改，只能被替换

- **集合特性**：类似固定集合，支持 `in`。但与 Python 原生 Set 不同，Pandas Index **可以包含重复标签**（选重复标签会展示所有结果）。

- **对比：`.index` vs `.columns`**（本质都是 Index 对象）：

| **特性**     | **.index**            | **.columns**                   |
| ------------ | --------------------- | ------------------------------ |
| **方向**     | **垂直方向 (Axis 0)** | **水平方向 (Axis 1)**          |
| **别名**     | 行标签 (Row Labels)   | 列名 (Column Names/Headers)    |
| **代表意义** | 每一行代表**“是谁”**  | 每一列代表**“有什么特征”**     |
| **切片方式** | `df.loc['行名']`      | `df['列名']`                   |
| **数据类型** | `pandas.Index` 对象   | **也是 `pandas.Index` 对象！** |

- **常用方法**：
  - **集合运算**：`append`, `difference` (差集), `intersection` (交集), `union` (并集)。
  - **修改重组**：`delete`, `drop`, `insert`（均返回新 Index）。
  - **属性逻辑**：`isin`, `is_monotonic` (单调递增), `is_unique` (无重复值), `unique` (返回唯一值数组)。

## 筛选查询与排序

> **分析师批注**：熟练掌握这一块，你就能精准提取任何需要分析的数据切片，为后续导入 Matplotlib 绘制可视化图表或进行统计分析做好准备。

### 1. 切片

- **Series 切片特殊性**：Pandas 的标签切片 `['a':'c']` **包含末端**（Python 原生 `[0:2]` 不包含）。
- **带有重复标签的轴索引**：用 `obj.index.is_unique` 检查。若标签对应单个值返回标量；对应多个值则返回 Series/DataFrame。
- **DataFrame 选取方法全景表**：

| **类型**            | **说明**                            |
| ------------------- | ----------------------------------- |
| `df[val]`           | 选取列；或是特殊的行切片/布尔过滤。 |
| `df.loc[val]`       | **按标签**选取行（单行或多行）。    |
| `df.loc[:, val]`    | **按标签**选取列。                  |
| `df.loc[row, col]`  | **按标签**同时选取行和列。          |
| `df.iloc[where]`    | **按整数位置**选取行。              |
| `df.iloc[:, where]` | **按整数位置**选取列。              |
| `df.iloc[r, c]`     | **按整数位置**同时选取行和列。      |
| `df.at[r, c]`       | 按标签选取单一标量（速度更快）。    |
| `df.iat[i, j]`      | 按位置选取单一标量（速度更快）。    |

> 以上所有的切片方式都可以和**布尔索引**结合起来使用
>
> 例如：`df.loc[df>10,:]`



### **2.条件筛选：`pd.query`**

`query`函数可以通过字符串来直接表达筛选条件，写法与`mysql`中的`where`非常相似

- 基础用法：

例：假设有一份员工表，老板想找出**“年龄大于 30 岁，且在销售部工作”**的人：

```python
import pandas as pd

df = pd.DataFrame({
    '姓名': ['张三', '李四', '王五', '赵六'],
    '年龄': [25, 35, 45, 28],
    '部门': ['销售', '销售', '技术', '技术'],
    '底薪': [5000, 8000, 12000, 6000]
})

# 传统写法（布尔索引）：极其繁琐，满眼都是 df 和括号
result_old = df[(df['年龄'] > 30) & (df['部门'] == '销售')]

# ✅ query 写法：
result_new = df.query("年龄 > 30 and 部门 == '销售'")

print(result_new)
```

> 在 `query` 的字符串里，可以直接写列名，甚至可以直接使用 `and`、`or`、`not` 这样的英文单词
> 注意：这里面的相等条件要用`python`中的`= =`（两个等号）

- **将外部`python`变量引入条件语句**：**在变量名前面加一个 `@` 符号**

```python
# 老板动态给了一个最低年龄限制
min_age_limit = 30
target_dept = '技术'

# 用 @ 符号直接把外面的变量“吸”进字符串里
result = df.query("年龄 > @min_age_limit and 部门 == @target_dept")

print(result)
# 输出王五的数据
```



- **处理带空格的列名**：用反引号``把列名框起来(和mysql中处理方式相同)

```python
df_messy = pd.DataFrame({'员工 薪资': [5000, 10000]})

# 使用反引号把带有空格（或特殊字符）的列名包起来
result = df_messy.query("`员工 薪资` > 6000")
```



- 使用列表判断`in`

```python
# 找销售部和技术部的人
target_list = ['销售', '技术']

# 写法 A：直接写在字符串里
df.query("部门 in ['销售', '技术']")

# 写法 B：配合 @ 符号引用外部列表（推荐）
df.query("部门 in @target_list")
```

> 类似myslq中的`WHERE name IN ('A', 'B')`





### 3. 索引的重排序

函数：`reindex`

- **作用**：按新索引顺序重新排序；新索引不存在则填 `NaN`。想引入原本不存在的行（并填充默认值），或者强制让数据符合特定顺序，`reindex` 是唯一选择。
- **参数**：`index`/`columns`（重排行或列）、`method`（插值方式，如 `ffill`）、`fill_value`（替代值，如 0）、`limit`（最大填充量）、`copy`。
- **实战场景**：时间序列补全。商店 2 号和 4 号没开门没数据，直接算平均值或画图会错，必须用 `reindex` 补全 1 到 5 号并用 `fill_value=0` 填充。

### 4. 排序和排名

- **按索引排序**：`sort_index(axis=0/1, ascending=False/True)`，根据字典顺序排。
- **按值排序**：`sort_values(by=['a', 'b'])`，缺失值 `NaN` 默认放末尾。
- **排名 (rank)**：计算数据在数组中的排名（这和 SQL 里的窗口函数 `RANK()` 逻辑是一致的）。
  - `method='average'` (默认)：平级分配平均排名（如两个第6名，均为 6.5）。
  - `method='first'`：按数据出现顺序排。
  - `method='min'` / `max`：使用分组的最小/最大排名。
  - `method='dense'`：类似 min，但排名间无间隔（+1 递增）。

## 数据计算与函数

### 1. 算术运算与自动对齐

> Pandas 的核心特性之一是：在进行算术运算时，会自动根据**索引标签**对齐数据。

- **机制**：根据**索引标签**自动对齐。结果索引是两者的**并集**，找不到的位置产生 `NaN`。
- **算术方法 vs 算术符号**：

| **算术符号** | **对应方法 (Method)**      | **反向方法 (Reverse Method)** | **说明**            |
| ------------ | -------------------------- | ----------------------------- | ------------------- |
| `+`          | **`add()`**                | `radd()`                      | 加法                |
| `-`          | **`sub()`**                | `rsub()`                      | 减法                |
| `*`          | **`mul()`**                | `rmul()`                      | 乘法                |
| `/`          | **`div()`** (或 `truediv`) | `rdiv()`                      | 真除法 (结果是小数) |
| `//`         | **`floordiv()`**           | `rfloordiv()`                 | 地板除 (整除)       |
| `%`          | **`mod()`**                | `rmod()`                      | 取模 (求余)         |
| `**`         | **`pow()`**                | `rpow()`                      | 幂运算              |

算数方法示例：

```python
df1.add(df2)
```

- **核心优势**：符号适合简单场景；而方法（如 `add`）可以通过传入 `fill_value=0` 参数，先填充缺失值再计算，避免结果产生`NaN`。

### 2. DataFrame 与 Series 的运算 

- **默认行为**：Series 的索引匹配 DataFrame 的**列**，沿着**行向下广播**（Series 横在上面往下划）。不匹配的形成并集产生 `NaN`。
- **改变方向**：若想匹配**行索引**，必须使用算术方法并指定轴：`frame.sub(series, axis='index')` 或 `axis=0`（Series 竖在旁边往右划）。

### 3. 函数应用

- **内置函数**：如 `sum()`, `mean()`, `median()`, `max()`, `idxmax()`（返回最值的索引）, `count()`（非空计数）, `unique()`, `nunique()`
  - 内置函数底层有优化，速度极快，能解决 90% 需求。
- **自定义函数**：
  - `map()`：针对 Series/DataFrame，**逐元素**作用。常用于字典映射（如把“是否及格”文本转为 1/0）。
  - `apply()`：针对 Series/DataFrame，作用于**整行或整列**。是智能的循环器，无需写 `for`，自动拆分列/行喂给自定义函数并拼回结果。

例：计算每一科目的“极差”（最大值减最小值）

```python
data = {
    '语文': [80, 90, 85],
    '数学': [70, 95, 60],
    '英语': [60, 80, 90]
}
df = pd.DataFrame(data, index=['张三', '李四', '王五'])
#自定义一个函数
def calculate_range(x):
    return x.max - x.min  #x是Series对象，可以使用Series的方法
df.apply(calculate_range,axis = 0)  #逐列应用函数
```

结果：

```
语文    10
数学    35
英语    30
dtype: int64
```

## 处理文本格式数据

### 2.1 常用读取函数全景图

Pandas 提供了一组函数用于将表格型数据读取为 DataFrame。其中最常用的是 `read_csv`。

| **函数**         | **说明**                                            |
| ---------------- | --------------------------------------------------- |
| **`read_csv`**   | 加载带分隔符的数据。默认分隔符为**逗号**。          |
| **`read_table`** | 加载带分隔符的数据。默认分隔符为**制表符 (`\t`)**。 |
| `read_fwf`       | 读取定宽列格式数据（没有分隔符）。                  |
| `read_clipboard` | 读取剪贴板数据（网页转表格神器）。                  |
| `read_excel`     | 读取 Excel (XLS/XLSX) 文件。                        |
| `read_html`      | 读取 HTML 文档中的所有表格。                        |
| `read_json`      | 读取 JSON 字符串中的数据。                          |
| `read_pickle`    | 读取 Python pickle 格式的对象。                     |
| `read_sql`       | 读取 SQL 查询结果。                                 |



### 2.2 `read_csv` 核心参数解析

 `read_csv` 的参数分为四组：

#### 第一组：基础寻址与解码

- **`path` (文件路径)**：告诉 Pandas 读取哪个文件

  > `path`可以省略不写： df = pd.read_csv("数据.csv")
  >
  > 当文件不在工作目录中：使用`../`来表示父目录

- **`sep` (分隔符)**：默认值是逗号 `,`。有些数据是用“制表符（Tab）”或者“分号”隔开的，这时候就要改这个参数。

- **`encoding` (编码方式)**：英文系统通常用 `utf-8`，中文 Windows 里的 Excel 经常搞出 `gbk` 编码。如果你打开文件发现全是**“烫烫烫”**或者乱码，就把这个改成 `'gbk'`。



#### 第二组：表头与索引

- **`header` (表头/列名行)**：“第几行是表头/列名行”。默认值 `0`（第一行是表头）。如果文件里全是数据，没有表头，设为 `None`。
- **`names` (自定义列名)**：“放弃 csv 原本自带的的表头，指定新的列表作为表头”。用法：传一个列表 `['名字', '年龄', '班级']`。

  > 在实际工作中，可能会遇到**”读取文件的某几列，并把这几列的列名改掉“**
  >
  > 但是pandas 的处理顺序是：
  >
  > 1. 先用 `names` 替换表头 → 列名变成 `['ID', '岗位名称', '薪资']`
  > 2. 再用 `usecols` 按原列名筛选 → 找不到 `positionId` 等列了
  >
  > 所以报错：`columns expected but not found: ['positionName', 'positionId', 'salary']`
  >
  > 解决方法：
  >
  > - 使用`usecols=[0,1,17]`，用位置标号代替名字，避免冲突
  > - 先`usecols=["positionId","positionName","salary"]`,之后再用`rename`函数改名
- **`index_col` (索引列)**：放弃默认生成的 0,1,2… 行索引，指定某一列来作为行索引。比如让第一列“学号”变成唯一标识，设 `index_col=0`。



#### 第三组：初步清洗

- **`nrows`(只读前n行)**
- **`skiprows` (跳过行)**：忽略前 n 行，从 n+1 行开始读取。

  > `skiprows`后面除了跟一个数字，还可以跟整数列表、布尔值列表、函数
  >
  > | 传法           | 说明                 | 示例                         |
  > | -------------- | -------------------- | ---------------------------- |
  > | **整数列表**   | 指定要跳过的行索引   | `[0, 2, 4]` 跳过第1,3,5行    |
  > | **布尔值列表** | `True` 表示跳过该行  | `[False, True, False, True]` |
  > | **函数**       | 返回 `True` 表示跳过 | `lambda x: x % 2 == 1`       |
- **`usecols`(读取指定列):**  "A:C"`、`[0, 2, 3]`、`["姓名", "年龄"]

  > 下标是从0开始的
- **`na_values` (替换空值)**：什么字算是`空`？。正常的空值是空的，但有时会写`未知`或`?`。Pandas 不认识这些，你要告诉它：只要看到`?`，就把它当成`NaN`处理。用法:`na_values=['NULL','未知','?']`。
- **`keep_default_na=False`(禁止自动转化NA)：**读取 CSV 文件时，不要把空值自动转换成 `NaN`，而是保留原始的空字符串或特定标记。

  > 如果不使用 `keep_default_na=False`，pandas 默认会将以下值自动识别为 `NaN`：
  >
  > | 值           | 说明             |
  > | ------------ | ---------------- |
  > | `''`         | 空字符串         |
  > | `'#N/A'`     | Excel 错误值     |
  > | `'#N/A N/A'` | Excel 错误值变体 |
  > | `'#NA'`      | Excel 错误值     |
  > | `'-1.#IND'`  | 浮点错误值       |
  > | `'-1.#QNAN'` | 浮点错误值       |
  > | `'-NaN'`     | 负 NaN           |
  > | `'-nan'`     | 负 nan           |
  > | `'1.#IND'`   | 浮点错误值       |
  > | `'1.#QNAN'`  | 浮点错误值       |
  > | `'N/A'`      | 常见缺失标记     |
  > | `'NA'`       | 常见缺失标记     |
  > | `'NULL'`     | 数据库空值       |
  > | `'NaN'`      | Not a Number     |
  > | `'n/a'`      | 小写变体         |
  > | `'nan'`      | 小写变体         |
  > | `'null'`     | 小写变体         |
  >
  > **什么时候会用到这个参数？**
  >
  > | 原始数据         | 自动转 NaN 后 | 问题                           |
  > | ---------------- | ------------- | ------------------------------ |
  > | `""`（空字符串） | `NaN`         | 分不清是"没填"还是"真为空"     |
  > | `"NULL"`         | `NaN`         | 丢失了原始标记                 |
  > | `"NA"`           | `NaN`         | 可能"NA"是正常值（如国家代码） |
  > | `-999`           | `NaN`         | 可能`-999`是业务上的特殊标记   |
- **`na_filter = false`(关闭缺失值检测机制)**

  > `keep_fault_na`和`na_filter`有什么区别？
  >
  > | 参数              | 作用                       | 效果                                          |
  > | ----------------- | -------------------------- | --------------------------------------------- |
  > | `keep_default_na` | 控制**默认缺失值列表**     | `False` 时不用默认列表，但 `na_values` 仍有效 |
  > | `na_filter`       | 控制**整个缺失值检测机制** | `False` 时完全关闭检测，包括 `na_values`      |
- **`parse_dates` (解析日期)**：“把字符串变成真正的日期”。开启此参数，Pandas 会把 `2023-01-01` 变成“时间对象”，方便做“下个月”等时间计算。
- **`dtype`(强制数据类型)：**设置 `dtype={'股票代码': str}` 保留文本格式。



#### 第四组：大文件处理

- **`nrows` (读取行数)**：只读取一部分数据。场景：文件有 1 亿行，只想读前 5 行看看长什么样，防止内存爆掉。
- **`chunksize` (块大小)**：分批次处理。设置后 Pandas 会把数据切块读取（如一次 1000 行），适合用于循环处理。
  例如：`data = pd.read_csv("某招聘网站数据.csv", chunksize= 1000)`



### 2.3 将数据保存为`csv`文件

处理完数据后，我们通常需要保存结果。使用 `to_csv` 函数。

相关参数：

- `path_or_buf`：保存文件的路径或文件对象。如果不指定，则返回 CSV 格式的字符串。
- `sep`：字段分隔符，默认为逗号。
- `na_rep`：缺失值的表示，默认为空字符串。
- `columns`：要写入文件的列，默认为所有列。
- `header` / `index`：是否将列名/索引写入文件，默认为 True。
- `mode`：打开文件的模式，如 `'w'`（覆盖写入）。

## 处理读取MySQL数据

### 4.1 建立数据库连接

连接数据库需要两个工具：数据库驱动 `pymysql` 和连接引擎 `SQLAlchemy`。核心是使用 `create_engine` 函数生成连接对象。

建立连接的代码模板如下：

```python
from sqlalchemy import create_engine
import pandas as pd

# 格式：数据库类型+驱动://用户名:密码@主机IP:端口/数据库名
db_url = "mysql+pymysql://root:123456@localhost:3306/test_db"
engine = create_engine(db_url)
df = pd.read_sql("SELECT * FROM users", engine)
```

**⚠️ 避坑指南：密码包含特殊字符**

如果密码里有 `@`、`:` 等特殊字符，会破坏 URL 结构。必须先用 `urllib.parse.quote_plus` 对密码进行编码：

```python
import urllib.parse  # 引入 URL 处理库

# 假设你的密码很复杂
password = "My@Password:123"

# 先编码
encoded_password = urllib.parse.quote_plus(password)

# 然后再拼接到 URL 里
db_url = f"mysql+pymysql://root:{encoded_password}@localhost:3306/test_db"

engine = create_engine(db_url)
```

### 4.2 `read_sql`函数详解

| **参数**          | **含义与实战解析**                                           |
| ----------------- | ------------------------------------------------------------ |
| **`sql`**         | **SQL语句 或 表名**。最核心参数。可写查询语句 `"SELECT * FROM users WHERE age > 18"`，也可直接写表名 `"users"`。 |
| **`con`**         | **连接对象**。通常是 `engine` 对象，告诉 Pandas 连哪个库。   |
| **`index_col`**   | **索引列**。指定某一列（如 `id` 或 `date`）直接变成 DataFrame 的索引，非常常用。 |
| **`parse_dates`** | **解析日期**。告诉 Pandas 哪几列是时间，直接转为 `datetime` 类型。 |
| **`params`**      | **查询参数**。用于安全传递参数防 SQL 注入。如 `sql="SELECT * FROM table WHERE id = %s"`, `params=[10]`。 |
| **`chunksize`**   | **分块读取**。表太大数据撑爆内存时，设置 `chunksize=10000` 返回迭代器，分批处理。 |

# 第三章：数据清洗和准备 

## 缺失数据的处理

> 遇到缺失数据，第一步永远是“诊断”它有多少，第二步才是决定“删掉”还是“填补”。

### 1. 缺失值的表示与检测

- **数值数据**：使用浮点值 `NaN` (Not a Number) 作为“哨兵值”。

- **对象/字符串**：使用 `NaN` 或 Python 内置的 `None`。

- **统计特性**：Pandas 的描述性统计方法（如 `sum`, `mean`）默认都会**排除**缺失数据。

- **检测方法**：

  - `isnull()`：返回布尔对象（True 表示缺失）。

  - `notnull()`：isnull 的否定式（True 表示非空）。

    >  输出结果是一个形状一样的表格，来表示每个位置是否是`NaN`

### 2. 过滤缺失数据 

`dropna`：用于果断丢弃含有缺失值的行或列

- **Series 操作**：`data.dropna()` 返回一个仅含非空数据和索引值的 Series。这等价于 `data[data.notnull()]`。
- **DataFrame 操作**：
  - **丢弃行 (默认)**：`data.dropna()`。只要某行包含**任何** NaN，该行就会被丢弃。
  - **只丢弃全空行**：`data.dropna(how='all')`。
  - **丢弃列**：传入 `axis=1` 或 `axis='columns'`（如 `data.dropna(axis=1, how='all')`）。
  - **阈值控制 (`thresh`)**：`thresh=n` 表示保留至少有 `n` 个非 NA 值的行（即丢弃非 NA 值少于 n 的行）。例如 `df.dropna(thresh=2)`。

### 3. 填充缺失数据

`fillna`：用于将缺失值替换为指定值或使用插值方法填充。

**基础填充方式：**

- **常数填充**：`df.fillna(0)`（将所有 NaN 替换为 0）。
- **字典填充**：为不同的列指定不同的填充值，如 `df.fillna({1: 0.5, 2: 0})`（第 1 列填 0.5，第 2 列填 0）。
- **统计值填充**：使用平均值或中位数填充，如 `data.fillna(data.mean())`。

**参数解析 **

- **`method` (如何填充？)**
  - **`ffill` (Forward Fill)**：**“抄上面的”**。比如如果不记得今天股价多少，就默认等于昨天的，这是处理时间序列最常用的。
  - **`bfill` (Backward Fill)**：**“抄下面的”**。（比较少用，因为通常无法预知未来）。
- **`axis` (填充方向)**
  - **`axis=0` (默认)**：**竖着填**（按行操作）。用 `ffill` 就是竖着看，把上一行的值拿下来填。
  - **`axis=1`**：**横着填**（按列操作）。拿左边列的值填右边列的坑（较少用）。
- **`inplace` (是否就地修改)**
  - **`False` (默认)**：返回一张修补好后的**新表格**，**原表格**不变，不赋值就白做了。
  - **`True`**：直接在**原表格**上修改，空值被彻底覆盖。
- **`limit` (连续填充上限)**
  - **场景**：用 `ffill` 时，如果中间**连续空了 100 行**（如传感器坏了 3 天），全填 3 天前的数据太假了。
  - **用法**：设置 `limit=2` 表示遇到连续的空值，**最多允许抄 2 次**，后面的继续保留空缺，因为参考价值不大。

## 更改列/行名与异常处理

> **分析师视角**：现实业务中，各种拼写错误、重复打点日志、离群极大值层出不穷。这一模块的工具专门用来给数据“正骨”。

### 1. 删除重复数据

- **检测**：`duplicated()` 返回一个布尔型 Series，表示各行是否是前面出现过的**重复行**。
- **删除**：`drop_duplicates()` 返回移除了重复行的 DataFrame。
- **参数**：
  - 默认判断**全部列**。
  - 传入列名列表（如 `['k1']`）只根据**指定列**判断重复。
  - `keep`：默认保留**第一个**出现的值；`keep='last'` 保留**最后一个**出现的值。

### 2. 数据的映射与替换

- **利用函数或映射 (`map`)**：
  - **场景**：根据列中的值实现转换（如将“肉类名”映射为“动物名”）。
  - **参数 `arg`**：可以是**字典**（定义映射关系，如 `{'bacon': 'pig'}`），也可以是**函数**（对每个元素应用）。
  - **实战注意**：映射前通常需先统一数据格式（如 `str.lower()` 转小写），确保匹配成功。
- **替换值 (`replace`)**：`fillna` 是替换空值的特例，而 `replace` 是更通用的替换功能。
  - **单值替换**：`data.replace(-999, np.nan)`。
  - **多值替换**：
    - 列表对单值：`data.replace([-999, 1000], np.nan)`。
    - 列表对列表（一一对应）：`data.replace([-999, 1000], [np.nan, 0])`。
    - 字典：`data.replace({-999: np.nan, 1000: 0})`。

### 3. 重命名索引/列名和行名

- **`map` 方法**：轴索引对象（Index）可使用 `map` 格式化（如转大写：`data.index = data.index.map(str.upper)`，此为就地修改）。

- **`rename` 方法**：结合字典更新部分标签，返回新对象。例如 

  ```python
  data.rename(
      index={'id': '工号'}, 
      columns={'name': '姓名'})
  ```

  加参数 `inplace=True` 可就地修改。

### 4. 检测和处理异常值

**核心思想**：运用数组运算。

- **第一步：检测异常值**。例如找出绝对值大于 3 的值：`np.abs(data) > 3`。

- **第二步：找出包含异常值的行**。使用 `data[(np.abs(data) > 3).any(1)]`。

  - **原理解释**：假设有如下表格：

    | | A | B | C |

    | :--- | :--- | :--- | :--- |

    | **行0** | 0.1 | 0.5 | -0.2 |

    | **行1** | 0.2 | **5.5** | 0.3 |

    | **行2** | -0.1 | 0.1 | **-4.0** |

  - `np.abs(data) > 3` 会变成全是布尔值的表格。

  - `.any(1)` 中的 `1` 代表 **axis=1 (横向/按行扫描)**，`any` 代表**“只要有一个是 True，就算 True”**。最终会把包含异常值的**行1**和**行2**筛选出来。

- **第三步：处理异常值**。将其修正到合理的值，如 `data[np.abs(data) > 3] = np.sign(data) * 3`。

## 特征构建与高级排布

> **分析师视角**：这一步主要是为了做统计建模和画图服务。把连续值分箱、把类别转为 0/1 矩阵，是喂给机器学习模型的前置步骤。

### 1. 数据分类 (分箱/离散化)

作用是将**连续的数值**按照大小，映射成有限的**类别标签**。例如把 1 到 100 岁的年龄化为“未成年/青年/中年/老年”，方便分析与制图。二者均返回 `Categorical` 对象。

| **函数**     | **pd.cut (按数值距离划分)**                                  | **pd.qcut (按数量比例划分)**                                 |
| ------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **核心逻辑** | 关注**“数值的大小”**。如果均分 4 组，每组数值跨度一样大      | 关注**“排名的比例”**（分位数）。均分 4 组保证**每一组里的人数一样多**。 |
| **分割参数** | `bins`：传整数（自动等分）或传列表 `[0, 60, 100]` **手动指定划分点**（ | `q`：传整数（按四分位数切分）或传比例列表 `[0, 0.8, 1]`（前80%一组，后20%一组）。 |
| **开闭区间** | `right`：默认 `True`，即 `(a, b]` 左开右闭（0 不算，60 算）。 | /                                                            |
| **标签命名** | `labels`：设置标签名称（如 `['不及格', '及格']`）。          | `labels`：设置标签名称。                                     |

### 2. 计算指标 / 哑变量 

常用于统计建模或机器学习。

- **基础转换**：`pd.get_dummies(Series)` 将分类变量转换为 0/1 指标矩阵。如果一列有 k 个不同值，则派生出 k 列。
- **列名前缀**：使用 `prefix='key'` 给生成的列名加前缀，便于后续合并。
- **结合分箱**：经常与分类函数连用：`pd.get_dummies(pd.cut(values, bins))`。
- **多重归属难题**：例如一部电影类别是 "Action|Comedy"。需要结合字符串拆分迭代：
  1. 构建全零矩阵。
  2. 使用 `dummies.columns.get_indexer` 获取对应列的索引。
  3. 将对应位置设为 1。

### 3. 排列和随机采样

- **随机重排**：先用 `np.random.permutation(length)` 生成随机整数序列，再用 `df.take(sampler)` 或 `df.iloc[sampler]` 按序列重排数据。
- **随机采样**：`df.sample(n=3)` 随机抽取 3 行。`replace=True` 允许重复选择（有放回采样）。

## 处理字符串与文本

> **分析师视角**：处理文本数据是进阶必备技能。Pandas 的 `.str` 方法是最强辅助，它不仅让你能直接批量操作字符串，还能完美避开空值报错。

### 1. Python 内置字符串方法

适合大部分基础文本操作。

- **拆分连接**：`split(',')` 拆分字符串，可实现**分列**；`'::'.join(list)` 将序列连接，比 `+` 更快。
- **去空白**：`strip` 去除两端空白及换行，`lstrip`/`rstrip` 仅去一侧。常结合：`[x.strip() for x in val.split(',')]`。
- **子串查询**：`in` 关键字检测是否存在（最佳）；`index` 找首字符位置（找不到会抛 `ValueError`）；`find` 找首字符位置（找不到返回 `-1`）；`count` 统计次数。
- **替换转换**：`replace`（传入 `''` 可实现删除模式）；`lower`/`upper`（大小写）；`casefold`（通用可比较形式，比 lower 更激进）。

### 2. 正则表达式 (`re` 模块)

- **核心函数**：
  - `re.compile`：编译正则对象。**强烈建议**：如果对大量字符串应用同一正则，预编译极省 CPU 时间。
  - `re.split('\s+', text)`：按模式（如任意数量空白）拆分。
  - `findall`：返回所有非重叠匹配列表。若有分组 `()`，则返回元组列表。
  - `search`：返回第一个匹配对象（含位置）。
  - `match`：只匹配首部，首部不符则返回 None。
  - `sub`：替换模式。可使用 `\1`, `\2` 引用匹配的分组内容。
- **分组提取**：用 `()` 包裹提取部分，调用匹配对象的 `groups()` 返回元组。

### 3. Pandas 矢量化字符串函数 

`Series.str`:对整组数据应用字符串操作，**能自动处理并跳过缺失数据 (NaN)**，避免报错。

- **基础与切片**：`data.str.contains('gmail')` 返回布尔 Series；`data.str.findall(pattern)` 返回列表 Series；`data.str.match(pattern)` 应用正则匹配。可直接切片 `data.str[:5]` 或获取元素 `data.str.get(i)` / `data.str[0]`。
- **常用核心方法清单**：

| **方法**                    | **说明**                              | **方法**                  | **说明**                             |
| --------------------------- | ------------------------------------- | ------------------------- | ------------------------------------ |
| **`cat`** / **`join`**      | 元素级字符串连接 / 根据分隔符连接元素 | **`contains`**            | 返回是否含指定模式的布尔数组         |
| **`count`** / **`len`**     | 计算模式出现次数 / 计算字符串长度     | **`extract`**             | **用带分组正则提取，返回 DataFrame** |
| **`findall`** / **`match`** | 计算模式列表 / 根据正则执行匹配       | **`get`** / **`slice`**   | 获取第 i 个字符 / 子串截取           |
| **`pad`** / **`center`**    | 两边加空白符 / 居中                   | **`repeat`**              | 重复值（如 `x*3`）                   |
| **`replace`**               | 替换模式                              | **`split`**               | 根据分隔符或正则拆分                 |
| **`strip`**                 | 去除两边空白符                        | **`lower`** / **`upper`** | 大小写转换                           |

# 第四章：数据塑形 

## 为什么要数据塑形？

在数据建模的规范里，**事实表必须是“长表”（一维表）**。所以无论是用 Python 写脚本，还是用 Power Query 点按钮，我们最常做的动作其实都是 **逆透视 (Unpivot / melt)**，把人类爱看的二维横向报表，洗成数据库喜欢的一维流水线。

### 1. 宽表VS长表

**宽表（给人类看的报告）**：列数很多，常见于 Excel 表格。

| **分店** | **1月销量** | **2月销量** | **3月销量** |
| -------- | ----------- | ----------- | ----------- |
| 北京店   | 100         | 120         | 150         |
| 上海店   | 80          | 90          | 110         |

**长表（给机器和数据库看的表）**：核心原则是**“每一行只记录一个不可分割的最小业务动作或观察结果”**。列数极少，只有清晰的“属性（维度）”和“数值（度量）”。行数变多了（原来的 2 行变成了 6 行）。

| **分店 (维度)** | **月份 (维度)** | **销量 (度量)** |
| --------------- | --------------- | --------------- |
| 北京店          | 1月             | 100             |
| 北京店          | 2月             | 120             |
| 北京店          | 3月             | 150             |
| 上海店          | 1月             | 80              |
| 上海店          | 2月             | 90              |
| 上海店          | 3月             | 110             |

### 2. 长表的优势分析

- **极度灵活**：如果想算总销量，直接对“销量”这一列做 `SUM()` 即可，不管你有几个月。
- **方便切片**：可以直接拿“月份”这一列作为筛选器（切片器），选“1月”，数据就自动过滤了。
- **扩展性强**：如果 4 月份的数据来了，宽表需要“增加一列”，这会破坏表结构；而长表只需要在最下面“追加几行”即可，表结构坚如磐石。

## 多级索引

> 多级索引（MultiIndex）是处理高维数据（例如“省份-城市-门店”层级结构）的利器。把它想象成文件管理系统中的文件夹路径。

### 1. 多级索引的概念与创建

**含义**：在一个轴（行或列）上拥有两个或两个以上的索引层级。就像一本教科书，有**章**，章下面有**节**，节下面才是**内容**。

- **对象本质**：生成的索引对象是 `MultiIndex`，包含 `levels`（唯一值）和 `labels`（整数标签），它是 `Index` 的**子类**。

**创建方式：**

1. **基于已有列转换**：将已有数据转换为多级索引`set_index([索引名列表])`

   ```python
   data = {
       '班级': ['一班', '一班', '二班', '二班'],
       '姓名': ['张三', '李四', '王五', '赵六'],
       '分数': [80, 90, 85, 95]
   }
   df = pd.DataFrame(data)
   
   # 把“班级”设为第一层，“姓名”设为第二层
   df_multi = df.set_index(['班级', '姓名'])
   # 注意要用另一个变量去接收，原DataFrame不会改变
   ```

2. **数组拉链式拼装**：索引数据分散在两个独立的列表里，把它们“拉链”式地拼起来`pd.MultiIndex.from_arrays`

   `index = pd.MultiIndex.from_arrays(arrays, names=['班级', '性别'])`

   ```python
   # 两个等长的列表
   arrays = [
       ['一班', '一班', '二班', '二班'],  # 第一层
       ['男', '女', '男', '女']         # 第二层
   ]
   
   # 拼装
   index = pd.MultiIndex.from_arrays(arrays, names=['班级', '性别'])
   
   df = pd.DataFrame({'人数': [10, 20, 15, 25]}, index=index)
   print(df)
   ```

   

### 2. 选取与切片速查

| **需求**                            | **推荐方法**     | **代码示例**               |
| ----------------------------------- | ---------------- | -------------------------- |
| **只选最外层** (比如所有 2023)      | **直接 `loc`**   | `df.loc['2023']`           |
| **精准选某一行** (比如 2023 年上海) | **元组 `loc`**   | `df.loc[('2023', '上海')]` |
| **只选内层** (跨层选，比如所有上海) | **利用冒号 `:`** | `df.loc[:,'上海']`         |

### 3. 层级重组

在处理多级索引时，经常需要调整索引的层级顺序（比如把“内层”换到“外层”），这就是 `swaplevel` 的作用。

但调整完顺序后，数据的物理排列往往是乱序的，这会导致切片报错或性能下降，所以紧接着必须用 `sort_index` 来重新整理数据。

- **`swaplevel(i, j, axis=0)`**：**交换层级（换位置）**。

  - **含义**：i, j 是你要交换的两个层级的名称

  - **比喻**：原来按“年份 -> 城市”分类，改成按“城市 -> 年份”分类。

    ```python
    # 1. 原始数据：第一层是年份，第二层是城市
    index = pd.MultiIndex.from_product(
        [
            ['2023', '2024'], 
            ['北京', '上海']], 
        names=['年份', '城市'])
    df = pd.DataFrame({'销量': [100, 200, 150, 300]}, index=index)
    
    # 2. 交换层级：把“城市”换到外层，“年份”换到内层
    df_swapped = df.swaplevel('年份', '城市')
    df_swapped
    ```

    ```
                销量
    城市  年份      
    北京  2023   100  <-- 北京
    上海  2023   200  <-- 上海 (乱入)
    北京  2024   150  <-- 北京 (又回来了)
    上海  2024   300
    ```

  - **隐患**：交换后，数据的物理排列往往是乱序的（北京和上海穿插在一起），这会导致切片报错或性能下降。

- **`sort_index()`**：**重新排序（理顺）**。

  - **作用**：把索引按字典序（A-Z 或 0-9）排列整齐。

  - **语法**：

    ```python
    # level: 指定按哪一层排（通常不写，默认全排）
    # ascending: True(升序) / False(降序)
    df.sort_index(level=0, inplace=True)
    ```

    接上面的代码演示:

    ```python
    df_sorted = df_swapped.sort_index()
    print(df_sorted)
    ```

    ```
               销量
    城市  年份      
    上海  2023   200  <-- 上海的数据聚在一起了
        2024   300
    北京  2023   100  <-- 北京的数据聚在一起了
        2024   150
    ```

  - **必要性**：Pandas 的切片操作强烈依赖索引有序性。如果索引是乱的，`loc` 切片会失效

在实际工作中，这两个函数通常是连在一起写的：

```python
df_final = df.swaplevel('年份','城市').sort_index()
```

## 多表查询

> **分析师视角**：这是日常工作的高频场景。`merge` 用于横向扩展字段（类似 Excel 的 VLOOKUP 或 SQL 的 JOIN）；`concat` 用于纵向追加数据（比如把 1-12 月的 CSV 报表叠在一起）。

### 1. 横向合并

函数：`pd.merge`

可根据一个或多个键将不同的 DataFrame 连接起来，类似于 SQL 中的 JOIN 操作。

- **连接键指定**：
  - 默认将**重叠列的列名**当作键。
  - `on='key'`：指定相同列名。
  - `left_on='lkey', right_on='rkey'`：左右列名不同时分别指定。
  - `left_index=True` / `right_index=True`：使用索引作为连接键。
- **连接方式 (`how` 参数)**：
  - `inner` (默认)：使用两个表都有的键（交集）。
  - `left` / `right`：使用左表 / 右表中所有的键。
  - `outer`：使用两个表中所有的键（并集）。
  - `cross`：产生笛卡尔积。
- **多对多与重复列名**：多对多产生行的**笛卡尔积**。重叠的非连接键会通过 `suffixes=('_x', '_y')` 自动添加后缀。

### 2. 竖向堆叠 

函数：`pd.concat`

核心思想是沿着某条轴，把两个表硬生生地“粘”在一起（如：1月份表 + 2月份表 = 总表）。

- **方向**：`axis=0` (默认，纵向堆叠)；`axis=1` (横向堆叠)。
- **连接逻辑**：`join='outer'` (默认并集)；`join='inner'` (交集)。
- **实用参数进阶**：
  - **`keys`**：在连接轴上创建**多级索引**，用于区分数据来源（如区分数据属于哪个月）。
  - **`ignore_index`**：如果不保留连接轴上的索引，产生新整数索引 `0, 1, ..., n-1`。
  - **`verify_integrity`**：检查新轴上的重复情况（默认 False）。

### 3. 合并重叠数据

函数：`combine_first`

用于处理两个数据集索引部分重叠，需要修补的情况。

- **逻辑**：优先用自己的数据，如果自己是 NaN，就用别人的数据补上（同时带有数据自动对齐功能）。
- **示例**：`df1.combine_first(df2)` 用 df2 修补 df1 的空缺。

## 重塑与轴向旋转

> **分析师视角**：当你能熟练区分并在脑海中可视化 `stack/unstack` 和 `pivot/melt` 的变形过程时，你的 Pandas 功底就已经超越了 80% 的初学者。

### 1. 依托于多级索引的重排

多级索引为数据重排提供了一致性方式。

- **`stack()` (堆叠)**：把最里面的**列索引 (Columns)** 旋转 90 度，压到**行索引 (Index)** 的最内层。
  - 结果：表格变长、变瘦。默认滤除缺失数据（NaN）使运算可逆（可传 `dropna=False` 保留）。
- **`unstack()` (拆堆)**：把最里面的**行索引 (Index)** 旋转 90 度，推到**列索引 (Columns)** 上去。
  - 结果：表格变宽、变扁。如果级别值未在各分组找全，会引入缺失数据 (NaN)。

### 2. 透视与逆透视

时间序列或数据库多为长表，分析制图多需宽表。

**将“长表”旋转为“宽表”/透视 (`pivot`)**

- `index`：指定作为新表**行索引**的列（类似 SQL 的 `GROUP BY`）

  > 相当于 SQL 里 `GROUP BY` 后面的那个字段。

- `columns`：指定作为新表**列名**的列

  > 即把原来**竖着排一列的数据横过来，作为表头**

- `values`：填入**单元格**的数据列

  > 如果不填这个参数，Pandas 会默认把**剩下所有列**都作为数据填进去（形成多级**列**索引，其中`columns`的列是最内层）。

- *底层本质*：`pivot` 等价于 `set_index` 再 `unstack`。

**场景：有一份简单的销售数据（长表）**

```python
data2 = {
    '日期': ['1月1日', '1月1日', '1月2日', '1月2日'],
    '商品': ['苹果', '香蕉', '苹果', '香蕉'],
    '销量': [100, 200, 150, 300],
    '价格': [5, 3, 5, 3]
}
df2 = pd.DataFrame(data2)

print("--- 原始长表 ---")
print(df2)
```

原始数据：

```
    日期  商品   销量  价格
0  1月1日  苹果  100   5
1  1月1日  香蕉  200   3
2  1月2日  苹果  150   5
3  1月2日  香蕉  300   3
```

要求：我想看“不同日期中，各个商品的销量”

```python
df2_pivoted = df2.pivot(index='日期', columns='商品', values='销量')
print(df2_pivoted)
```

结果：（宽表）

```
商品    苹果   香蕉
日期           
1月1日  100  200
1月2日  150  300
```

如果不填`values`参数：形成多级索引

```
		 销量       	 价格     
商品   苹果   香蕉   苹果  香蕉
日期                     
1月1日  100  200    5   3
1月2日  150  300    5   3
```



**将“宽表”旋转为“长表”/逆透视 (`melt`)**

`pivot` 的逆运算，将多个列合并为一个，产生比输入更长的 DataFrame。

- `id_vars` (身份列/锚点)：不想融化的列，作为**标识符**保留（如姓名）

  > `id_vars`参数的那一行就是在变成长表时，自动向下复制填充（拉伸）的那一行，即`pivot()`中的`index`参数的反面：
  > 假如我对一张表进行了pivot，如果我想使用melt函数再把它变回原样，我pivot的index参数和melt的id_vars参数应该是同一个列名

- `value_vars` (要融化的列)：要把哪些列表头折叠进一列（默认融化除 `id_vars` 外所有列）。

- `var_name`：给融化后的表头（新产生的一列）起的新变量名（如“科目”）。

- `value_name`：给融化后的数值产生的新一列起的名字（如“分数”）

**场景：有一张成绩单(宽表)**

```python
df = pd.DataFrame({
    '姓名': ['张三', '李四'],
    '语文': [90, 80],
    '数学': [85, 95]
})

print("--- 原始宽表 (Wide) ---")
print(df)
```

结果

```python
   姓名  语文  数学
0  张三  90  85
1  李四  80  95
```

把“语文”和“数学”这两列融化掉，变成“科目”，剩下的数据形成新的一列，取名“分数”

```python
df_melted = pd.melt(df, 
                    id_vars=['姓名'], 
                    value_vars=['语文', '数学'],
                    var_name='科目', 
                    value_name='分数')

print("\n--- 融化后的长表 (Long) ---")
print(df_melted)
```

```
   姓名  科目  分数
0  张三  语文  90  <-- 张三的第一条数据
1  李四  语文  80
2  张三  数学  85  <-- 张三的第二条数据 (姓名重复了)
3  李四  数学  95
```



### 3. 四大变形函数对比

| **特性**     | **pivot**             | **melt**                  | **stack**                 | **unstack**                 |
| ------------ | --------------------- | ------------------------- | ------------------------- | --------------------------- |
| **变换方向** | **长 -> 宽**          | **宽 -> 长**              | **宽 -> 长**              | **长 -> 宽**                |
| **操作对象** | **列里的值** (Values) | **列的名字** (Headers)    | **列索引** (Column Index) | **行索引** (Row Index)      |
| **主要用途** | 制作透视表、报表展示  | 数据清洗、为绘图/ML做准备 | 多级索引处理、矩阵压缩    | `groupby`结果展开，矩阵展开 |
| **索引变化** | 指定列变为行/列索引   | 不涉及复杂索引变化        | **列索引变行索引**        | **行索引变列索引**          |
| **逆运算**   | 对应 `melt`           | 对应 `pivot`              | 对应 `unstack`            | 对应 `stack`                |

# 第五章：聚合与分组运算 

## 分组：`groupby`

Hadley Wickham（R 语言大神）提出的 **Split-Apply-Combine（拆分-应用-合并）** 是整个现代数据分析的灵魂所在。

### 1. `GroupBy` 基础用法

**group by的过程逻辑（与MySQL相同）：**

- **第一步：拆分 (Split)**：将 Pandas 对象中的数据，根据提供的一个或多个键 (Key) 在特定轴上（如行 `axis=0`）拆分为多组。
- **第二步：应用 (Apply)**：将函数应用到每一个独立的分组上，产生新值。
- **第三步：合并 (Combine)**：将所有函数的执行结果合并到最终的结果对象中。

**基础语法格式**：

```python
df.groupby('按谁分组')['对谁计算'].怎么算()
```

> **⚠️ 核心特性注意：**
>
> - 调用 `df.groupby('班级')` 返回的 `GroupBy` 对象**没有进行任何计算**，只含有分组键的中间数据。
> - 如果不写 `['对谁计算']`，默认：**把你表格里除了“分组列”之外的所有剩余列，全都一股脑地拉过来，强行执行那个“怎么算”的操作。**
> - 作为分组依据的键中，**任何缺失值（NA）都会被自动从最终结果中剔除**，不会作为单独的一组出现。
> - **强烈建议：**多列分组时会在最后产生多级索引，加上 `.reset_index()` 可以把它变回清爽的普通二维表格。



### 2. 分组方式

Pandas 极其灵活，不仅能按列名分组，还能用外部数据分组：

| **分组方式**       | **语法示例**                                        | **实战应用场景解析**                                         |
| ------------------ | --------------------------------------------------- | ------------------------------------------------------------ |
| **按单列/多列**    | `df.groupby('key1')`/`df.groupby(['key1', 'key2'])` | **最常用**。多列分组会产生多级索引，即用两个键的组合来唯一标识一行 |
| **按外部数组**     | 传入与行数等长的 NumPy 数组                         | 只要数量对得上，Pandas 就会按顺序把外部数组里的标签，跟表格里的每一行“贴”在一起进行分组。**相当于把数组粘贴上了，构成了临时一列** |
| **按字典映射**     | `df.groupby(mapping_dict)`                          | **临时统计神器**。字典键需对应**行索引 (Index)**。字典没写全的行会被自动忽略，不报错。 |
| **按自定义函数**   | `df.groupby(len)`                                   | 将**行索引**喂给函数（如按名字长度分组）。函数吐出什么结果就分到哪组。 |
| **混合分组**       | 混合函数、字典、列名列表                            | Pandas 会分别计算标签，拼成多级索引 (MultiIndex)。           |
| **按多级索引层级** | `df.groupby(level='cty')`                           | 数据自带多级索引时，直接告诉 Pandas 按哪一级聚合。           |

> **例：按字典映射分组**
>
> 假设你有一张表，以及一个字典，记录了每个员工所属的部门
>
> ```python
> df = pd.DataFrame(
>     {'销量': [100, 200, 150, 300, 250],
>      '退货量': [5, 10, 0, 15, 8]},
>     index=['张三', '李四', '王五', '赵六', '孙七']
> )
> 
> mapping_dict = {
>     '张三': '一部',
>     '李四': '一部',

## 数据聚合：`agg`

> 所谓数据聚合，指的就是把一个数组（多行数据），经过某种数学运算后，**浓缩/合并成一个单一的值**的过程（即**使用聚合函数的过程**）。之前的 `.mean()`、`.sum()`、`.max()`、`.count()`，它们全都是标准的“聚合函数”。
>
> **不是所有的计算都叫聚合**
>
> 为了让你更清晰地理解，我们对比一下**“非聚合”**的计算：
>
> - **聚合计算（多变一）**： 一班有 50 个人，计算这 50 个人的**平均分**。结果只有 1 个数字。
> - **非聚合计算（一对一，比如映射或转换）**： 一班有 50 个人，给这 50 个人的分数**每人加 10 分**。算完之后，结果还是 50 个数字。这就不能叫聚合。

### 1. 官方函数与自定义函数

- **官方底层优化函数**：`count` (非NA值数量), `sum`, `mean`, `median`, `std`/`var`, `min`/`max`, `prod` (乘积), `first`/`last`（首尾非NA值） 。

- **自定义函数**：直接将自定义函数传入 `agg`。例如 `grouped.agg(get_diff)`。

  ```python
  # 自定义函数：计算最大值与最小值的差
  def get_diff(x):
      return x.max() - x.min()
  grouped = df.groupby('班级')
  
  result = grouped.agg(get_diff)
  # 计算每一列的极差
  ```

  

### 2. 同时应用多种函数

> 在实际分析中，我们往往希望**对同一列应用多个函数**，或者**对不同的列使用不同的函数**。`agg`方法对此提供了极其强大的支持。

- **对同一列用多个函数**：传入列表 `grouped['销量'].agg(['sum', 'mean', get_diff])`。**官方函数需加引号，自定义函数不加**。列名会自动变为函数名。

- **不同列用不同函数**：传入字典 `grouped.agg({'销量':'sum', '利润':'mean'})`。

- **自定义结果列名 (强烈推荐)**：传入 `(自定义列名, 函数)` 元组组成的列表。

  ```python
  opt_result = grouped3.agg(
      总销量=('销量','sum'),
      平均利润=('利润','mean'),
      商品种类数=('商品','count')
  )
  ```

- **消除多级索引小技巧**：在 `groupby` 时传入 **`as_index=False`**，可以从一开始就阻止分组列变成索引，直接返回清爽的普通表格。

## `apply` 与 `transform`

> **分析师视角**：当业务逻辑变得复杂（比如：算转化率需要 A 列除以 B 列，或者提取每组的 TOP 3），`agg` 就会报错，此时必须请出 `apply`。



### 1. `agg` 对比`apply`

`agg()` 是“收敛的计算器”（必须多变一），而 `apply()` 是“无所不能的加工厂”（随便你怎么变）。

| **对比维度**              | **agg() (聚合)**                   | **apply() (应用)**                      |
| ------------------------- | ---------------------------------- | --------------------------------------- |
| **底层逻辑**              | **收敛汇总**（多变一）             | **灵活加工**（极其自由）                |
| **函数接收的输入(核心!)** | 当前分组的**每一列** (一维 Series) | 当前分组的**整个子表** (二维 DataFrame) |
| **函数输出结果**          | **单一值** (Scalar)                | 单一值、Series 甚至 DataFrame 都行      |
| **跨列计算能力**          | ❌ 极弱（只能看一列算一列）         | ✅ **极强**（可调用多列互相加减乘除）    |
| **多任务能力**            | ✅ 极强（支持字典法同时算多个）     | ❌ 弱（通常一次执行一个自定义逻辑）      |

> **核心区别：函数“看到”的输入不同**
>
> - **`agg()`：按“列”切片送进去**
>
>   当你在 `groupby` 后使用 `agg(自定义函数)` 时，Pandas 会把当前分组的**每一列**（作为一个一维的 Series）分别、单独地送进你的函数里。你的函数在同一时间，只能看到“某一列”的数据，看不到其他列。
>
> - **`apply()`：把“整个组”打包送进去**
>
>   当使用 `apply(自定义函数)` 时，Pandas 会把当前分组的**整个子表格**（作为一个二维的 DataFrame）一次性端给你的函数。你的函数可以同时调用这个组里的好几列数据来做复杂运算。
>
> **场景对比：**
>
> 假设我们要计算“利润率（利润/销售额）”。
>
> - `agg()` **做不到**。因为它每次只能拿到“利润”这一列或者“销售额”这一列，它没法同时拿着两列相除。
> - `apply()` **可以做到**。因为它拿到的是包含了这两列的整个表格。



### 2. `apply` 的三大应用场景

> 本质上是一个披着 Pandas 外衣的 for 循环。

- **对单列 (Series) 使用**：
  **场景**：有一列“成绩”，你想把大于等于 60 分的变成“及格”，小于 60 分的变成“不及格”。

  ````python
  df = pd.DataFrame({'姓名': ['张三', '李四', '王五'], '分数': [85, 50, 90]})
  
  # 1. 自己写个加工函数
  def check_score(score):
      if score >= 60:
          return '及格'
      else:
          return '不及格'
  
  # 2. 用 apply 把函数“应用”到这列的每一个数据上
  df['状态'] = df['分数'].apply(check_score)
  
  print(df)
  
     姓名  分数   状态
  0  张三  85   及格
  1  李四  50  不及格
  2  王五  90   及格
  ```
  
  ````

- **对整表 (DataFrame) 使用**：必须加上 **`axis=1`**，实现跨列横向运算（如：单价 $\times$ 数量 = 总价）。
  **场景**：表里有“单价”和“数量”，新增一列“总价”（单价 $\times$ 数量）

  ```python
  df2 = pd.DataFrame({
      '商品': ['苹果', '香蕉'],
      '单价': [5, 3],
      '数量': [10, 20]
  })
  
  def sum_money(x):
      return x['单价']*x['数量']
  df2['总价'] = df2.apply(sum_money, axis=1)
  
  print(df2)
  ```

  ```
  姓名  分数   状态
  0  张三  85   及格
  1  李四  50  不及格
  2  王五  90   及格
  ```

- **在 GroupBy 之后使用**：

  **场景（经典面试题）**：找出每个部门业绩排名前 2 的员工。

  **痛点**：`agg()` 算不出排名（因为他是聚合函数）。必须用 `apply()`

  ```python
  df3 = pd.DataFrame({
      '部门': ['销售', '销售', '销售', '技术', '技术', '技术'],
      '姓名': ['A', 'B', 'C', 'D', 'E', 'F'],
      '业绩': [100, 300, 200, 500, 400, 600]
  })
  ```

  ```python
  # 1. 写一个处理“小表格”的函数
  def top_2(group_df):
      # 对进来的小表格按业绩降序排，然后取前两行
      return group_df.sort_values(by='业绩', ascending=False).head(2)
  
  # 2. 分组并 apply
  # 这里的 apply 会把“销售部的小表”和“技术部的小表”分别送进 top_2 函数里
  result = df3.groupby('部门').apply(top_2)
  
  print(result)
  ```

  - *消除额外索引*：如果不需要外层的分组键索引，调用时传入 `group_keys=False`。



### 3. `transform`：窗口函数

- **等价于**：SQL 里的 `OVER (PARTITION BY ...)`。也等同于使用 apply 后，再手动把结果列逐个加回原表。

- **实战场景**：算出每个部门的平均工资，**不浓缩表格**，直接作为新的一列原封不动贴回 1000 行的明细表中，用于计算每个人的工资差额。

  ```python
  df = pd.DataFrame({
      '姓名': ['张三', '李四', '王五', '赵六'],
      '部门': ['销售', '销售', '技术', '技术'],
      '工资': [5000, 7000,  8000, 12000]
  })
  ```

  使用`transform`:

  ```python
  df['部门平均工资'] = df.groupby('部门')['工资'].transform('mean')
  
  # 算一下每个人比部门平均工资高多少
  df['工资差额'] = df['工资'] - df['部门平均工资']
  
  print(df)
  ```

## 透视表与交叉表

### 1. `pivot_table` (数据透视表)

完全对应 Excel 数据透视表的拖拽动作：

- **`index`**：拖到“行”区域（左侧表头）。
- **`columns`**：拖到“列”区域（上方表头）。
- **`values`**：拖到“值”区域（用来计算的数字列）。
- **`aggfunc`**：值的汇总方式，默认 `'mean'`，可改 `'sum'`, `'count'` 或传字典。
- **`margins=True`**：自动生成总计(All)行列。
- **`fill_value`**：填补空白单元格（如填 `0`）。

**对决 `groupby`**：

在底层源码里，`pivot_table` 其实就是调用了 `groupby`。 `groupby` 配合 `unstack()` 也能做二维表，但 `pivot_table` 的优势在于**极简生成总计行列**和**一键填补空缺值**。不过，`pivot_table` 仅限于聚合，而 `groupby` 还能进行转换和过滤。

### 2. `crosstab` (数据交叉表)

专门用来**“数人头”（统计频次）**的顶层函数。

- **基础陷阱**：必须传入具体的列对象（如 `df['列名']`），而不是列名字串。
- **杀手锏参数 (`normalize`)**：用于计算百分比。
  - `'all'`：全局占比。
  - **`'index'` (最常用)**：行的百分比（如各个部门内的男女比例）。
  - `'columns'`：列的百分比。

## Pandas 独有功能

Pandas 是用 Python 写的，这意味着它不仅能做关系型代数，还能做大量复杂的数据结构转换和科学计算。以下这些函数，如果在 MySQL 里写，往往需要几百行复杂的 SQL 语句，而在 Pandas 里只需一行

### 1. 结构重塑

这部分是数据清洗师最爱的功能，MySQL 里需要用无数个 `SUM(CASE WHEN...)` 才能勉强实现。

- **`pd.pivot_table()`**：数据透视表（长表转宽表，带行列总计）。
- **`pd.melt()`**：逆透视（宽表变长表），这是机器学习特征工程前必做的动作。
- **`pd.crosstab()`**：交叉频次表，秒算各类别的占比 (`normalize='index'`)。

### 2. 高阶缺失值处理

MySQL 遇到 NULL 通常只能用 `COALESCE` 给个默认值，但 Pandas 提供了插值等高级玩法：

- **`df.fillna(method='ffill')`**：向下填充（比如把上面的“北京”自动填充给下面的空单元格）。
- **`df.interpolate()`**：线性插值。如果第一天卖了 10，第三天卖了 30，第二天是空值，它会自动帮你算出 20 填进去。

### 3. 文本与时间操作

Pandas 有两个超级命名空间：`.str` 和 `.dt`。

- **`df['时间'].dt.day_name()`**：直接告诉你这一天是星期几。
- **`df['文本'].str.extract(r'([A-Z]\d{3})')`**：完美融合正则表达式，瞬间从杂乱的一大段文本中把包含“一个大写字母加3个数字”的特定编号提取成新的一列。

### 4. 自定义函数

- **`df.apply(自定义函数)`**：这就是我们在上一个回合讲的，如果所有的内置规则都搞不定，直接扔一个 Python 函数进去暴力遍历，MySQL 虽然有自定义函数，但在易用性和生态上完全无法与 Python 匹敌。





# Pandas 链式编程

和 SQL 里的视图或者很多原地生效的编程命令不同，Pandas 非常“谨慎”。默认情况下，Pandas 里的绝大多数函数（比如 `rename`, `drop`, `fillna`）都**不会**修改原本的表格。想要对函数生效后的表格进行下一步操作，就必须用一个新的变量去接收它。

> 在实际工作中，经常需要对同一个表进行多个操作，如果中间每个操作都要一个新的变量来接收的话，就会产生大量的临时变量，这样难写也难读

**解决方法：链式编程**

Pandas支持链式编程，多个操作之间可以用`.`来连接，避免产生过多的中间临时变量

## 处理换行

Python 原生是不允许代码随便换行的。如果用 `.` 一直连写下去，代码会变成长长的一条线，极其反人类。

**解决方法：** 在整段代码的最外层套上一个大括号 `()`。之后就可以在每个 `.` 前面自由换行，写出像 SQL 一样层次分明的代码

```python
# 优雅的链式排版，结构就像 SQL 一样清晰
result = (
    df
    .query("age > 18")           # 相当于 WHERE
    .groupby('department')       # 相当于 GROUP BY
    .agg({'salary': 'mean'})     # 相当于 SELECT AVG(salary)
    .reset_index()               # 把索引还原成普通列
    .sort_values('salary')       # 相当于 ORDER BY
)
```

> 虽然排版看起来像`mysql`，但是这些命令的执行顺序是按照你编写的顺序从上到下执行，而不是像mysql那样每个命令之间有先后关系



## 新增列

在传统的写法里，只要你想新增一列，就必须打断链条，单独写一行 `df['new_col'] = df['A'] + df['B']`。 `assign()` 就是专门为了在链条中**“顺手”新增列**而发明的，它等价于 SQL 里的 `SELECT *, A + B AS new_col`

```python
# 在链条中直接新增一列，无需打断流水线
result = (
    df
    .query("status == 'active'")
    .assign(total_income = df['base'] + df['bonus']) # 直接派生新列
)
```
